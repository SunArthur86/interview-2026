# 什么是用户自定义类？

用户自定义类是 Java 程序的基本构建单元，主要由字段（数据）和方法（行为）组成。

### 1. 类的基本结构
*   **实例字段**：存储对象的状态（数据）。
*   **方法**：定义对象的行为，包括构造器、实例方法等。
*   **封装**：通常将字段设为 `private`，通过 `public` 方法（getter/setter）访问，保护内部数据。

### 2. 关键概念
1.  **构造器**：
    *   名字与类名相同，无返回值。
    *   只能配合 `new` 关键字调用，用于初始化对象。
2.  **隐式参数与显式参数**：
    *   **隐式参数**：调用方法的对象本身（即 `this`）。
    *   **显式参数**：方法括号内传入的参数。
3.  **可变对象引用的安全性**：
    *   如果 getter 返回一个可变对象（如 `Date`、数组），可能会破坏封装性（外部可修改内部状态）。
    *   **解决**：返回对象的克隆副本，而不是原始引用。

### 3. 使用注意事项
*   **null 检查**：对于可能为 null 的参数或字段，应进行防御性检查（如 `Object.requireNonNull`）。
*   **不可变对象**：尽量设计为不可变类（如 `String`，所有字段 `final`，无修改方法），线程安全且简单。

### 4. 实战深化

**实战案例**：在 DTO 返回给前端时，直接返回内部 `Date` 引用导致前端无意间修改了服务端的时间对象，造成缓存 Key 错乱。修复方案是使用 Defensive Copy 返回新对象，或使用 Java 8 的 `Instant` 等不可变类型。

**代码示例 (防御性拷贝)**：
```java
public class UserSchedule {
    private final Date start;

    public UserSchedule(Date start) {
        // 构造器也需防御性拷贝，防止外部引用修改
        this.start = new Date(start.getTime());
    }

    public Date getStart() {
        // 返回副本，保护内部状态
        return (Date) start.clone();
    }
}
```

**代码示例 (Builder 模式与不可变性)**：
```java
public final class ImmutableUser {
    private final String name;
    private final int age;

    private ImmutableUser(Builder builder) {
        this.name = builder.name;
        this.age = builder.age;
    }

    public static class Builder {
        private String name;
        private int age;
        // setter... 
        public ImmutableUser build() {
            return new ImmutableUser(this);
        }
    }
}
```


## 记忆要点
- 封装原则：字段设 private，通过 public 方法（getter/setter）访问保护数据。
- 参数对比：隐式参数是对象本身（this），而显式参数是括号内传入的值。
- 防御性拷贝：getter 返回可变对象（如Date）需返回克隆副本，防止破坏封装。
- 不可变设计：尽量全设为 final 并搭配 Builder 模式，确保对象的线程安全。

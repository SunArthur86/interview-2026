# defer 和 async

### defer 和 async 的区别

两者都是用于异步加载外部脚本（`<script src="...">`），不会阻塞 HTML 解析。它们的区别在于**执行时机**和**依赖关系**的处理。

#### 1. 普通脚本 (`<script>`)
- **行为**：HTML 解析遇到 `<script>` 标签时，暂停解析，下载并执行脚本，执行完毕后继续解析 HTML。
- **缺点**：阻塞页面渲染。

#### 2. async (`<script async>`)
- **下载**：异步下载，不阻塞 HTML 解析。
- **执行**：下载完成后**立即**执行，且会阻塞 HTML 解析。
- **时机**：无法保证执行顺序，哪个先下载完先执行哪个。
- **适用场景**：无依赖的独立第三方脚本（如百度统计、Google Analytics）。

#### 3. defer (`<script defer>`)
- **下载**：异步下载，不阻塞 HTML 解析。
- **执行**：等待 HTML 解析完成后，在 `DOMContentLoaded` 事件之前按照在文档中的顺序依次执行。
- **时机**：保证执行顺序。
- **适用场景**：有依赖关系的 DOM 操作脚本或需要操作 DOM 的库（如 jQuery）。

#### 脚本加载与执行流程图

```
时间轴 ────────────────────────────────────────────────────────>

[普通 Script]
  HTML解析 ======[阻塞]======> [下载] -> [执行] ======[继续]======>

[Async Script]
  HTML解析 ==========================> [继续] ========================>
                     ↓ (并行)            ↓ (下载完随时暂停解析执行)
                   [下载A]...[下载完A] -> [执行A]
                   [下载B]............[下载完B] -> [执行B] (顺序不保证)

[Defer Script]
  HTML解析 ============================================> [解析完毕] ┐
                     ↓ (并行)                                        │ [DOMContentLoaded]
                   [下载A] [下载B]                                   ↓ 之前
                        │     │                                       │
                        ▼     ▼                                       ▼
                   [按顺序排队] ====> [执行A] -> [执行B] ==========> [触发 DOMContentLoaded]
```

### 四、实战案例与代码

**实战案例**：
某项目中引入了百度统计 (`async`) 和 jQuery 库 (`defer`)。偶尔出现 jQuery 未定义的错误，导致页面功能失效。这是因为页面加载快时，jQuery 可能还未下载完或未到执行时机，而依赖 jQuery 的业务代码（使用 defer 或放在底部）就已经开始执行；或者如果有其他 async 脚本修改了全局变量，也会导致竞态问题。最终将所有核心业务库统一改为 `defer` 并调整加载顺序解决了问题。

**代码示例**：
```html
<!-- 推荐的脚本加载策略 -->

<!-- 1. 关键且无依赖的第三方统计，使用 async -->
<script async src="https://analytics.com/script.js"></script>

<!-- 2. 核心框架与依赖库，使用 defer 保证顺序 -->
<script defer src="/js/vendor/jquery.min.js"></script>
<script defer src="/js/vendor/bootstrap.min.js"></script>

<!-- 3. 业务逻辑，defer 且依赖于上面的库，按顺序执行 -->
<script defer src="/js/main.js"></script>
```

### 对比表格

| 特性 | `<script>` | `<script async>` | `<script defer>` |
| :--- | :--- | :--- | :--- |
| **是否阻塞 HTML 解析** | 是 (下载+执行) | 否 | 否 |
| **下载时机** | 同步，立即 | 异步，并行 | 异步，并行 |
| **执行时机** | 立即 (下载后) | 立即 (下载后，但在 `load` 前) | `DOMContentLoaded` 之前，按顺序 |
| **执行顺序** | 按文档顺序 | 无序 (看网络速度) | 按文档顺序 |
| **DOMContentLoaded** | 等待其执行完才触发 | 不等待其执行完 | 等待其执行完才触发 |
| **适用场景** | 核心必须尽早执行的逻辑 | 独立的第三方脚本、广告、统计 | 依赖 DOM 或其他脚本的业务逻辑 |

### ## 常见考点
1. **DOMContentLoaded 触发时机**：defer 脚本是在 DOMContentLoaded 之前还是之后执行？（答：之前）。
2. **内联脚本支持吗**：`async` 和 `defer` 对内联脚本（没有 `src` 的 `<script>`）是否生效？（答：不生效，会被忽略，当成普通脚本处理）。
3. **动态插入脚本**：使用 JS 动态创建 `script` 标签并插入页面，默认是 `async` 还是 `defer`？（答：默认行为类似 `async`，但可通过显式设置属性改变）。

# 什么是List接口？

**List 接口**是 Java 集合框架中继承自 Collection 的接口，它是一个有序集合，允许元素重复。

## 核心特点
- **有序性**：插入顺序与遍历顺序一致（部分实现类如 LinkedList 支持首尾操作）。 
- **索引访问**：支持通过索引（`get(int index)`）随机访问元素。
- **允许重复**：可以包含相同的元素。

## 常用实现类
- **ArrayList**：基于动态数组实现，查询快（O(1)），增删慢（需要移动元素），非线程安全。
- **LinkedList**：基于双向链表实现，增删快（O(1)），查询慢（O(n)），非线程安全。

## 特有迭代器：ListIterator
List 接口提供了一个特殊的迭代器 `ListIterator`，它继承自 `Iterator` 接口。

- **双向遍历**：支持向前遍历，增加了 `previous()` 和 `hasPrevious()` 方法。
- **修改元素**：允许在遍历过程中通过 `set()` 修改元素或通过 `add()` 添加元素。
- **获取索引**：可以通过 `nextIndex()` 和 `previousIndex()` 获取当前指针位置。

> 注意：ListIterator 仅适用于 List 接口的实现类。

### 实战案例
在处理分页查询结果时，前端往往需要知道当前数据在总列表中的位置。如果直接使用 `Iterator` 无法获取索引，而使用 `for(int i=0...)` 循环在 `ArrayList` 中效率尚可但在 `LinkedList` 中是 O(n) 灾难。最佳实践是使用 `ListIterator`，它无论底层实现如何，都能高效地遍历并知晓索引位置。

### 代码示例
```java
List<String> list = new ArrayList<>(Arrays.asList("A", "B", "C"));
// 获取 ListIterator，从索引 1 开始
ListIterator<String> it = list.listIterator(1);

while (it.hasNext()) {
    String s = it.next();
    if ("B".equals(s)) {
        it.set("B-Modified"); // 修改元素
        it.add("B-Plus");      // 在当前位置后面插入元素
    }
}
System.out.println(list); // 输出 [A, B-Modified, B-Plus, C]
```

### 核心对比
| 特性 | Collection | List | Set |
| :--- | :--- | :--- | :--- |
| **元素有序** | 不一定 | **是** (插入序) | 否 (HashSet) / 是 (LinkedHashSet) |
| **元素重复** | 不一定 | **允许** | 不允许 |
| **索引访问** | 不支持 | **支持** | 不支持 |
| **ListIterator** | 不支持 | **支持** | 不支持 |

## 记忆要点

- 核心特征：元素有序、允许重复、支持基于索引的随机访问。
- 特有迭代器：ListIterator支持双向遍历（hasPrevious），允许在遍历中修改或添加元素。
- 对比Set：List有序且允许重复，而Set通常无序且不允许重复。

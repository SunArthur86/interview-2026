# 什么是拥塞避免？

TCP 拥塞避免是拥塞控制机制中的一个阶段，旨在防止网络拥塞。在 TCP 连接建立初期，会先经历**慢启动**阶段，此时拥塞窗口 cwnd 呈指数级增长（每收到一个 ACK，cwnd 加 1）。当 cwnd 增长到超过**慢启动门限**时，进入拥塞避免阶段。

**拥塞避免算法规则**：
每当收到一个 ACK，cwnd 增加 1/cwnd。这意味着每经过一个往返时间 RTT，cwnd 大约增加 1 个 MSS（最大报文段长度）。这使拥塞窗口从指数增长转为**线性增长**，以减缓发送速度，探测网络带宽上限，避免造成拥塞。

### 原理与数据流示意图

在拥塞避免阶段，发送方小心翼翼地探测带宽：

```
发送方                            网络                          接收方
  │                                 │                             │
  ├─ 发送窗口 N (cwnd) ────────────>│                             │
  │                                 │─── ACK(确认 1 个包) ────────>│
  │<──────── ACK ───────────────────┤                             │
  │ (cwnd = cwnd + 1/N)              │                             │
  │ ... (RTT 后) ...                │                             │
  │ (cwnd = N + 1)                  │                             │
```

- **MSS (Maximum Segment Size)**：最大报文段长度，cwnd 的单位通常是 MSS。
- **增长公式**：`cwnd += 1 / cwnd`（每收到一个 ACK）。在一个 RTT 内收到 cwnd 个 ACK，总增加量为 `cwnd * (1/cwnd) = 1`。

### 状态流转对比

```
慢启动           拥塞避免              快速恢复
(指数增长)        (线性增长)             (加法增大/乘法减小)
    │                 │                       │
    │ cwnd >= ssthresh │  丢包(3个DupACK)      │
    └────────────────>│──────────────────────>│
                     │                       │
                     │  丢包(Timeout)        │
                     │─────────────┐         │
                     │             ▼         │
                     │         慢启动(重置)   │
```

### 实战案例
在高速内网（如 10Gbps 数据中心）进行大文件传输时，如果 TCP 拥塞窗口增长过慢（始终在拥塞避免阶段线性爬升），将无法填满管道带宽，导致带宽利用率不足 50%。BBR 算法正是为了解决传统 TCP（如 Reno/Cubic）在拥塞避免阶段过于保守的问题，通过测量带宽和 RTT 直接计算发送速率。

### 窗口增长机制对比

| 阶段 | 触发条件 | 增长方式 | 增长斜率 | 目的 |
| :--- | :--- | :--- | :--- | :--- |
| **慢启动** | 连接建立/超时重启 | 指数增长 | 极陡 (y=2^x) | 快速抢占带宽，探测网络瓶颈 |
| **拥塞避免** | cwnd >= ssthresh | 线性增长 | 平缓 (y=x) | 温和探测，避免引发丢包 |

### 代码示例 (Python伪代码)
```python
# 模拟拥塞避免阶段的 cwnd 更新
def congestion_avoidance(cwnd, mss):
    # cwnd 以包为单位，假设 cwnd 当前为 10
    # 收到 1 个 ACK
    increment = 1 / cwnd  # 增加 1/cwnd 个 MSS
    new_cwnd = cwnd + increment
    return new_cwnd

# 循环模拟 RTT 效果
for _ in range(10): # 模拟收到 10 个 ACK
    cwnd = congestion_avoidance(cwnd, 1)
# 一个 RTT 后，cwnd 总共增加了约 1 个 MSS
```

## 常见考点
1. **拥塞避免真的是“避免”拥塞吗？**：这个名字其实有点误导，它并不能完全“避免”拥塞，而是将拥塞窗口的增长速度从指数降为线性，从而**延迟**拥塞的发生，通过温和探测来寻找网络瓶颈。
2. **慢启动和拥塞避免的切换点**：当 cwnd >= ssthresh 时切换。如果在慢启动期间发生丢包，ssthresh 会更新为 cwnd 的一半，下一次进入拥塞避免时起点会更低。
3. **为什么不是收到 ACK 就加 1 MSS？**：如果在拥塞避免阶段仍然指数增长，一旦达到接近带宽上限，网络流量会瞬间溢出，导致剧烈丢包。线性增长能提供足够的反馈时间让发送方感知到拥塞信号（如 ACK 延迟增加或丢弃）。

- 核心目的并非避免拥塞，而是温和探测带宽，延缓拥塞发生

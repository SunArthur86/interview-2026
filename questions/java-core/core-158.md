# Java RMI的实现方式和工作原理是什么？

Java RMI（Remote Method Invocation）是一种用于实现远程过程调用的 Java API，允许一个 JVM 中的对象调用另一个 JVM 中的对象的方法。

### 1. 核心角色
- **Stub（客户端桩）**：位于客户端，负责将调用请求封装并发送给服务端，看起来像本地对象。
- **Skeleton（服务端骨架，JDK 1.2 后由反射替代）**：位于服务端，负责接收请求、解析参数并调用实际对象。
- **RMI Registry（注册表）**：服务端将远程对象注册到这里，客户端从中查找获取 Stub。

### 2. 工作原理（流程）
1. **服务端**：创建远程对象，继承 `UnicastRemoteObject`，并绑定到 RMI Registry（`Naming.rebind`）。
2. **客户端**：通过 RMI Registry 查找远程对象（`Naming.lookup`），获取 Stub 对象。
3. **调用**：客户端调用 Stub 的方法，Stub 将方法名、参数序列化为字节流。
4. **传输**：通过 TCP 网络将请求发送给服务端。
5. **执行**：服务端接收到请求，反序列化参数，调用实际对象的方法。
6. **返回**：服务端将结果序列化，通过网络返回给客户端，客户端反序列化得到结果。

### 4. 实战经验与对比
**实战案例**：在分布式监控系统中曾使用 RMI 暴露 MBean 服务。遇到过一个经典坑：服务端重启但客户端未重连，导致客户端调用报 `java.rmi.ConnectException`。解决方案是引入“自定义 Socket 工厂”并实现心跳检测，或者客户端捕获异常后重新从 Registry 查找最新的 Stub。

**代码示例**：
```java
// 服务端：创建并绑定远程对象
MyRemoteInterface impl = new MyRemoteImpl(); // 继承 UnicastRemoteObject
LocateRegistry.createRegistry(1099); // 启动注册表，默认端口 1099
Naming.rebind("MyService", impl); // 绑定名称

// 客户端：查找并调用
MyRemoteInterface stub = (MyRemoteInterface) Naming.lookup("rmi://localhost:1099/MyService");
String response = stub.sayHello("World");
```

**RMI vs RPC 对比**：
| 特性 | Java RMI | Hessian/Dubbo (通用 RPC) |
| :--- | :--- | :--- |
| **传输协议** | JRMP (Java Remote Method Protocol) | HTTP/TCP (二进制/Hessian2) |
| **语言支持** | 仅限 Java | 跨语言 (Java, C++, Python等) |
| **穿透性** | 较难穿透防火墙 | 易穿透 (基于 HTTP) |
| **性能** | 中等 (标准序列化效率低) | 高 (优化的二进制协议) |
| **适用场景** | Java 间纯远程调用，简单内部系统 | 复杂的微服务架构，异构系统通信 |

# 在浏览器中输入URL并按下回车之后会发生什么是什么？

该过程是一个经典的网络通信流程，涉及应用层、传输层、网络层、链路层等多个层面的协作。

### 1. URL 解析
浏览器首先检查输入的内容是否是合法的 URL，或者是否是搜索关键字。如果是 URL，则进行解析：
*   **协议**：如 `http`、`https`。决定通信的端口（默认 80/443）和加密方式。
*   **域名**：如 `www.google.com`，用于寻找服务器 IP。
*   **端口**：如果 URL 显式指定了端口（如 `:8080`），则使用指定端口，否则使用默认端口。
*   **路径**：请求的服务器文件路径，若省略则默认访问根目录（如 `/`），服务器通常配置为返回 `index.html` 或 `default.html`。

### 2. DNS 域名解析
浏览器需要将域名转换为 IP 地址才能进行通信。

#### 查询流程
```text
[浏览器缓存] -> [操作系统缓存(hosts)] -> [路由器缓存] -> [ISP DNS服务器] -> [根域名服务器] 
                                                                   |
                                                                   v
                                                            [顶级域名服务器] 
                                                                   |
                                                                   v
                                                            [权威域名服务器] -> 返回 IP
```
*   **递归查询**：主机向本地域名服务器发出的查询。
*   **迭代查询**：本地域名服务器向根域等发出的查询。

### 3. 建立 TCP 连接 (三次握手)
获得 IP 后，浏览器通过操作系统内核发起 TCP 连接（SYN -> SYN/ACK -> ACK）。

### 4. 发送 HTTP 请求
连接建立后，浏览器构建 HTTP 请求报文发送给服务器。
*   **请求行**：`GET /index.html HTTP/1.1`
*   **请求头**：包含浏览器信息、Cookie、缓存策略等。

### 5. 服务器处理请求
服务器接收请求，解析参数，处理业务逻辑（可能涉及查询数据库等），生成响应数据。

### 6. 返回 HTTP 响应
服务器返回 HTTP 响应报文。
*   **状态码**：如 200 (成功), 301 (重定向), 404 (未找到), 500 (服务器错误)。
*   **响应头**：包含内容类型、内容长度、缓存控制等。
*   **响应体**：实际的 HTML、JSON 或图片数据。

### 7. 浏览器渲染页面
浏览器解析 HTML，构建 DOM 树和 CSSOM 树，合并生成渲染树，最终绘制页面。

### 8. 断开 TCP 连接 (四次挥手)
数据传输完毕后，通过四次挥手断开连接（FIN -> ACK -> FIN -> ACK）。

## 常见考点
1.  **输入 URL 后，如果还没等到 DNS 结果，浏览器在做什么？**
    现代浏览器（如 Chrome）会进行预解析，在 UI 线程解析 URL 的同时，网络线程可能已经开始尝试建立连接或查找缓存。
2.  **为什么要进行三次握手，两次不行吗？**
    为了防止失效的连接请求报文段被服务端接收，从而产生错误。主要目的是确认双方的接收与发送能力是正常的。
3.  **HTTP 1.1 和 2.0 的主要区别？**
    HTTP 1.1 默认持久连接；HTTP 2.0 支持多路复用（一个连接并发处理多个请求）、头部压缩、服务器推送等。
4.  **什么是 HTTPS，它与 HTTP 的区别？**
    HTTPS 在 HTTP 下加入了 SSL/TLS 协议，通过加密传输数据、验证服务器身份、保证数据完整性。端口默认为 443。

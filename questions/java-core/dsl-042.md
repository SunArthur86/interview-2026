# meta标签

`<meta>` 标签位于 HTML 文档的 `<head>` 区域，用于描述页面的元数据，如字符集、作者、描述、关键词等。这些数据不会直接显示在页面上，但供浏览器或搜索引擎使用。

**1. 常用属性分类**
- **name 属性**：主要用于描述网页信息，对应 `content`。
  - `keywords`：网页关键词，便于 SEO。
  - `description`：网页描述，搜索引擎结果页显示的摘要。
  - `author`：作者信息。
  - `viewport`：针对移动端优化，控制视口大小、缩放比例。

- **http-equiv 属性**：模拟 HTTP 响应头，对应 `content`。
  - `content-type` (或 `charset`)：定义文档字符编码（推荐 `charset="UTF-8"`）。
  - `X-UA-Compatible`：指定 IE 浏览器的渲染模式（如 `IE=edge`）。
  - `refresh`：定时跳转或刷新（如 `30;url=http://example.com`）。
  - `expires`：设定网页过期时间，控制缓存。

**2. 移动端视口设置**
`<meta name="viewport" content="width=device-width, initial-scale=1.0">`
这是移动端开发的基石，参数含义如下：
- `width=device-width`：视口宽度等于设备宽度。
- `initial-scale=1.0`：初始缩放比例为 1:1。
- `user-scalable=no`：禁止用户缩放（可选）。

**3. 实战案例**
解决 iOS 上点击高亮背景色问题：如果在移动端 Web App 中不希望用户点击按钮时出现灰色半透明背景，通常会添加 `<meta name="format-detection" content="telephone=no">` 和 CSS 的 `-webkit-tap-highlight-color: transparent` 来优化原生体验。

**4. 代码示例 (HTML)**
```html
<head>
  <meta charset="UTF-8">
  <!-- 强制IE使用最新内核渲染 -->
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  
  <!-- 移动端标准适配，禁止缩放（适合H5应用） -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
  
  <!-- SEO设置 -->
  <meta name="description" content="这是一个关于前端技术的深度解析页面。">
</head>
```

## 常见考点
1. `http-equiv="X-UA-Compatible" content="IE=edge"` 的作用？（强制 IE 使用最新内核渲染）
2. viewport 标签中 `width=device-width` 的作用？（适配移动端屏幕宽度，禁止默认缩放）
3. SEO 中 keywords 和 description 的作用？（搜索引擎抓取关键词，展示搜索结果摘要）

# PriorityQueue是什么？

### PriorityQueue（优先队列）

**概述**：
PriorityQueue 是 Java 集合框架中基于**优先级堆**（Priority Heap）实现的队列。它不保证元素的内部整体有序，但保证每次取出的元素（`poll` 或 `peek`）是队列中优先级最高（或最低）的那个。

**核心原理**：
1.  **数据结构**：底层是一个**二叉小顶堆**（Min-Heap），实际存储使用 `Object[]` 数组（顺序存储）。
2.  **堆性质**：树中任一节点的值都小于等于其子节点的值。数组索引关系：`parent = (i-1)/2`, `left = 2*i+1`, `right = 2*i+2`。
3.  **动态调整**：
    -   **入队 (`offer`)**：将元素放入数组末尾，执行 `siftUp`（上浮）操作，调整堆结构。
    -   **出队 (`poll`)**：取出堆顶（数组[0]）元素，将数组末尾元素移至堆顶，执行 `siftDown`（下沉）操作，调整堆结构。

**关键细节**：
1.  **容量**：默认初始容量为 11。
2.  **扩容机制**：
    -   当容量小于 64 时，扩容为 `oldCapacity * 2 + 2`。
    -   当容量大于等于 64 时，扩容为 `oldCapacity * 1.5`。
3.  **比较器**：默认自然排序（小根堆），可通过构造函数传入 `Comparator` 实现大根堆或自定义排序。
4.  **非线程安全**：`PriorityQueue` 不是线程安全的，并发场景需使用 `PriorityBlockingQueue`。
5.  **不允许 null**：`PriorityQueue` 不允许插入 `null` 元素，否则会抛出 `NullPointerException`。
6.  **迭代器无序**：使用 `iterator()` 遍历时，不保证任何特定顺序，若需有序遍历需使用 `Arrays.sort(pq.toArray())`。

**架构示意图（二叉堆数组映射）**：
```text
      索引 0 (堆顶/最小值)
             /    \\
        索引 1    索引 2
        /    \\    /    \\
     索引 3 索引 4 索引 5 索引 6

数组存储: [0, 1, 2, 3, 4, 5, 6, ...]
父子关系计算:
- Parent(i) = (i - 1) / 2
- LeftChild(i)  = 2 * i + 1
- RightChild(i) = 2 * i + 2
```

## 常见考点
1.  **时间复杂度**：插入（`offer`）和删除（`poll`）的时间复杂度是多少？（均为 O(logN)，因为涉及堆的高度调整）。
2.  **Top K 问题**：如何利用 PriorityQueue 实现海量数据中找出最大的前 K 个数？（构建小顶堆，堆大小维持在 K）。
3.  **堆排序实现**：能否手写代码基于 PriorityQueue 进行堆排序？
4.  **与普通队列区别**：`LinkedList` 实现的队列是 FIFO，而 PriorityQueue 是优先级级，无 FIFO 保证。

### 5. 实战深化
#### 实战案例
在实现 **“每小时热门商品 Top 10”** 统计功能时，若对全量商品排序（O(N log N)）极慢。实战中通常用 PriorityQueue 维护一个大小为 10 的**小顶堆**：遍历商品数据，堆未满则入队，堆满则对比当前销量与堆顶，若比堆顶大则替换（`poll`后`offer`）。最终堆中留下的即为销量前 10 的商品，时间复杂度仅为 O(N log K)。

#### 代码示例
```java
// 实现大顶堆（取最大值）：传入倒序比较器
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
maxHeap.offer(10);
maxHeap.offer(30);
maxHeap.offer(20);

// 输出 30, 10, 20 (注意：iterator无序，poll有序)
System.out.println(maxHeap.poll()); // 30
System.out.println(maxHeap.poll()); // 20
```

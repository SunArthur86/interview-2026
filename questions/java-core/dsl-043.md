# HTML5特性

HTML5 引入了许多新特性，主要包括语义化标签、增强的表单、多媒体支持、Canvas 绘图、本地存储以及新的 API。

**1. 新增语义化标签**
如 `<header>`, `<nav>`, `<article>`, `<section>`, `<aside>`, `<footer>` 等，使页面结构更清晰。

**2. 多媒体标签**
- `<video>`：用于嵌入视频内容，支持 `controls`, `autoplay`, `loop`, `muted` 等属性，支持多种源格式（MP4, WebM）。
- `<audio>`：用于嵌入音频内容，属性与 video 类似。
（注：替代了老旧的 Flash 插件方案）

**3. 新增表单特性**
- **新输入类型**：`email`, `url`, `date`, `time`, `number`, `range`, `color` 等，自带基础验证。
- **新属性**：`placeholder`（占位符）、`required`（必填）、`autofocus`（自动聚焦）、`pattern`（正则验证）。

**4. 本地存储**
- **LocalStorage**：除非被手动清除，否则永久存储。
- **SessionStorage**：仅在当前会话（窗口）有效，关闭窗口后清除。
- **Cookies**：旧机制，有大小限制（4KB）且会随请求发送到服务器。

**5. Canvas 与 SVG**
- `<canvas>`：通过 JavaScript 脚本绘制 2D 图形（位图），适合游戏、图表。
- SVG：基于 XML 的矢量图形，可缩放不失真，适合图标、地图。

**6. 新的 API**
- Geolocation（地理定位）：获取用户位置信息。
- Web Worker：允许 JavaScript 运行在后台线程，避免阻塞 UI。
- WebSocket：提供全双工通信协议，实现服务器推送。

**7. 实战案例**
开发实时大屏监控项目时，利用 Web Worker 将海量的数据解析和计算逻辑移至后台线程，成功解决了主线程频繁阻塞导致的图表渲染卡顿问题，保证动画维持在 60fps。

**8. 代码示例 (JavaScript - LocalStorage)**
```javascript
// 处理 localStorage 超出配额（通常是5MB）的常见封装
function safeSetItem(key, value) {
  try {
    localStorage.setItem(key, value);
  } catch (e) {
    if (e.name === 'QuotaExceededError') {
      console.error('Storage full, cleaning old data...');
      // 清理旧数据的逻辑，例如只保留最新的10条
      localStorage.clear(); 
      localStorage.setItem(key, value);
    }
  }
}
```

**9. 对比表格：Canvas vs SVG**

| 特性 | Canvas | SVG |
| :--- | :--- | :--- |
| **图形类型** | 位图 (基于像素) | 矢量图 (基于 XML/DOM) |
| **事件支持** | 不支持单个图形事件 (需计算坐标) | 支持标准 DOM 事件 (click, hover) |
| **性能** | 适合对象数量多、频繁重绘 (游戏) | 适合对象数量少、高保真缩放 (图标) |
| **可操作性** | 脚本绘制，导出为图片 | DOM 节点，可用 CSS 样式化 |
| **动画实现** | requestAnimationFrame + 重绘 | CSS 动画或 SMIL |

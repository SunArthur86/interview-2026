# Service Worker

### Service Worker

#### 1. 概念
Service Worker 是一个运行在浏览器背后的**独立线程**。它本质上是一个**代理服务器**，位于浏览器和网络之间。它可以拦截网络请求，根据网络状态采取适当的动作（如返回缓存），旨在创建有效的离线体验。

#### 2. 核心特点
*   **后台线程**：独立于主线程，不会阻塞页面执行。
*   **网络代理**：可以拦截 `fetch` 请求，自定义返回内容（缓存、模拟数据）。
*   **持久化**：一旦注册，即使页面关闭，Worker 仍在后台运行，直到被手动卸载。
*   **HTTPS 要求**：出于安全考虑，Service Worker 必须在 HTTPS 环境下才能运行（localhost 除外）。

#### 3. 生命周期
1.  **注册**：主线程调用 `navigator.serviceWorker.register()`。
2.  **安装**：触发 `install` 事件，通常在此阶段缓存静态资源。需调用 `event.waitUntil()` 确保安装完成。
3.  **激活**：触发 `activate` 事件，清理旧缓存。需调用 `event.waitUntil()` 确保旧缓存清理完毕。新 SW 只有在所有页面都关闭旧 SW 后才会接管控制（或者使用 `clients.claim()` 立即接管）。
4.  **处理请求**：Worker 处于 `activated` 状态，可以拦截网络请求 (`fetch` 事件)。

#### 4. 生命周期状态流转

```text
    (parsed)   ->    (installing)    ->    (installed)   ->   (activating)  ->   (activated)
       │              │                    │                   │                  │
       │    解析脚本  │   install 成功     │                   │                  │
       │              └──────────────────>│                   │                  │
       │                                   │  激活条件满足    │                  │
       │                                   │ (无旧SW/页面关闭) │                  │
       │                                   │                   │                  │
       │                                   │   activate 成功  │                  │
       │                                   │                   │                  │
       │                                   │                   │                  │
       │<───────────────────────────────────<───────────────────<──────────────────<
       │               (redundant - 废弃/被替换)
```

#### 5. 使用场景
*   **离线缓存**：PWA 核心技术，缓存关键资源实现离线访问。
*   **消息推送**：接收服务器推送的消息。
*   **后台同步**：在网络恢复时推迟同步数据。
*   **请求拦截与模拟**：根据请求内容返回不同数据。

#### 6. 常用缓存策略
*   **Cache First**：缓存优先，适用于静态资源（CSS/JS/图片）。
*   **Network First**：网络优先，失败回退缓存，适用于动态数据（API 请求）。
*   **Stale While Revalidate**：快速返回缓存，同时在后台更新缓存，适用于内容更新不频繁但需要及时性的场景。

## 常见考点
1.  **Service Worker 的生命周期是怎样的？为什么需要 `waitUntil`？**（考察点：异步操作的完成保证，防止 SW 状态切换过早导致安装/激活失败）
2.  **Service Worker 与 Web Worker 的区别？**（考察点：SW 是代理服务器，生命周期更长，能拦截请求；WW 是后台计算，随页面销毁）
3.  **如何更新一个 Service Worker？**（考察点：文件变动触发 install，等待所有客户端关闭后 activate，或者使用 `skipWaiting()`）
4.  **PWA (Progressive Web App) 的核心要素有哪些？**（考察点：Service Worker, Manifest, HTTPS）。


## 记忆要点

- 本质是浏览器背后的网络代理服务器，核心用于拦截请求与离线缓存
- 因为涉及请求拦截，所以强制要求必须在HTTPS环境下运行
- 生命周期：注册->install->waitUntil->activate->activated控制
- 缓存策略口诀：静态资源Cache First，动态API Network First，兼顾速度用SWR

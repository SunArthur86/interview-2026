# 什么是IP地址？

IP 地址（Internet Protocol Address）是互联网上设备的逻辑地址，用于标识网络中的主机并进行路由寻址。

**1. IP 地址分类**
IPv4 地址为 32 位，通常分为 A、B、C、D、E 五类：
*   **A 类**：`0.0.0.0` 到 `127.255.255.255`。第一段为网络号，后三段为主机号。适用于大型网络。
*   **B 类**：`128.0.0.0` 到 `191.255.255.255`。前两段为网络号，后两段为主机号。适用于中型网络。
*   **C 类**：`192.0.0.0` 到 `223.255.255.255`。前三段为网络号，最后一段为主机号。适用于小型网络。
*   **D 类**：`224.0.0.0` 到 `239.255.255.255`。用于**多播**（组播）。
*   **E 类**：`240.0.0.0` 到 `255.255.255.255`。保留用于研究使用。

**2. CIDR（无类别域间路由）**
摒弃了传统的 A/B/C 类划分，使用**子网掩码**或斜杠记法（如 `192.168.1.0/24`）来灵活划分网络位和主机位，提高了 IP 地址的利用率。

**3. 公有 IP 与私有 IP**
*   **公有 IP**：全球唯一，需向 ICANN 申请，用于互联网通信。
*   **私有 IP**：仅在局域网内使用，可重复。范围包括：
    *   A 类：`10.0.0.0` - `10.255.255.255`
    *   B 类：`172.16.0.0` - `172.31.255.255`
    *   C 类：`192.168.0.0` - `192.168.255.255`
私有 IP 访问互联网需通过 NAT（网络地址转换）。

**4. 特殊 IP 地址**
*   `127.0.0.1`：**环回地址**，用于本地主机自我通信。
*   `0.0.0.0`：通常表示默认路由或所有本地 IP。
*   `255.255.255.255`：**受限广播地址**，仅在本网络内广播。

### 实战案例
在 Docker 容器网络配置中，曾出现因宿主机和容器默认使用 `172.17.0.0/16` 网段导致 IP 冲突，容器无法访问外网。解决方案是将 Docker 网桥配置为 B 类私有地址的另一个子网（如 `172.20.0.0/16`）或使用 C 类地址段。

### 对比表格：IPv4 vs IPv6

| 特性 | IPv4 | IPv6 |
| :--- | :--- | :--- |
| **地址长度** | 32 位 (约 43 亿个) | 128 位 (近乎无限) |
| **表示法** | 点分十进制 (192.168.1.1) | 冒号十六进制 (2001:0db8::1) |
| **头部大小** | 固定 20-60 字节 | 固定 40 字节 (简化的头部) |
| **安全性** | 无内置安全 (依赖 IPSec) | 内置 IPSec 支持 |
| **配置** | 通常需要 DHCP 或手动配置 | 支持无状态自动配置 (SLAAC) |
| **广播** | 支持广播 (ARP, Broadcast) | 废除广播，使用组播 任播 |

**## 常见考点**
1.  **子网掩码的作用**：用于区分 IP 地址中的网络部分和主机部分。将 IP 地址与子网掩码进行“与”运算，即可得到网络地址。
2.  **IPv6 的优势**：地址长度 128 位，解决了 IPv4 地址枯竭问题；简化了报头，提升路由效率；内置 IPsec 支持，安全性更高；支持即插即用。
3.  **127.0.0.1 和 localhost 的区别**：`127.0.0.1` 是 IPv4 的环回地址；`localhost` 是域名，通常指向 `127.0.0.1`，但在某些系统配置中也可能指向 IPv6 的环回地址 `::1`。
4.  **为什么私有 IP 可以重复**：因为私有 IP 地址不在公网上路由，仅在局域网内部有效，不同局域网使用相同的私有 IP 不会冲突。

## 记忆要点

- 核心定义：网络设备的逻辑地址，用于寻址和路由，由网络位与主机位构成。
- 私网IP段口诀：A(10.)、B(172.16.-31.)、C(192.168.)，公网唯一但私网可复用（靠NAT）。
- 特殊IP对比：127.0.0.1是本机环回，0.0.0.0代表本机所有网卡，255.255.255.255为全网广播。
- 进化对比：IPv4为32位且依赖ARP广播，IPv6为128位且废除广播改用组播。

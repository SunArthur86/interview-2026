# 作业提交与初始化

作业提交与初始化是 MapReduce 作业生命周期的第一阶段，主要涉及客户端准备资源、提交请求，以及 JobTracker（或 ResourceManager）对作业的初始化准备。

### 实战案例
在一次提交 Spark on YARN 任务时，作业一直处于 ACCEPTED 状态。经排查，是因为客户端上传的依赖 jar 包过大（超过 2GB），导致资源上传至 HDFS 耗时过长，超过了 RM 的等待超时时间。优化方案是减小依赖包或使用 `--archives` 共享库。

### 流程详解

#### 1. 资源上传
*   **角色**：JobClient
*   **动作**：用户提交作业后，JobClient 将作业所需的资源上传到 HDFS 的特定目录（通常是以 JobID 命名的目录）。
*   **上传内容包括**：
    *   程序 JAR 包。
    *   作业配置文件（`job.xml`，包含所有配置参数）。
    *   输入分片元信息文件（计算得到的 InputSplits）。
    *   Distributed Cache 相关文件（如有）。

#### 2. 提交请求
*   **动作**：资源上传完毕后，JobClient 通过 RPC 调用 `submitJob` 方法通知 JobTracker（或 YARN 中的 ResourceManager）。

#### 3. 作业初始化
*   **角色**：JobTracker (MRv1) 或 MRAppMaster (MRv2/YARN)
*   **动作**：收到请求后，将作业加入内部队列，由调度器安排初始化。
    *   **创建对象**：创建一个 `JobInProgress` 对象（MRv2 中为 `Job` 对象）来封装作业信息，跟踪整个作业的运行状态。
    *   **任务分解**：`JobInProgress` 根据输入分片的数量，为每个分片创建对应的 `TaskInProgress` 对象。
        *   Map 任务数 = InputSplit 数。
        *   Reduce 任务数 = 用户配置的 `mapreduce.job.reduces`。
    *   **任务尝试**：每个 `TaskInProgress` 可能需要管理多个 `TaskAttemptID`，用于处理任务失败后的重试机制。

### 关键代码 (Java - 作业提交配置)
```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "Word Count");

// 实战：设置 Reduce 任务数，默认为1，生产环境通常根据数据量设置
job.setNumReduceTasks(5); 

// 提交作业并等待完成
System.exit(job.waitForCompletion(true) ? 0 : 1);
```

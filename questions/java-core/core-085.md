# 什么是三次握手？

### 三次握手详细过程

1. **第一次握手**：
   - 客户端发送 `SYN=1` 的报文，并随机生成一个序列号 `seq=x`。
   - 客户端进入 `SYN_SENT` 状态。
   - **作用**：告诉服务端，客户端请求建立连接。

2. **第二次握手**：
   - 服务端收到 SYN 报文，需要确认客户端的序列号，即 `ack=x+1`。
   - 同时自己也发送一个 SYN 请求，即 `SYN=1`，并随机生成自己的序列号 `seq=y`。
   - 服务端进入 `SYN_RCVD` 状态。
   - **作用**：服务端告诉客户端，“我收到你的请求了（ACK），我也想和你建立连接（SYN）”。

3. **第三次握手**：
   - 客户端收到服务端的 SYN+ACK 报文。
   - 检查 `ack` 是否为 `x+1`，以及 `ACK` 标志位是否为 1。若正确，则发送确认包 `ACK=1`，`ack=y+1`，`seq=x+1`。
   - 客户端进入 `ESTABLISHED` 状态。
   - 服务端收到该包后也进入 `ESTABLISHED` 状态。
   - **作用**：客户端告诉服务端，“我收到你的确认了，连接建立成功”。

#### TCP 报文选项补充
- **MSS (Maximum Segment Size)**：在 SYN 报文中协商最大报文段长度，避免分片。
- **Window Scale**：协商窗口扩大因子，突破 TCP 头部 16 位窗口大小限制。
- **SACK (Selective Acknowledgment)**：选择性确认，允许只重传丢失的数据包，提高效率。

# 什么是工厂模式？

### 工厂模式

**概述**
工厂模式是一种非常常用的创建型设计模式，其提供了创建对象的最佳方式。在创建对象时，不会对客户端暴露对象的创建逻辑，而是通过使用共同的接口来创建对象。其用来封装和管理类的创建，本质是对获取对象过程的抽象。

一般情况下，工厂模式分为三种更加细分的类型：简单工厂、工厂方法和抽象工厂。

**优点**
1. **解耦**：将对象的创建和使用进行分离，客户端代码与具体产品类的实例化过程解耦。
2. **可复用**：对于创建过程比较复杂且在很多地方都使用到的对象，通过工厂模式可以提高代码复用性。
3. **易于扩展**：添加新的产品类时，只需要扩展相应的具体产品和具体工厂，符合开闭原则。
4. **符合面向对象设计原则**：将对象的创建封装在工厂类中，使得系统更符合单一职责原则。

**1. 简单工厂模式**

**概念**
简单工厂模式也被称为静态工厂方法模式。它并不是一个标准的设计模式，更像是一种编程习惯。在简单工厂模式中，一个工厂类负责创建多个产品类的实例，通过传入不同的参数来决定创建哪种产品。

**缺点**
- 工厂类集合了所有产品的创建逻辑，职责过重。
- 新增一个产品就需要在原工厂类内部添加一个分支，违反了开闭原则。

**2. 工厂方法模式**

**概念**
工厂方法模式中，将简单工厂中的工厂类变为一个抽象接口。负责给出不同工厂应该实现的方法，自身不再负责创建各种产品，而是将具体的创建操作交给实现该接口的子工厂类来做。

**优点**
- 通过多态的形式解决了简单工厂模式过多的分支问题。
- 更符合开闭原则。

**架构图解**：
```text
         ┌─────────────┐
         │   Product   │ (抽象产品接口)
         └──────┬──────┘
                │ implements
    ┌───────────┼───────────┐
    ▼           ▼           ▼
┌────────┐ ┌────────┐ ┌────────┐
│ProductA│ │ProductB│ │ProductC│ (具体产品)
└────────┘ └────────┘ └────────┘
    ▲           ▲           ▲
    │ creates   │ creates   │ creates
┌───┴─────┐ ┌───┴─────┐ ┌───┴─────┐
│FactoryA │ │FactoryB │ │FactoryC │ (具体工厂实现 Factory 接口)
└─────────┘ └─────────┘ └─────────┘
```

**3. 抽象工厂模式**

**概念**
抽象工厂模式提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们的具体类。抽象工厂模式通常涉及多个抽象产品、多个具体产品和多个具体工厂。例如：不同风格的 UI 组件（Windows 风格的 Button + Textbox，Mac 风格的 Button + Textbox）。

**优点**
- 将一些相关的产品组成一个“产品族”，由同一个工厂来统一生产。
- 对于新增一个产品族很方便，符合开闭原则。

**缺点**
- 新增一个产品等级结构，会对整个工厂结构进行大改。

**实战案例**：
在对接多种支付渠道（支付宝、微信、PayPal）时，如果代码中充斥着 `if (type == "alipay") { new Alipay() }`，维护将非常困难。实战中通常定义 `PaymentFactory` 接口和 `Payment` 接口，通过配置文件或策略模式动态获取对应的工厂实例，从而实现新增渠道只需加类不改旧代码。

**## 常见考点**
1. 简单工厂、工厂方法、抽象工厂的区别？
   - 简单工厂：一个工厂类对应多种产品（参数决定）；工厂方法：一个具体工厂对应一个具体产品（一对一）；抽象工厂：一个具体工厂对应一个**产品族**（多个产品接口）。
2. Spring 框架中用到了工厂模式吗？
   - 用到了。`BeanFactory` 和 `ApplicationContext` 是典型的工厂模式，负责创建和管理 Bean 对象（IOC 容器的本质就是超级大工厂）。
3. 什么时候不推荐使用工厂模式？
   - 当产品类非常少，且创建逻辑简单时，使用工厂模式会增加类的数量，导致系统结构过于复杂，反而降低可读性（适得其反）。

**代码示例**：
```java
// 工厂方法模式示例
interface LoggerFactory {
    Logger createLogger();
}

class FileLoggerFactory implements LoggerFactory {
    public Logger createLogger() {
        return new FileLogger();
    }
}

// 使用时
LoggerFactory factory = new FileLoggerFactory();
Logger logger = factory.createLogger();
```

**对比表格**：

| 模式类型 | 工厂类数量 | 产品类数量 | 扩展性 (新增产品) | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **简单工厂** | 1 个 | 多个 | 需修改工厂代码 (差) | 产品少，创建逻辑简单 |
| **工厂方法** | 多个 (一一对应) | 多个 | 新增工厂类 (好) | 产品种类多，需灵活扩展 |
| **抽象工厂** | 多个 (对应产品族) | 多个 (多个等级) | 新增工厂类，但新增等级难 | 产品有家族概念（如跨平台UI） |

# 程序执行流程是什么？

程序执行流程包括：源代码经编译系统转为可执行文件，存储于磁盘。在Shell中输入命令后，Shell加载该文件，通过DMA技术将代码和数据从磁盘复制到内存。处理器从内存读取指令并执行，如将数据从内存复制到寄存器，再输出到显示设备，最终完成程序运行。

**## 补充细节**
1.  **编译与链接**：源代码首先由编译器编译成目标文件，再经链接器将目标文件与库文件链接，生成可执行文件（如 Windows 下的 .exe 或 Linux 下的 ELF 文件）。
2.  **加载**：Shell 调用 `fork` 创建子进程，再调用 `execve` 系统调用。操作系统加载器将可执行文件从磁盘加载到内存的用户空间。如果是大文件，可能会用到按需加载。
3.  **DMA 作用**：直接内存访问（DMA）允许外设（磁盘）直接将数据写入内存，而无需 CPU 连续干预，提高了 CPU 利用率。
4.  **执行阶段**：CPU 的程序计数器（PC）指向内存中的代码段入口。CPU 依次执行“取指-译码-执行”循环。数据段的数据被加载到通用寄存器中进行运算。

**## 常见考点**
1.  程序从磁盘到内存经历了哪些步骤？（编译、链接、加载）
2.  为什么 CPU 不能直接从磁盘读取指令？（速度差异过大，且寻址方式不同）
3.  Shell 在程序执行中扮演什么角色？（命令解释器，负责创建进程和加载程序）

**## 实战案例**
在高并发服务器启动时，如果未开启 `mmap` 且依赖 `read` 系统调用，大文件的加载会导致 CPU 在用户态和内核态之间频繁切换，不仅消耗 CPU 资源，还会导致启动延迟。此时利用操作系统的**按需加载**（Demand Paging）和内存映射文件技术，可以显著降低进程的内存占用峰值并加快启动速度。

**## 代码示例 (C)**
```c
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork(); // 创建子进程
    if (pid == 0) {
        // 子进程调用 execve 加载程序
        // 内核加载器介入，通过DMA将磁盘代码读入内存
        char *argv[] = {"/bin/ls", NULL};
        execve("/bin/ls", argv, NULL);
    } else {
        wait(NULL); // 父进程等待
    }
    return 0;
}
```

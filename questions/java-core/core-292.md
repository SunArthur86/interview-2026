# 归并排序的原理和时间复杂度是什么？

### 归并排序

#### 基本原理
归并排序采用**分治法**（Divide and Conquer）策略：
1.  **分**：将序列从中间切分，递归地把长度为 n 的序列分成两个长度为 n/2 的子序列。
2.  **治**：对每个子序列分别进行归并排序。
3.  **合**：将两个已经有序的子序列合并成一个最终的有序序列。

#### 代码实现

```java
public class MergeSort {
    // 优化：临时数组作为成员变量，避免递归中频繁创建销毁
    private static int[] tmpArr;

    public static void sort(int[] data) {
        if (data == null || data.length < 2) return;
        tmpArr = new int[data.length];
        sort(data, 0, data.length - 1);
    }

    private static void sort(int[] data, int left, int right) {
        if (left >= right) return;
        int center = (left + right) / 2; // 防止溢出也可写 left + (right - left)/2
        sort(data, left, center);      // 递归排序左边
        sort(data, center + 1, right); // 递归排序右边
        
        // 优化：如果 data[center] <= data[center+1]，说明左右已有序，无需 merge
        if (data[center] > data[center + 1]) {
            merge(data, left, center, right);
        }
    }

    private static void merge(int[] data, int left, int center, int right) {
        int mid = center + 1; // 右数组起始索引
        int third = left;     // 临时数组索引
        int tmp = left;       // 缓存左数组起始索引，用于回拷

        while (left <= center && mid <= right) {
            // 取较小的值存入临时数组
            if (data[left] <= data[mid]) {
                tmpArr[third++] = data[left++];
            } else {
                tmpArr[third++] = data[mid++];
            }
        }

        // 剩余部分直接拷贝
        while (mid <= right) {
            tmpArr[third++] = data[mid++];
        }
        while (left <= center) {
            tmpArr[third++] = data[left++];
        }

        // 将临时数组拷回原数组
        while (tmp <= right) {
            data[tmp] = tmpArr[tmp++];
        }
    }
}
```

## 实战案例
在**大数据外部排序**中，当数据量（如 100GB）超过内存限制（如 4GB）时，无法一次性加载。工程上采用归并排序思想：先将数据分块在内存中排序并写入临时文件（归并排序的“分”），然后使用多路归并策略，维护一个最小堆，从多个有序临时文件中读取数据进行合并（归并排序的“合”），最终生成全量有序文件。

## 复杂度分析

| 指标 | 值 | 说明 |
|------|-----|------|
| 时间复杂度 | O(n log n) | 无论最好、最坏、平均均为 O(n log n) |
| 空间复杂度 | O(n) | 需要额外的临时数组存储合并结果 |
| 稳定性 | ✅ 稳定 | 合并时 `<=` 保证了相等元素的相对顺序不变 |

## 常见考点
*   **考点 1**：归并排序的空间复杂度是多少？如何优化？（标准 O(n)，可结合插入排序在小区间内优化，但空间仍需 O(n)）
*   **考点 2**：为什么归并排序比快速排序更稳定？（考察快排 O(n²) 的最坏情况和归并排序恒定 O(n log n) 的特性）
*   **考点 3**：如何在单链表上实现归并排序？（考察链表归并无需额外空间，只需修改指针的特性）

## 记忆要点

- 核心三步曲：选基准（Pivot）、分区排列（小的放左大的放右）、递归排序左右子序列
- 最坏时间复杂度为O(n²)，发生在数组正序或逆序，退化为冒泡
- 空间复杂度O(log n)，因存在跨距离交换，所以是不稳定排序
- 常考优化点：三数取中法选基准、小数组切换插入排序、大量重复元素用三路切分

# DOM和事件

### DOM 事件流

当事件发生时，事件会在元素节点之间按照特定的顺序传播，这个过程称为事件流。DOM2级事件规范规定事件流包含三个阶段：

```text
                   │  Window (Document)             
                   │     ┌─────────────────┐
                   │     │   HTML Element  │
                   │     └────────┬────────┘
                   │              │
   捕获阶段(Capture)│    ┌─────────▼─────────┐
(自顶向下)          │    │    Body Element   │
                   │    └─────────┬─────────┘
                   │              │
                   │    ┌─────────▼─────────┐
                   │    │   Div (Parent)    │
                   │    └─────────┬─────────┘
                   │              │
                   │    ┌─────────▼─────────┐  
                   └───▶│  Button (Target)  │◀─────── 冒泡阶段
                        └───────────────────┘     (自底向上)
```

1.  **捕获阶段**：事件从 `window` 对象开始，向下传递到目标节点。
2.  **目标阶段**：事件到达目标节点，执行目标节点上的事件处理程序。
3.  **冒泡阶段**：事件从目标节点向上回传，逐级触发父级元素的同名事件，直到 `window`。

*   **注意**：并非所有事件都冒泡，如 `focus`, `blur`, `mouseenter`, `mouseleave` 不冒泡。

#### 实战案例
在做“无限下拉加载”的列表组件时，如果直接给每个列表项绑定点击事件，页面滚动几屏后会导致严重的内存泄漏和卡顿。通过在父容器上委托事件，不仅解决了性能问题，还能自动处理动态加载的新元素点击响应，无需重新绑定。

### 事件绑定模型

| 特性 | DOM0 级 | DOM2 级 (`addEventListener`) | IE 模型 (`attachEvent`) |
| :--- | :--- | :--- | :--- |
| **绑定方式** | `el.onclick = fn` | `el.addEventListener('click', fn, false)` | `el.attachEvent('onclick', fn)` |
| **事件名** | 带 `on` | 不带 `on` | 带 `on` |
| **执行顺序** | 后覆盖前 | 按注册顺序执行 | 倒序执行（先注册后触发） |
| **移除监听** | `el.onclick = null` | `removeEventListener` | `detachEvent` |
| **this 指向** | 当前元素 | 当前元素 | `window` |
| **支持阶段** | 仅冒泡 | 捕获/冒泡（可选） | 仅冒泡 |

#### 1. DOM0 级（原始模型）
通过元素属性绑定（如 `btn.onclick = fn`）。
*   **特点**：代码简单，绑定速度快。
*   **缺点**：只支持冒泡；同一个事件只能绑定一个处理函数（后者覆盖前者）。
*   **解绑**：`btn.onclick = null`。

#### 2. DOM2 级（标准模型）
使用 `addEventListener(eventType, handler, useCapture)`。
*   **参数**：
    *   `eventType`：事件名（不带 on，如 'click'）。
    *   `handler`：回调函数。
    *   `useCapture`：布尔值。`true` 表示在捕获阶段触发，`false`（默认）表示在冒泡阶段触发。
*   **特点**：支持同一元素绑定多个监听器，按注册顺序执行。
*   **解绑**：`removeEventListener`，传入的 `handler` 必须是同一个函数引用。

#### 3. IE 模型（已淘汰）
使用 `attachEvent` 和 `detachEvent`（仅 IE8-）。
*   **特点**：支持多个绑定，但事件名带 `on`；`this` 指向 `window`；只支持冒泡。

### 事件委托

**原理**：利用事件冒泡机制，将事件处理程序绑定在父元素上，而不是子元素上。

**优点**：
1.  **性能优化**：减少 DOM 操作和内存占用（无需为每个子元素绑定事件）。
2.  **动态绑定**：新添加的子元素无需手动绑定也能触发事件。

#### 实战案例
在一个复杂的数据表格中，单元格内包含“编辑”、“删除”等多个操作按钮。如果给每个按钮单独绑定事件，渲染几千行数据时极慢。使用事件委托，只需在 `<table>` 上绑定一个监听器，通过判断 `e.target.closest('.btn-delete')` 即可精准响应用户操作，且对虚拟 DOM 的 Diff 操作友好。

**实现示例**：
```javascript
// 更加健壮的事件委托实现，兼容深层嵌套
ul.addEventListener('click', function(e) {
  // 兼容性获取事件源
  let target = e.target || e.srcElement;
  
  // 关键优化：使用 closest 匹配自身或父级，防止点到 li 内部的 span 或图标失效
  let li = target.closest('li');
  
  // 确保点击的是 ul 内部的 li
  if (li && this.contains(li)) {
    console.log(li.innerText);
  }
});
```

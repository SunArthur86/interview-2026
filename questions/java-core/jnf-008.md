# 什么是Sequenced Collections（顺序集合）？Java 21引入了哪些新接口和方法？

Sequenced Collections是Java 21（JEP 431）引入的一组新接口，为Java现有集合层次结构补充了一致的顺序访问API。在Java 21之前，Java集合接口在访问有序集合的首尾元素时有不一致且令人沮丧的API：List使用get(0)和get(list.size()-1)，Deque使用getFirst()和getLast()，LinkedHashSet根本没有直接的首尾访问方法。

Java 21引入了三个新接口来统一这些操作：SequencedCollection, SequencedSet, SequencedMap。

```java
// SequencedCollection - 新接口
public interface SequencedCollection<E> extends Collection<E> {
    E getFirst();          // 获取第一个元素
    E getLast();           // 获取最后一个元素
    void addFirst(E e);    // 添加到头部
    void addLast(E e);     // 
}
```

这些接口被追溯地添加到现有集合类上——List、Deque、LinkedHashSet、LinkedHashMap、SortedSet、SortedMap及其实现类——使得新方法在所有现有Java代码中立即可用。

#### 实战案例
在实现“最近浏览记录”功能时，通常使用LinkedHashMap保持插入顺序并淘汰最旧数据。以前必须通过复杂的迭代器或获取entrySet来判断“首尾”，现在可以直接使用`map.removeFirst()`和`map.putLast()`，代码逻辑从“实现细节导向”转变为“业务语义导向”。

#### 实战代码：LRU Cache 简化实现

```java
// 实战代码：利用 SequencedMap 快速实现 LRU 淘汰逻辑
public class SimpleLRU<K, V> extends LinkedHashMap<K, V> {
    private final int maxSize;

    public SimpleLRU(int maxSize) {
        this.maxSize = maxSize;
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        // 使用 size() 直观判断，无需额外维护计数器
        return size() > maxSize; 
    }
    
    // 实战：获取最旧数据（尾部）
    public V getOldest() {
        return this.lastEntry().getValue();
    }
}
```

#### 对比表格：Sequenced API vs 传统 API

| 操作 | 传统 API | Sequenced API (Java 21) |
| :--- | :--- | :--- |
| **获取第一个** | `list.get(0)` | `list.getFirst()` (空指针友好) |
| **获取最后一个** | `list.get(list.size()-1)` | `list.getLast()` |
| **添加到头部** | `list.add(0, elem)` | `list.addFirst(elem)` |
| **逆序遍历** | `for (int i=size-1; i>=0; i--)` | `for (var e : list.reversed())` |
| **适用集合** | List, Deque 等 (不统一) | List, Set, Map (统一接口) |

## 记忆要点

- 核心痛点：Java 21前获取末尾元素API割裂，如List需算size减一，Map更是无直接方法。
- 三大统一接口：新增SequencedCollection, SequencedSet, SequencedMap。
- 核心方法：提供统一的getFirst()、addLast()及逆序遍历reversed()方法。
- 实战应用：为LinkedHashMap直接补充pollLastEntry()，让LRU淘汰逻辑从繁杂迭代变为一行代码。

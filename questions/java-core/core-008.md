# 什么是Java概述？

Java 是一门面向对象、跨平台的编程语言，核心特性是「Write Once, Run Anywhere（一次编写，到处运行）」。

## 核心组件详解

1.  **JVM (Java Virtual Machine)**：
    -   **作用**：加载 `.class` 字节码文件，解释/编译为机器码执行，管理内存（堆、栈、方法区等）。
    -   **跨平台核心**：不同操作系统有不同实现版的 JVM（如 HotSpot, J9），屏蔽了底层 OS 差异。
2.  **JRE (Java Runtime Environment)**：
    -   **组成**：JVM + Java 核心类库（如 String, System 等 API）。
    -   **用途**：仅用于**运行** Java 程序，不包含编译器 (`javac`)。
3.  **JDK (Java Development Kit)**：
    -   **组成**：JRE + 开发工具（javac, jshell 等）。

**实战案例**：
排查生产环境 OOM（内存溢出）时，必须明确区分是 JDK 版本问题还是 JVM 参数配置问题。例如，从 Java 8 升级到 Java 11 时，若未调整 JVM 内存参数，可能会因为元空间取代永久代且默认大小较小而导致 `java.lang.OutOfMemoryError: Metaspace`。另外，Linux 服务器上安装 JRE 而非 JDK 会导致监控工具（如 jstat, arthas）无法启动，影响故障排查效率。

**代码示例（通过代码获取 JDK/JRE 版本信息）**：
```java
public class JavaEnvInfo {
    public static void main(String[] args) {
        // 获取运行时版本（JRE信息）
        System.out.println("Java Version: " + System.getProperty("java.version"));
        System.out.println("Java Home: " + System.getProperty("java.home"));
        
        // 获取编译器版本（JDK独有）
        javax.tools.JavaCompiler compiler = javax.tools.ToolProvider.getSystemJavaCompiler();
        if (compiler != null) {
            System.out.println("JDK Compiler available: " + compiler);
        } else {
            System.out.println("Running on JRE only, no compiler detected.");
        }
    }
}
```

## 记忆要点

- Java核心特性是面向对象与「一次编写，到处运行」的跨平台能力。
- 包含关系：JDK包含JRE和开发工具，而JRE包含JVM和核心类库。
- 跨平台核心是JVM：不同系统装不同版JVM，负责将字节码解释为机器码。
- 实战避坑：仅装JRE无法使用jstat等监控工具排查生产故障。

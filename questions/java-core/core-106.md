# 什么是数字证书？

### 数字证书

数字证书是互联网通信中用于验证实体身份（如服务器、个人）的电子文档，由权威的证书颁发机构（CA）签发。它解决了“中间人攻击”的隐患，确保公钥确实属于声称的持有者。

### 核心流程与原理
数字证书基于公钥基础设施（PKI）体系，包含以下关键步骤：

1.  **密钥生成**：
    实体（如服务器）生成一对密钥：公钥（公开，用于加密/验签）和私钥（保密，用于解密/签名）。

2.  **证书签名请求（CSR）**：
    实体生成 CSR 文件，包含公钥、实体信息（域名、组织、CN等）以及实体的数字签名（证明自己拥有私钥）。

3.  **证书颁发**：
    CA 收到 CSR 后，验证实体身份（DNS 验证、组织验证等）。验证通过后，CA 使用**自己的私钥**对证书内容（包括实体的公钥和身份信息）进行签名，生成数字证书。

4.  **证书验证**：
    客户端（如浏览器）收到证书后，使用**内置的 CA 公钥**验证证书上的签名。若验证通过，说明证书未被篡改，且确实由该 CA 签发。客户端随后提取证书中的服务器公钥用于后续通信。

### 证书链信任机制
在实际场景中，客户端通常不直接信任根 CA，而是通过证书链建立信任：

```text
                    [根证书] 
                   (预埋在OS/浏览器中)
                         | (签发)
                         v
              [中间证书] 
                         | (签发)
                         v
    [服务器证书] ---发给客户端---> [客户端/浏览器]
                     1. 验证服务器证书签名（使用中间证书公钥）
                     2. 验证中间证书签名（使用根证书公钥）
                     3. 检查有效期、吊销状态(CRL/OCSP)、域名匹配
```

### 证书内容详解
数字证书遵循 X.509 标准，主要包含：
- **版本号**：V1, V3 等版本。
- **序列号**：CA 唯一标识。
- **签名算法**：如 SHA256withRSA。
- **颁发者**：CA 的名称。
- **有效期**：起始日期和结束日期。
- **主体**：证书持有者的信息（如 Common Name: google.com）。
- **公钥信息**：算法及公钥本身。
- **扩展字段**：如 Key Usage（密钥用途）、Subject Alternative Name（SAN，多域名支持）。

### 证书验证相关对比
| 概念 | 作用 | 常见场景 | 备注 |
| :--- | :--- | :--- | :--- |
| **证书链** | 建立从服务器证书到受信根证书的信任路径 | 几乎所有 HTTPS 请求 | 需正确配置中间证书，否则报链不完整 |
| **CRL** | 证书撤销列表，下载并查询已吊销证书的序列号 | 传统金融系统 | 实时性差，文件可能很大 |
| **OCSP** | 在线证书状态协议，实时查询特定证书状态 | 现代浏览器 | 实时性好，但增加了 CA 服务器负载 |
| **SAN** | 主题备用名称，支持多域名 | 通配符证书、多域名证书 | Chrome 强制要求 SAN，单 CN 已不兼容 |

#### 实战案例
在配置 Nginx HTTPS 时，若只上传了服务器证书而忽略了中间证书，部分移动端客户端（尤其是较旧的 Android 版本）会报“证书不可信”导致连接失败。这是因为客户端本地只有根证书，无法构建完整的信任链。**解决**：将服务器证书和中间证书合并为一个 `.crt` 文件（服务器证书在前，中间证书在后）配置在 Nginx 中。

#### 关键代码示例 (Nginx 配置)
```nginx
server {
    listen 443 ssl;
    server_name example.com;
    
    # 合并证书文件：cat server.crt intermediate.crt > bundle.crt
    ssl_certificate     /etc/ssl/bundle.crt;
    ssl_certificate_key /etc/ssl/private.key;
    
    # 开启 OCSP Stapling 提高 TLS 握手性能
    ssl_stapling on;
    ssl_stapling_verify on;
}
```

## 常见考点
1. **HTTPS 握手过程中证书的作用**：面试官常问 TLS 握手细节，特别是服务器如何发送证书以及客户端如何验证证书链。
2. **证书续期与撤销**：了解 CRL（证书撤销列表）和 OCSP（在线证书状态协议）的区别及工作原理。
3. **自签名证书**：什么情况下使用？为什么浏览器会报不安全？（因为没有受信任的 CA 签名，需手动导入信任）。
4. **SAN 与 CN**：现代浏览器强制要求使用 SAN（Subject Alternative Name）来匹配域名，仅依靠 CN 已被废弃。

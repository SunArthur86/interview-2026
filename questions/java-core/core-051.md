# TCP和UDP的概念和特点是什么？

### TCP 和 UDP 的概念和特点

TCP 和 UDP 是传输层最主要的两种协议。

#### 1. TCP (传输控制协议)

*   **概念**：TCP 是一种面向连接的、可靠的、基于字节流的传输层通信协议。
*   **特点**：
    *   **面向连接**：通信前必须建立连接（三次握手），结束后需断开连接（四次挥手）。
    *   **可靠性**：通过序列号、确认应答（ACK）、超时重传机制保证数据无差错、不丢失、不重复、且按序到达。
    *   **流量控制与拥塞控制**：拥有滑动窗口机制防止接收方处理不过来，拥有拥塞控制机制防止网络拥塞。
    *   **点对点**：每一条 TCP 连接只能有两个端点。

#### 2. UDP (用户数据报协议)

*   **概念**：UDP 是一种无连接的传输层协议，提供面向事务的简单不可靠信息传送服务。
*   **特点**：
    *   **无连接**：不需要建立连接，发送数据直接启动。
    *   **不可靠**：没有确认机制，不重传，不保证有序到达，丢包不负责。
    *   **面向报文**：保留报文边界，应用层交给 UDP 多长的报文，UDP 就照样发送。
    *   **高效**：头部开销小（仅 8 字节），由于不需要建立连接和保证可靠，传输速度快。
    *   **支持多播/广播**：支持一对一、一对多、多对多的交互通信。

## 记忆要点

- TCP核心：面向连接，通过序列号、ACK和重传保证可靠有序。
- TCP机制：点对点通信，拥有滑动窗口实现流量与拥塞控制。
- UDP核心：无连接，不保证可靠与有序，直接发送面向报文的数据。
- UDP优势：首部仅8字节开销极小，传输高效，支持多播和广播。

# src 和 href 的区别

**src** (Source) 和 **href** (Hypertext Reference) 的核心区别在于**处理机制**不同：**src 用于替换当前元素**，**href 用于建立与资源的连接**。

### 一、src (替换资源)
src 是 source 的缩写，指向外部资源的位置，指向的内容将会**嵌入**到文档中当前标签所在位置。

- **作用**：将资源嵌入到文档当前标签内（如 `<script>`, `<img>`, `<iframe>`）。
- **加载行为**：当浏览器解析到 `src` 时，会**暂停**对当前文档的渲染（或脚本执行），去下载并执行/渲染该资源。这是因为该资源通常被视为文档内容不可或缺的一部分。
- **关键细节**：
  - `<script src="...">`：浏览器会下载并立即执行 JS。为了防止阻塞页面渲染（FOUC），通常建议将 JS 放在 `<body>` 底部或使用 `defer`/`async`。
  - `<img src="...">`：虽然是替换元素，但现代浏览器会进行预加载扫描，但若未指定宽高，图片加载完成后可能会引起页面布局抖动。

### 二、href (建立连接)
href 是 Hypertext Reference 的缩写，指向网络资源所在位置，建立和当前元素（锚点）或当前文档（链接）之间的链接。

- **作用**：建立当前文档与外部资源的关系（如 `<link>`, `<a>`）。
- **加载行为**：浏览器遇到 `href` 时，会**识别**该资源并行的去下载，**不会停止**对当前文档的解析和处理。这使得页面加载更加流畅。
- **关键细节**：
  - `<link href="..." rel="stylesheet">`：CSS 资源会并行下载，且浏览器会根据 CSSOM 构建规则进行渲染，不阻塞 HTML 解析（但可能会阻塞渲染，具体取决于浏览器策略）。
  - `<a href="...">`：仅仅是跳转的指引，不会下载目标内容直到用户点击。

### 三、实战案例
在排查首屏加载缓慢的问题时，发现开发者在 `<head>` 中直接引入了第三方库（使用 `src` 而非带有 `preload` 的 `link` 或合理的 `defer`/`async`），导致 HTML 解析被该大文件阻塞长达 800ms。改用 `<link rel="preload" href="..." as="script">` 预加载资源后，白屏时间减少了约 40%。

### 四、代码示例 (HTML)
```html
<head>
  <!-- href: 建立连接，并行下载，不阻塞解析 (CSS) -->
  <link rel="stylesheet" href="styles.css">
  
  <!-- src: 替换元素，阻塞渲染 (如果不加 defer/async) -->
  <!-- 推荐加载 JS 方式 -->
  <script src="app.js" defer></script> 
</head>
<body>
  <!-- src: 替换内容，若未设宽高会导致 CLS (Cumulative Layout Shift) -->
  <img src="hero-image.jpg" width="800" height="600" alt="Hero">
</body>
```

### 五、总结对比

| 特性 | src | href |
| :--- | :--- | :--- |
| **全称** | Source | Hypertext Reference |
| **核心动作** | 替换 | 链接/引用 |
| **解析行为** | 暂停当前文档处理，下载并嵌入资源 | 并行下载资源，不暂停当前文档处理 |
| **常见标签** | script, img, iframe, embed | link, a, @import |

### ## 常见考点
1.  **script 的 defer 和 async**：
    - `defer`：脚本延迟到 HTML 解析完成后，`DOMContentLoaded` 事件前执行。按顺序执行。
    - `async`：脚本加载完成后立即执行，不保证顺序，可能在 HTML 解析完成前或后执行。
    - 不加属性：阻塞 HTML 解析，立即执行。
2.  **为什么 CSS 放在 head 中，JS 放在 body 尾部？**
    - CSS 在 head：尽早构建 CSSOM，避免页面样式闪烁（FOUC），且渲染树构建依赖 CSSOM 和 DOM。
    - JS 在 body 尾部：避免 JS 执行阻塞 DOM 解析（若 JS 中有 DOM 操作），提高首屏加载速度（使用了 `defer/async` 除外）。

## 记忆要点

- 核心区别：src用于嵌入并替换当前元素，而href用于建立并行的资源连接
- 加载行为对比：src会暂停解析阻塞页面，而href并行下载不阻塞文档处理
- 常见归属：src常见于script/img/iframe，而href常见于link/a标签
- 异步处理：因script阻塞解析，所以用defer(顺序延迟)或async(立即乱序)优化

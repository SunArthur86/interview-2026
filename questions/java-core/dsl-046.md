# link 与 @import 的区别

引入 CSS 主要有 `link` 标签和 `@import` 两种方式。从功能和性能上考虑，**强烈推荐使用 `link`**。

### 一、语法结构

```html
<!-- link 方式：在 HTML 头部使用 -->
<link href="style.css" rel="stylesheet" type="text/css" />

<!-- @import 方式：在 CSS 文件或 <style> 标签中使用 -->
<style>
  @import url("style.css");
</style>
```

### 二、核心区别

| 维度 | Link 标签 | @import |
| :--- | :--- | :--- |
| **从属关系** | HTML 标签 | CSS 提供的语法规则 |
| **加载顺序** | **并行**加载，随 HTML 解析同时进行 | **串行**加载，待 HTML 解析完成后才加载 |
| **兼容性** | 无兼容性问题（HTML 元素） | CSS2.1 提出，IE5+ 才能识别（老旧浏览器问题） |
| **DOM 可控性** | 可通过 JS 动态插入 `link` 标签改变样式 | 无法通过 JS 动态插入（依赖 CSSOM 构建） |
| **权重** | 相比同源的 @import，权重稍高（但在同一级中通常后者覆盖前者） | 低于 link 引入的同名样式 |
| **其他功能** | 可定义 RSS、rel 连接属性 (如 icon)、prefetch 等 | 仅能导入 CSS 样式表 |

### 三、深入解析加载顺序与性能

**Link 的加载流程（推荐）：**
浏览器解析 HTML -> 遇到 `<link>` -> **并行**发起网络请求下载 CSS -> 继续解析 HTML（构建 DOM）。

**@import 的加载流程（不推荐）：**
浏览器解析 HTML -> 遇到 `<style>` 或 CSS 文件 -> 解析 CSS 内容 -> 遇到 `@import` -> **暂停** CSS 解析 -> 发起网络请求下载 CSS -> 下载完成并解析后，继续页面渲染。

这会导致 **FOUC (Flash of Unstyled Content)** 问题，即页面可能出现无样式内容闪烁，且延长了页面渲染完成的时间。

### 架构流程图

```text
页面加载时间轴对比：

[Link 模式 - 并行]
HTML 解析 ─────────────────────────────────────▶ DOM 构建
   │
   ├── [请求 CSS 1] ───▶ [CSS 1 完成]
   │
   └── [请求 CSS 2] ───▶ [CSS 2 完成]
                               │
                               └──▶ DOM + CSSOM = RenderTree

[@import 模式 - 串行阻塞]
HTML 解析 ────────────────────────▶ DOM 构建
   │
   └── [解析 CSS] ── 遇 @import ── [阻塞 CSS 解析]
                      │
                      └── [请求 CSS A] ───▶ [CSS A 完成]
                                            │
                                            └──▶ 继续解析 CSS ───▶ 渲染
```

### 四、实战案例与代码

**实战案例**：
在某次电商后台性能优化中，发现首屏 FCP（First Contentful Paint） 时间长达 1.5s。排查发现核心 CSS 文件中引用了一个第三方字体库使用了 `@import`，导致主 CSSOM 构建被阻塞，直到字体文件下载完才开始渲染。将 `@import` 替换为 HTML 中的 `<link>` 后，FCP 降至 0.8s。

**代码示例**：
```javascript
// 动态加载 CSS 的最佳实践（使用 Link）
function loadCSS(url) {
  const link = document.createElement('link');
  link.rel = 'stylesheet';
  link.href = url;
  // 可选：添加 preload 提示
  // link.as = 'style'; 
  document.head.appendChild(link);
}
// 使用 @import 无法简单地通过 JS 动态加载并保证执行顺序
```

### ## 常见考点
1.  **为什么 @import 会导致样式闪烁？**
    - 因为它是串行加载的，且只有在主 CSS 解析完成后才开始加载，导致页面在加载期间可能先显示无样式的内容（HTML），待 CSS 下载后才跳变。
2.  **预加载的区别：**
    - `link` 可以配合 `rel="preload"` 进行资源预加载，优先级更高，而 `@import` 无法做到这一点。
3.  **JS 动态插入样式：**
    - JS 无法动态创建 `@import` 规则来高效加载新文件（虽然可以通过 `insertRule` 实现，但不常用且复杂），而创建 `link` 元素并追加到 `head` 是标准的动态加载样式方案。

## 记忆要点

- 本质区别：link是HTML标签能操作DOM，而@import是CSS语法。
- 加载机制：link随HTML并行加载，而@import需等CSS解析串行加载，容易导致样式闪烁。
- 扩展功能：link能做预加载(preload)等，而@import仅限引入CSS资源。
- 操作限制：因为@import依赖CSSOM，所以无法用JS动态插入修改样式。

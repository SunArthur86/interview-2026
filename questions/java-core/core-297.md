# 最小生成树算法（Prim/Kruskal）的原理是什么？

**最小生成树**

### 基本概念
最小生成树（Minimum Spanning Tree, MST）是指在一个连通无向网中，构造一棵生成树，使得该树中所有边的权值之和最小。MST 包含 n 个顶点和 n-1 条边。

### 应用场景
假设要在 n 个城市之间建立通信网络，连通 n 个城市需要布置 n-1 条线路。如何选择线路使得总成本最低？城市是顶点，线路是边，成本是权重，成本最低的连线方案就是最小生成树。

### MST 性质（剪边法）
设 N=(V,{E}) 是一个连通网，U 是顶点集 V 的非空子集。如果 (u,v) 是一条具有最小权值的边，其中 u∈U，v∈V-U，则必定存在一棵包含边 (u,v) 的最小生成树。

### 常见算法对比

#### 1. Prim 算法（普里姆算法）- 点法
- **原理**：归并点。从任意一个顶点开始，将其加入集合 U。每次在“已连接集合 U”和“未连接集合 V-U”之间寻找一条权值最小的边 (u, v)，将 v 加入 U。直到所有顶点都在 U 中。
- **数据结构**：通常使用**优先队列（小顶堆）**来维护 U 到 V-U 的最小边，或者使用邻接矩阵记录 `lowcost` 数组。
- **复杂度**：
    - 邻接矩阵朴素版：O(n²)
    - 堆优化版：O(E log E)
- **适用**：**稠密图**（边多）。

#### 2. Kruskal 算法（克鲁斯卡尔算法）- 边法
- **原理**：归并边。将所有边按权值从小到大排序。依次考察每条边，如果这条边连接的两个顶点属于不同的连通分量（即不会形成环），则选择这条边加入生成树。直到选取了 n-1 条边。
- **数据结构**：**并查集（Union-Find）**，用于快速判断两个顶点是否连通以及合并连通分量。
- **复杂度**：O(E log E)，主要取决于边的排序时间。
- **适用**：**稀疏图**（边少）。

#### 流程对比图

```text
Prim 算法流程 (起点为 A):

初始: {A} -----------------------> {B, C, D, E}
      选最小的边 AB (2)

第1步: {A, B} -------------------> {C, D, E}
      选连接 AB 的最小边 BC (3)

第2步: {A, B, C} ----------------> {D, E}
      选连接 ABC 的最小边 CD (4)
      ... 直到所有点加入

------------------------------------------------

Kruskal 算法流程:

1. 排序所有边: (2,3,4,5,6,7...)
2. 取边2 (A-B): A,B不连通 -> 采纳
3. 取边3 (B-C): B,C不连通 -> 采纳 (此时连通块 {A,B,C})
4. 取边4 (C-D): C,D不连通 -> 采纳 (连通块 {A,B,C,D})
5. 取边5 (A-D): A,D已连通(同属{ABCD}) -> 舍弃(成环)
6. ... 直到选够 n-1 条边
```

### 代码实现

#### Prim 算法 (堆优化版思路)
```java
// 伪代码示意
void prim(Graph G, int startNode) {
    PriorityQueue<Edge> pq = new PriorityQueue<>(); // 存储候选边
    boolean[] visited = new boolean[G.V()];
    pq.addAll(G.getEdges(startNode)); // 加入起点所有邻边
    visited[startNode] = true;
    
    int totalWeight = 0;
    while (!pq.isEmpty()) {
        Edge e = pq.poll(); // 取权值最小边
        if (visited[e.to]) continue; // 如果该点已访问，跳过（防止成环）
        
        visited[e.to] = true;
        totalWeight += e.weight;
        System.out.println("Add edge: " + e);
        
        // 将新加入点的邻边加入优先队列
        for (Edge next : G.getEdges(e.to)) {
            if (!visited[next.to]) pq.add(next);
        }
    }
}
```

#### Kruskal 算法 (并查集版)
```java
void kruskal(List<Edge> edges, int vertexCount) {
    // 1. 排序
    Collections.sort(edges, Comparator.comparingInt(e -> e.weight));
    
    UnionFind uf = new UnionFind(vertexCount);
    int totalWeight = 0;
    int count = 0;
    
    // 2. 遍历边
    for (Edge e : edges) {
        if (uf.find(e.u) != uf.find(e.v)) {
            uf.union(e.u, e.v); // 合并两个连通分量
            totalWeight += e.weight;
            System.out.println("Add edge: " + e);
            count++;
            if (count == vertexCount - 1) break; // 已选够 n-1 条边
        }
    }
}
```

### ## 常见考点
1.  **如何判断环**：Kruskal 算法中如何高效判断加边是否会成环？（使用并查集，两点根节点相同则连通）。
2.  **算法选择**：给一个图是稀疏还是稠密，应该选哪种算法？（Prim 适合稠密，Kruskal 适合稀疏）。
3.  **次小生成树**：了解如何求次小生成树（严格次小需要维护最大边权）。
4.  **唯一性**：最小生成树唯一吗？（如果所有边的权值都不相等，则MST唯一；如果有等权边，可能不唯一）。

## 记忆要点

- 核心目标口诀：连通n个顶点只选n-1条边，使得总权值最小绝不成环
- Prim算法（点法）：归并点，每次找连通集到非连通集的最短边，适合稠密图
- Kruskal算法（边法）：归并边，全部边排序后从小到大挑（并查集查环），适合稀疏图
- 数据结构对比：Prim靠优先队列维护候选边，Kruskal靠并查集防成环

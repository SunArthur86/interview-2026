# 提供生产就绪型功能，如指标，健康检查和外部配置

Spring Boot 提供了生产环境就绪的功能，直接监控应用的健康状况和运行指标，无需额外开发。

### 主要功能
1. **指标**：自动收集内存使用、JVM 线程、HTTP 请求统计等运行时指标。
2. **健康检查**：提供 `/actuator/health` 端点，检查数据库连接、磁盘空间等关键组件状态。
3. **外部化配置**：支持通过环境变量、配置文件、命令行参数等方式动态配置，方便在不同环境（开发、测试、生产）间切换。

这些功能通过 Spring Boot Actuator 模块实现，帮助运维人员实时监控系统状态。

### 深入原理与架构
Actuator 核心基于 `Endpoint`（端点）概念。每个端点暴露特定的监控数据。支持多种输出格式（HTTP, JMX）。

- **Metrics 原理**：集成 Micrometer 库，作为门面模式适配多种监控系统（Prometheus, InfluxDB）。它通过拦截 JVM `BufferPoolMXBean`、Spring MVC 请求处理链等收集数据。
- **Health 原理**：通过 `HealthIndicator` 接口。系统自动装配各种 Indicator（如 `DataSourceHealthIndicator`, `DiskSpaceHealthIndicator`），通过组合模式汇总最终状态。

### 深化实战

**实战案例**：
某次生产环境故障排查中，通过 Actuator 的 `/actuator/heapdump` 端点下载堆内存快照，发现 OOM 是由未限制速率的第三方 API 调用导致的。此外，利用 Prometheus + Grafana 接入 Metrics，在 QPS 突增时提前收到了告警通知。

**代码示例**：
```java
// 自定义业务健康检查（如核心支付渠道状态）
@Component
public class PaymentChannelHealthIndicator implements HealthIndicator {
    @Override
    public Health health() {
        boolean isUp = checkPaymentChannelStatus(); // 调用下游接口检查
        if (isUp) {
            return Health.up().withDetail("channel", "AliPay").build();
        }
        return Health.down().withDetail("error", "Connection Timeout").build();
    }
}
```

## 常见考点
1. **如何自定义 Health Indicator？**（实现 `HealthIndicator` 接口并注册为 Bean）
2. **如何保护 Actuator 端点安全？**（结合 Spring Security，仅允许特定 IP 或内部网络访问，或使用 `management.endpoints.web.exposure.include` 进行严格控制）

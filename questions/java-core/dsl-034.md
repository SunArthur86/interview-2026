# 基础知识讲解

### 在浏览器中输入 URL 并按下回车之后会发生什么

这是一个经典的综合面试题，涉及计算机网络、浏览器渲染原理、操作系统等知识。整个过程可以概括为以下几个步骤：

#### 1. URL 解析与缓存检查
- 浏览器解析 URL，提取协议、主机名、端口、路径等信息。
- **检查缓存**：浏览器会根据请求头判断是否有强缓存，如果有且未过期，直接使用缓存，不发起请求。

**实战案例**：曾遇到过上线新版本后，部分用户因强缓存策略未设置 `Cache-Control: no-cache` 导致看不到更新，最终通过在 HTML 文件名中增加 Hash 值（如 `app.ab12.js`）强制失效缓存解决。

#### 2. DNS 域名解析
- 浏览器需要将域名转换为 IP 地址。
- **解析顺序**：
  1. **浏览器缓存**：检查浏览器是否缓存过该域名对应的 IP。
  2. **系统缓存**：检查本地的 Hosts 文件。
  3. **路由器缓存**：部分路由器会缓存 DNS 记录。
  4. **ISP DNS 服务器**：向本地运营商（如移动、电信）的 DNS 服务器发起查询。
  5. **根域名服务器 -> 顶级域名服务器 -> 权威域名服务器**：通过迭代查询最终获取 IP。

#### 3. 建立 TCP 连接（三次握手）
- 拿到 IP 后，浏览器通过 TCP 协议与服务器建立连接。
- **三次握手**：
  1. 客户端发送 SYN 包，`seq=x`，进入 `SYN_SENT` 状态。
  2. 服务端收到 SYN，回复 SYN+ACK 包，`seq=y, ack=x+1`，进入 `SYN_RCVD` 状态。
  3. 客户端收到 SYN+ACK，回复 ACK 包，`ack=y+1`，双方进入 `ESTABLISHED` 状态。
- *如果使用 HTTPS，在 TCP 连接后会进行 TLS/SSL 握手。*

#### 4. 发送 HTTP 请求
- 浏览器构建 HTTP 请求报文（包含请求行、请求头、请求体），通过 socket 发送。

#### 5. 服务器处理请求并返回响应
- 服务器接收请求，Web 服务器（如 Nginx, Apache）处理路由、反向代理等，将请求交给应用服务器（如 Tomcat, Go Server）。
- 应用服务器处理逻辑，生成 HTTP 响应报文（包含状态码、响应头、响应体）。

#### 6. 浏览器渲染页面
- **解析 HTML**：构建 DOM 树。
- **解析 CSS**：构建 CSSOM 树。
- **合并生成渲染树**：将 DOM 和 CSSOM 合并。
- **布局**：计算每个节点的位置和大小。
- **绘制**：将节点绘制到屏幕上。

**实战案例**：首屏加载优化中，我们会将 CSS 放在 `<head>` 中（阻塞渲染，防止 FOUC），JS 放在 `<body>` 底部或使用 `defer/async`，避免 DOM 解析阻塞，曾通过此操作将 LCP 指标降低了 40%。

#### 7. 断开 TCP 连接（四次挥手）
- 数据传输完毕，断开连接。
- **四次挥手**：
  1. 客户端发送 FIN 包，`seq=u`，进入 `FIN_WAIT_1`。
  2. 服务端收到 FIN，回复 ACK，`ack=u+1`，进入 `CLOSE_WAIT`。
  3. 服务端发送 FIN 包，`seq=w`，进入 `LAST_ACK`。
  4. 客户端收到 FIN，回复 ACK，`ack=w+1`，进入 `TIME_WAIT`，等待 2MSL 后关闭。

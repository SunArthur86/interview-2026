# Vue-router

### Vue Router

#### 1. 路由模式

**Hash 模式**
*   **原理**：利用 URL 中的 `#` 后面的内容。通过监听 `hashchange` 事件来触发路由更新。
*   **特点**：兼容性好（兼容 IE）。`#` 后面的内容不会包含在 HTTP 请求中，因此不需要后端配置即可处理前端路由。但 URL 带有 `#`，不美观，且不利于 SEO（搜索引擎通常忽略 hash 内容）。
*   **底层细节**：当 hash 改变时，浏览器历史记录会新增一条记录，但不会触发页面刷新。

**History 模式**
*   **原理**：利用 HTML5 History API (`pushState`, `replaceState`) 改变 URL 而不刷新页面。通过监听 `popstate` 事件来响应浏览器前进后退操作。
*   **特点**：URL 美观，不带 `#`，利于 SEO。但刷新页面或直接访问深层次 URL（如 `/user/123`）时，会向后端发送请求。如果后端没有配置对应的路由接口，会返回 **404**。
*   **服务端配置**：需在 Nginx/Apache 等服务器配置 "Fallback" 策略，将所有非静态资源的请求重定向到 `index.html`，由前端路由接手处理。
*   **注意事项**：在单页应用中，调用 `pushState` 不会触发 `popstate` 事件，只有点击浏览器前进/后退按钮才会触发。

**Memory 模式**
*   **原理**：不依赖 URL，而是将路由状态保存在内存中的数组栈里。
*   **场景**：适用于非浏览器环境（如 Node.js 服务端渲染测试、移动端原生 App 容器），URL 始终保持不变。

#### 2. 路由传参

**Query 传参**
*   **方式**：`this.$router.push({ path: '/user', query: { id: 123 } })`
*   **URL表现**：`/user?id=123`
*   **获取**：`this.$route.query.id`
*   **特点**：参数拼接在 URL 后，刷新不丢失，类似 GET 请求。适合传递非敏感、可选的参数。

**Params 传参**
*   **动态路由匹配**：
    *   配置：`{ path: '/user/:id', component: User }`
    *   方式：`this.$router.push({ path: '/user/123' })`
*   **命名路由传参**：
    *   配置：`{ path: '/user', name: 'User', component: User }`
    *   方式：`this.$router.push({ name: 'User', params: { id: 123 } })`
*   **获取**：`this.$route.params.id`
*   **特点（重要）**：使用命名路由且不配置动态路径时，参数不会显示在 URL 中，**刷新页面参数会丢失**（因为参数没保存在 URL 里，仅保存在内存 history state 中）。若配置了动态路径（如 `/user/:id`），参数会显示在 URL 中，刷新不丢失。

#### 3. 全局路由守卫
完整的导航解析流程非常关键：

```text
触发导航
   │
   ▼
离开守卫: router.beforeEach (全局前置)
   │
   ├─> 返回 false: 取消导航
   └─> 返回路由对象/字符串: 重定向
   │
   ▼
复用组件: router.beforeRouteUpdate (组件内)
   │
   ▼
进入路由配置: router.beforeEnter (路由独享)
   │
   ▼
异步组件: 解析异步组件
   │
   ▼
进入组件: router.beforeRouteEnter (组件内)
   │
   ▼
全局解析守卫: router.beforeResolve (全局解析)
   │
   ▼
导航被确认
   │
   ▼
全局后置钩子: router.afterEach
   │
   ▼
DOM 更新触发
```

*   **参数**：`to, from, next` (Vue 3 支持返回值代替 next)。
*   **应用场景**：
    *   `beforeEach`：登录鉴权、权限控制。
    *   `afterEach`：修改页面 Title、页面滚动条复位。

#### 4. $router vs $route
*   **$router**：全局路由实例对象（`new VueRouter` 得到的实例）。包含 `push`, `go`, `replace`, `addRoutes` (Vue 2) / `addRoute` (Vue 3) 等控制路由跳转的方法。
*   **$route**：当前激活的路由信息对象（只读）。包含 `path`, `params`, `query`, `meta`, `matched` (当前路由匹配的所有路由记录) 等信息。

#### 5. 路由懒加载
*   结合 Vue 的异步组件和 Webpack 的代码分割功能：
    ```javascript
    const User = () => import('./User.vue')
    ```
*   **作用**：打包时将对应组件分离到单独的 chunk 文件中，当访问该路由时才加载，提升首屏加载速度。

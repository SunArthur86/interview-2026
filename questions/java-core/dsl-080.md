# Fiber

Fiber 是 React 16 引入的核心架构重写，旨在解决大型应用的性能瓶颈和交互卡顿问题。

### 1. 背景：React 15 的痛点

*   **同步渲染（递归）**：React 15 使用递归遍历 Virtual DOM 树。一旦开始更新，无法被打断，直到整棵树遍历完成。
*   **主线程阻塞**：JS 是单线程的，长时间的递归计算会占用主线程，导致浏览器无法响应用户交互（如点击、输入）和页面动画（掉帧）。

### 2. Fiber 的核心思想

*   **链表结构**：将原来的递归树结构变成了基于链表的“单元任务”结构。每个节点是一个 Fiber 节点。
*   **可中断的执行**：利用浏览器的 `requestIdleCallback` 或时间切片机制，将渲染任务拆分成小单元。每执行完一个单元，检查是否还有剩余时间。如果有，继续；如果没有，交还控制权给浏览器（处理输入/绘制），等下次空闲再继续。
*   **双缓存**：Fiber 架构中存在两棵树：Current Tree（当前屏幕显示的）和 WorkInProgress Tree（正在内存中构建的）。当 WorkInProgress 完成后，一次性替换指针，用户感觉不到中间过程。

### 3. Fiber 节点结构

每个 Fiber 节点包含三个重要指针，构成链表：

```text
┌──────────────────────────────────────┐
│            Fiber Node                 │
├──────────────────────────────────────┤
│  stateNode: 组件实例/DOM节点          │
│  props/state                          │
├──────────────────────────────────────┤
│  child:   第一个子节点      ───┐      │
│  sibling: 下一个兄弟节点    ───┼─►   │
│  return:  父节点          ◄──┘      │
└──────────────────────────────────────┘
```

### 4. 协程

Fiber 在 JS 层面实现了一种“协程”机制。它是一个可以暂停和恢复的执行上下文，允许 React 在执行单元之间“让出”CPU，从而实现并发模式的调度基础。

## 常见考点

1.  **为什么 React 15 的递归会导致卡顿？Fiber 是怎么解决的？**
    *   **考点**：递归不可中断导致主线程长任务阻塞；Fiber 通过链表遍历 + 时间切片实现任务可中断。
2.  **Fiber 的双缓存机制是什么？**
    *   **考点**：Current 树和 WorkInProgress 树。直接在 DOM 上修改会导致中间状态闪烁，Fiber 先在内存构建好整个树，确认无误后直接切换指针。
3.  **Fiber 带来的副作用（性能方面）？**
    *   **考点**：Fiber 的链表遍历比递归稍微耗时（内存占用略增），且在大量组件时，由于调度开销，首次渲染可能会比 15 略慢，但在交互响应性和大型应用更新性能上提升巨大。

### 深化内容

#### 实战案例
在 React 15 时代，如果一个页面包含深度嵌套的组件树（如 1000+ 层级），每次 setState 都会导致**长时间不可中断的 JS 执行**，此时用户点击按钮毫无反应（白屏）。Fiber 引入后，即使组件树很深，React 会将渲染任务切片（每帧约 5ms），用户点击会打断当前的渲染，优先响应点击，然后再恢复渲染，彻底解决了大型应用的“假死”现象。

#### 代码示例：模拟 Fiber 调度（简化版）
```javascript
// 模拟 Fiber 的工作循环：每个 Fiber 节点作为一个任务单元
function workLoop(deadline) {
  while (nextUnitOfWork && deadline.timeRemaining() > 1) {
    nextUnitOfWork = performUnitOfWork(nextUnitOfWork);
  }
  
  if (nextUnitOfWork) {
    // 如果时间片用完，请求下一帧继续
    requestIdleCallback(workLoop);
  } else {
    // 所有任务完成，提交到 DOM
    commitRoot();
  }
}
```

#### 对比表格：React 15 Stack Reconciler vs React 16+ Fiber Reconciler

| 维度 | Stack Reconciler (React 15) | Fiber Reconciler (React 16+) |
| :--- | :--- | :--- |
| **遍历算法** | 递归 | 链表遍历 (支持中断/恢复) |
| **更新过程** | 同步，不可中断 | 异步，可中断 (时间切片) |
| **任务调度** | 无优先级，先来先到 | 有优先级 (高优任务如输入优先) |
| **DOM 更新** | 直接更新 | 双缓存 |

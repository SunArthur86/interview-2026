# MyBatis

### MyBatis 解决 JDBC 的问题
1.  **连接管理**：使用数据库连接池（如 Druid、HikariCP）管理连接，避免频繁创建释放，解决资源耗尽问题。
2.  **SQL 解耦**：SQL 语句写在 XML 配置文件或注解中，与 Java 代码分离，便于维护和优化。
3.  **参数映射**：自动将 Java 对象的属性映射到 SQL 预编译语句（PreparedStatement）的参数位（`?`），防止 SQL 注入。
4.  **结果映射**：自动将 SQL 结果集映射到 Java 对象（POJO），支持复杂的嵌套映射和集合映射。


### MyBatis 插件原理
- **拦截对象**：Executor (执行器), StatementHandler (语句处理器), ParameterHandler (参数处理器), ResultSetHandler (结果集处理器)。
- **实现机制**：基于 JDK 动态代理 + 责任链模式。在配置插件时，MyBatis 会为这四大对象生成代理对象。代理对象在执行目标方法前后，会遍历拦截器链中的所有拦截器，执行拦截逻辑。
- **关键细节**：插件必须实现 `Interceptor` 接口，并指定 `@Signature` 注解（包含接口、方法、参数类型）来确定拦截的具体方法。

### Dao 接口与 XML 关联
1.  XML 中每个 SQL 标签（`<select>`, `<insert>` 等）封装为一个 `MappedStatement` 对象，其 `id` 属性默认为 `全限定接口名+方法名`（如 `com.example.UserMapper.selectById`）。
2.  运行时 MyBatis 使用 JDK 动态代理为 Mapper 接口生成代理对象（MapperProxy）。
3.  调用接口方法时，代理对象拦截调用，根据方法名和接口名组装 ID，从 `Configuration` 对象中找到对应的 `MappedStatement`，然后交由 Executor 执行 SQL。

#### 深化实战补充

1. **实战案例（拦截器实现分页与加密）**：
   在一个涉及敏感数据的报表系统中，通过实现 MyBatis 插件拦截 `Executor.update` 方法，自动对包含手机号字段的数据进行 AES 加密存储，并在 `ResultSetHandler` 阶段解密，业务层无感知。此外，利用 PageHelper 插件实现物理分页，避免内存 OOM。

2. **代码示例（TypeHandler 自定义类型转换）**：
   ```java
   // 示例：将 Java 中的 List<String> 存储为 MySQL 的 VARCHAR (逗号分隔)
   @MappedTypes(List.class)
   @MappedJdbcTypes(JdbcType.VARCHAR)
   public class ListTypeHandler extends BaseTypeHandler<List<String>> {
       @Override
       public void setNonNullParameter(PreparedStatement ps, int i, List<String> parameter, JdbcType jdbcType) throws SQLException {
           ps.setString(i, String.join(",", parameter));
       }
       @Override
       public List<String> getNullableResult(ResultSet rs, String columnName) throws SQLException {
           String val = rs.getString(columnName);
           return val == null ? null : Arrays.asList(val.split(","));
       }
   }
   ```

3. **对比表格（#{} vs ${}）**
   | 特性 | #{} | ${} |
   | :--- | :--- | :--- |
   | **SQL 解析** | 预编译，生成参数占位符 `?` | 字符串直接拼接 |
   | **安全性** | 安全，防止 SQL 注入 | 危险，严禁用于用户输入 |
   | **参数类型** | 自动处理类型转换 | 仅做字符串替换 |
   | **适用场景** | 绝大多数 SQL 参数 | 动态表名、排序字段 (`ORDER BY ${column}`) |

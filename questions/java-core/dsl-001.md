# JavaSE

### JVM、JRE、JDK 关系
- **JDK** (Java Development Kit)：Java 开发工具包，包含 JRE 和开发工具（javac 编译器、jdb 调试器、javadoc 等）。
- **JRE** (Java Runtime Environment)：Java 运行时环境，包含 JVM 和核心类库（如 rt.jar, java.* 包），用于运行 Java 程序。
- **JVM** (Java Virtual Machine)：Java 虚拟机，负责将字节码翻译成特定平台的机器码，实现跨平台的核心。

**架构关系图：**
```text
      JDK (Java Development Kit)
    ┌─────────────────────────────────┐
    │                                 │
    │     开发工具      ┌─────────────┴───────┐
    │  (javac, jar...) │    JRE               │
    │                  │  ┌────────────────┐ │
    │                  │  │   类库         │ │
    │                  │  │  (String... )  │ │
    │                  │  └────────┬───────┘ │
    │                  │           │         │
    │                  │  ┌────────▼───────┐ │
    │                  │  │      JVM       │ │
    │                  │  │  (解释器/JIT)  │ │
    │                  │  └────────────────┘ │
    └──────────────────┴─────────────────────┘
```
简而言之：JDK > JRE > JVM。开发用 JDK，运行只需 JRE，核心在 JVM。

### 基本数据类型
Java 有 8 种基本数据类型，注意各类型的字节大小及取值范围：
- **整型**：
  - byte (1字节, -128~127)
  - short (2字节)
  - int (4字节, **默认类型**)
  - long (8字节, 赋值需加 `L` 或 `l`)
- **浮点型**：
  - float (4字节, 赋值需加 `F` 或 `f`)
  - double (8字节, **默认类型**)
- **字符型**：char (2字节, 采用 Unicode 编码)
- **布尔型**：boolean (只允许 true 或 false，JVM 规范未明确指定字节大小，通常为 1 字节)

### 自动装箱与拆箱
- **自动装箱**：基本类型 -> 包装类（编译器调用 `valueOf` 方法，如 `Integer.valueOf(10)`）。
- **自动拆箱**：包装类 -> 基本类型（编译器调用 `xxxValue` 方法，如 `intValue()`）。
- **注意点**：
  1. 包装类对象值的比较必须使用 `equals`，`==` 比较的是内存地址。
  2. **缓存机制**：`IntegerCache` 缓存了 -128 到 127 之间的 Integer 对象。在此范围内，`==` 返回 true，否则返回 false（Byte, Short, Long 类似；Character 是 0-127；Boolean 是 true/false）。

### 值传递 vs 引用传递
Java 只有**值传递**（Pass-by-value），不存在引用传递。
- **基本类型**：传递的是数据值的副本，修改副本不影响原值。
- **引用类型**：传递的是对象地址值的副本。
  - 可以通过副本地址修改对象内部属性（如 list.add()）。
  - 但无法改变原引用指向的对象（如 `new Object()`），因为只是修改了副本的指向。

#### 深化实战补充

1. **实战案例（Integer 缓存陷阱）**：
   在做金融对账系统时，遇到金额比较逻辑错误。代码使用 `Integer` 包装类且通过 `==` 比较，当金额超过 127 时判定为不相等。修复方案是统一使用 `Long` 类型或使用 `.equals()` 方法。生产环境严禁使用包装类进行 `==` 运算（除判空外）。

2. **代码示例（值传递经典误区）**：
   ```java
   public class SwapTest {
       public static void swap(StringBuffer a, StringBuffer b) {
           a.append(b); // 修改了堆中对象内容，外部可见
           b = a;       // 仅修改了副本引用，外部不可见
       }
       public static void main(String[] args) {
           StringBuffer s1 = new StringBuffer("A");
           StringBuffer s2 = new StringBuffer("B");
           swap(s1, s2);
           System.out.println(s1 + "," + s2); // 输出 AB, B 而不是 AB, AB
       }
   }
   ```

3. **对比表格（int vs Integer）**
   | 维度 | int (基本类型) | Integer (包装类) |
   | :--- | :--- | :--- |
   | **存储方式** | 存储在栈中（局部变量）或堆（常量池） | 引用类型，对象在堆，引用在栈 |
   | **默认值** | 0 | null |
   | **泛型支持** | 不支持 | 支持（如 `List<Integer>`） |
   | **比较方式** | `==` 比较值 | 必须使用 `.equals()` 比较值 |
   | **线程安全** | 天然线程安全 | 非线程安全（对象本身可变） |
   | **性能** | 高，无装箱拆箱开销 | 较低，涉及对象创建和内存占用 |

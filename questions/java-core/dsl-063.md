# Webpack

### Webpack 核心概念与构建流程

Webpack 是一个静态模块打包工具。它分析项目结构，找到 JavaScript 模块以及其它的一些浏览器不能直接运行的拓展语言（Scss，TS 等），并将其打包为合适的格式以供浏览器使用。

#### 核心概念
1.  **Entry（入口）**：指示 Webpack 从哪个文件开始构建依赖图。
2.  **Output（输出）**：告诉 Webpack 在哪里输出打包后的 bundle，以及如何命名。
3.  **Loader（加载器）**：让 Webpack 能够处理非 JavaScript 文件（如 CSS、图片、Vue），将其转换为有效模块。
4.  **Plugin（插件）**：用于执行范围更广的任务，从打包优化和压缩，到重新定义环境中的变量。
5.  **Mode（模式）**：`development` 或 `production`，开启相应模式下的内置优化。

### 关键机制

#### 1. HMR (Hot Module Replacement)
允许在不刷新浏览器的情况下更新模块。
*   **原理**：通过 websocket 建立连接，监听文件变化。当文件变化时，Webpack 重新编译并发送两个补丁文件：`manifest.json`（更新说明）和 `update.js`（更新代码）。浏览器收到后，利用 HMR runtime 替换旧模块。

#### 2. Tree Shaking
用于消除死代码（未引用的代码）。
*   **原理**：依赖于 ES6 Module 的静态分析特性（`import`/`export`）。在 `production` 模式下，Webpack 通过 `UglifyJS` 或 `Terser` 插件删除未被 import 的代码。
*   **注意**：CommonJS (require) 因为是动态加载，无法进行 Tree Shaking。

#### 3. Code Splitting
将代码分割成多个 bundle，实现按需加载。
*   **方式**：
    1.  多入口（entry）。
    2.  `SplitChunksPlugin`：提取公共代码（如 node_modules）。
    3.  动态 import（`import()`）：实现懒加载。

#### 4. Source Map
将编译后的代码映射回源代码，便于调试。
*   `eval`：速度最快，不生成 SourceMap，仅用于开发。
*   `source-map`：生成独立 map 文件，质量最高。
*   `cheap-module-source-map`：平衡点，生产环境常用。

## 常见考点
1.  **Loader 和 Plugin 的区别是什么？**（Loader 转换文件，Plugin 扩展功能/作用于整个构建周期）
2.  **Tree Shaking 的原理及使用要求？**（必须使用 ES6 Module）
3.  **提高 Webpack 构建速度的优化手段？**（DllPlugin、多线程 Happypack、缓存 Cache、缩小构建范围 exclude/include）

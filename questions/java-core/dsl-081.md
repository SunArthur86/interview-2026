# dangerouslySetInnerHTML

### dangerouslySetInnerHTML

是 React 中 `innerHTML` 的替代品。它是 React 中 HTML 元素的一个属性，其命名中的 `dangerously` 就是为了警示开发者，使用它可能会带来安全风险（XSS 跨站脚本攻击），特别是当数据来自第三方或用户提交的内容时。

#### 原理与机制
React 会重写浏览器的 DOM 处理逻辑。普通的 React 元素渲染会经过 Virtual DOM Diff 和 JSX 转义，确保注入的内容是安全的文本。而 `dangerouslySetInnerHTML` 会绕过 React 的转义机制，直接将 HTML 字符串插入到 DOM 节点的 `innerHTML` 属性中。

如果 HTML 字符串中包含恶意的 `<script>` 标签或带有事件处理器（如 `onload`）的元素，浏览器会直接执行这些脚本。

```javascript
div dangerouslySetInnerHTML={{ __html: newsTexts }} />
```

这里传入的对象键名必须为 `__html`，值为要渲染的 HTML 字符串。

#### XSS 防护
为了安全起见，建议在使用前配合 HTML 净化工具（如 **DOMPurify**）对内容进行清洗，移除潜在的恶意脚本。

```javascript
import DOMPurify from 'dompurify';

// 清洗 HTML
const cleanHtml = DOMPurify.sanitize(userInput);

<div dangerouslySetInnerHTML={{ __html: cleanHtml }} />
```

## 常见考点
1. **为什么要叫 dangerouslySetInnerHTML？**
   - 为了明确警示开发者这会绕过 React 的安全检查，如果使用不当极易导致 XSS 攻击。
2. **React 为什么不直接使用 innerHTML 属性？**
   - 因为 React 采用 JSX 语法，为了符合 JS 对象规范并防止开发者误用，故意设计为这种略显繁琐的对象传值方式，强调其危险性。
3. **使用场景有哪些？**
   - 渲染富文本编辑器的内容（如 Draft.js, Quill 输出的 HTML）。
   - 展示从后端获取的带格式文本，且后端无法直接返回 JSON 结构化数据时。

### 深化内容

#### 实战案例
在一个**博客后台**中，管理员通过富文本编辑器发布文章，前端直接 `dangerouslySetInnerHTML` 渲染。某天管理员电脑中毒，文章内容被插入了 `<img src=x onerror=alert(1)>`。由于未做清洗，所有访问该文章的用户都弹出了警告框（甚至可能被盗取 Cookie）。引入 **DOMPurify** 后，此类恶意标签被自动过滤，但保留了正常的图片和样式。

#### 代码示例：服务端/客户端双重清洗
```javascript
import DOMPurify from 'dompurify';

// 配置只允许特定标签和属性，如 <p>, <br>, <strong>
const cleanConfig = { 
  ALLOWED_TAGS: ['p', 'b', 'i', 'u', 'br'],
  ALLOWED_ATTR: [] 
};

const UserContent = ({ htmlContent }) => {
  // 即使后端已经清洗过，前端再次清洗可防止漏网之鱼
  const safeHtml = DOMPurify.sanitize(htmlContent, cleanConfig);
  return <div className="content" dangerouslySetInnerHTML={{ __html: safeHtml }} />;
};
```

#### 对比表格：安全渲染方案对比

| 方案 | 安全性 | 灵活性 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **JSX 直接渲染** | 最高 (自动转义) | 低 (无法渲染 HTML 标签) | 纯文本展示，绝大多数场景 |
| **dangerouslySetInnerHTML** | 最低 (裸奔) | 最高 (任意 HTML) | 仅限可信内容 (如静态构建的 HTML) |
| **dangerously + DOMPurify** | 高 (可控) | 中 (白名单控制) | 富文本、UGC 内容展示 |
| **第三方组件 (Slate等)** | 高 (结构化数据) | 高 (自定义节点) | 编辑器场景，复杂文档交互 |

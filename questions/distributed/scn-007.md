# 什么是Spring Boot Actuator？如何利用它进行生产级监控？

🎯 本质：Spring Boot Actuator提供生产就绪功能（健康检查、指标、审计、日志等），是Spring Boot运维监控的核心组件。

**1. 核心端点与功能**
| 端点 | 功能 | 生产建议 |
|------|------|----------|
| /health | 健康状态（DB/Redis/MQ等） | 必须开启，配合K8s Probe |
| /info | 应用信息 | 开启，展示Git版本、构建时间 |
| /metrics | Prometheus格式的指标 | 必须开启，用于监控大盘 |
| /env | 环境变量 | **关闭**，防止配置泄露 |
| /loggers | 日志级别管理 | 仅内网开启，用于临时调参 |
| /threaddump | 线程转储 | 仅内网开启，排查死锁 |
| /heapdump | 堆转储 | 仅内网开启，排查OOM |
| /shutdown | 优雅停机 | **强烈建议关闭** |

**2. 监控架构集成图**：
┌──────────────┐      Pull      ┌──────────────────────┐
│  Prometheus  │ <───────────── │  Spring Boot App     │
│ (Metrics DB) │  /actuator/... │  :8080               │
└──────┬───────┘                 └──────────────────────┘
       │                               │
       │ Push (Alerts)                 │ Integration
       ↓                               ↓
┌──────────────┐               ┌──────────────────────┐
│ AlertManager │               │  Grafana Dashboard   │
│ (告警路由)   │               │  (可视化展示)        │
└──────────────┘               └──────────────────────┘

**3. 安全配置**
```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus,loggers  # 白名单模式
      base-path: /actuator
  endpoint:
    health:
      show-details: when-authorized  # 仅认证用户看详情
  server:
    port: 9090  # 物理隔离管理端口，防止业务端口被扫描
```

**4. 与Prometheus + Grafana集成**
1. 依赖：`micrometer-registry-prometheus`
2. 配置：Actuator自动暴露 `/actuator/prometheus` 端点。
3. 抓取：Prometheus配置 `scrape_configs`。
4. 展示：Grafana导入现成的 JVM Micrometer Dashboard (ID: 4701 等)。

**5. 自定义业务指标**
通过 `MeterRegistry` 自定义 Counter、Gauge、Timer。
```java
@Service
public class OrderService {
    private final MeterRegistry registry;
    private final Counter orderCounter;

    public OrderService(MeterRegistry registry) {
        this.registry = registry;
        // 定义带Tag的计数器
        this.orderCounter = Counter.builder("order.created")
            .tag("region", "cn")
            .description("Number of orders created")
            .register(registry);
    }

    public void createOrder() {
        // 业务逻辑...
        orderCounter.increment();
    }
}
```

**实战案例**：在电商大促期间，通过自定义 `Timer` 指标监控“第三方支付接口”的调用耗时。当 P99 延迟超过阈值时，结合 AlertManager 自动发送告警至钉钉群，帮助团队在流量洪峰前快速发现外部依赖的性能退化。注意高并发下不要在业务代码中频繁创建 Meter 对象，应在 Bean 初始化时注册并复用。

## 记忆要点

- 端点安全策略：health/metrics端点对外暴露，而env/shutdown等高危端点必须关闭或物理隔离端口。
- 监控闭环：Actuator与Prometheus和Grafana深度集成，提供标准的拉取指标格式。
- 自定义指标：高并发下必须复用MeterRegistry注册的Counter/Timer对象，禁止在业务代码中频繁创建。

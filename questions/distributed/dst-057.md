# Seata 的三大模块

Seata 包含三大核心模块：TC (Transaction Coordinator)、TM (Transaction Manager) 和 RM (Resource Manager)。

### 实战案例
在 K8s 环境部署时，TC (Seata Server) 通常作为独立的 StatefulSet 部署以存储事务日志，而 TM 和 RM 则作为 Sidecar 或直接嵌入在 Spring Boot 业务 Pod 中，通过 Nacos 注册中心发现 TC 地址，实现服务解耦。

### 1. 核心模块定义
- **TC (事务协调者)**：
  - 维护全局和分支事务的状态。
  - 负责全局事务 ID 的生成、注册、提交或回滚。
- **TM (事务发起者)**：
  - 定义全局事务的边界。
  - 负责告知 TC 开启、提交或回滚全局事务。
- **RM (资源管理者)**：
  - 管理分支事务处理的资源。
  - 向 TC 注册分支事务，汇报分支事务状态，并驱动分支提交或回滚。

### 2. 部署架构
- **Client**：TM 和 RM 作为 SDK 的一部分，与业务服务集成在一起。
- **Server**：TC 是独立部署的服务，通过服务注册与发现机制暴露给 Client。

### 3. 分布式事务执行流程
1. TM 向 TC 开启全局事务，注册全局事务记录。
2. RM 按业务场景执行本地事务，并向 TC 汇报资源准备状态（注册分支事务）。
3. TM 结束全局事务，通知 TC 提交或回滚。
4. TC 汇总所有分支事务信息，决定全局提交或回滚。

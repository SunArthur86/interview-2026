# Sleuth 或 SkyWalking 等全链路追踪系统中，Trace ID 是如何在多线程（如线程池）或异步调用（如 CompletableFuture）中传递的？

在 Java 中，全链路追踪上下文存储在 `ThreadLocal` 中，但线程切换会导致上下文丢失。解决方案是利用阿里巴巴开源的 `TransmittableThreadLocal` (TTL) 或框架提供的装饰器机制。TTL 会在任务提交前捕获主线程上下文，并在子线程执行时进行装饰。对于 `CompletableFuture`，SkyWalking 等插件会代理其回调方法，确保将 Trace ID 注入当前线程。RPC 调用时，Trace ID 通过 HTTP Header 或 RPC Metadata 透传给下游服务，从而实现跨线程、跨进程的链路串联。

## 记忆要点

- 根本痛点：原生ThreadLocal无法跨线程，子线程和线程池中极易丢失上下文。
- 核心方案：使用阿里TransmittableThreadLocal(TTL)，在任务提交前捕获并装饰到子线程。
- 异步处理：对于CompletableFuture，框架会代理其回调方法自动完成上下文注入。
- 跨进程透传：RPC或HTTP调用时，需将TraceId塞入请求Header或Metadata透传给下游。

## 结构化回答

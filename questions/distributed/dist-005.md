# LVS NAT模式的工作原理是什么？

### LVS NAT 模式工作原理
NAT（Network Address Translation）模式通过网络地址转换实现负载均衡。它是 LVS 最基础的模式，利用 Linux 内核的 Netfilter 框架修改 IP 报文头。

#### 工作原理与数据流
在 NAT 模式下，调度器作为所有服务器节点的网关，同时处理请求和响应流量的地址转换。

```text
    Client (CIP)           Director (VIP/DIP)        Real Server (RIP)
       |                        |                           |
       |---[Request: CIP->VIP]-->|                           |
       |                        |---[DNAT: CIP->RIP]------->|
       |                        |                           |
       |                        |<--[Response: RIP->DIP]----|
       |<--[SNAT: VIP->CIP]------|                           |
```

**详细流程**：
1.  **请求报文**：客户端发送请求，目标 IP 为 VIP（虚拟 IP），源 IP 为 CIP。
2.  **DNAT（目标地址转换）**：LB 收到报文后，根据负载均衡算法选择一台 Real Server。内核 IPVS 模块将报文的目标 IP（VIP）修改为选定的 Real Server 的 IP（RIP），目标端口也可能修改。此时源 IP 保持不变（CIP）。
3.  **路由转发**：Real Server 收到报文，发现目标是自己的 IP，处理请求。因为 Real Server 的默认网关指向 DIP（LB 的内网 IP），所以响应报文会发回给 LB。
4.  **SNAT（源地址转换）**：LB 收到响应报文后，将源 IP（RIP）修改为 VIP，然后发送给客户端。

#### 关键细节与原理
*   **端口映射**：由于使用了 DNAT，请求的目的端口可以与后端服务器的实际端口不同（例如：客户端访问 80 端口，LB 转发到后端 8080 端口）。
*   **连接跟踪**：LVS 依赖内核的 `ip_conntrack` 模块记录每一个连接的状态，确保响应包能正确地反向转换。
*   **边界条件**：Real Server 必须与 Director 在同一个物理网络（VLAN）内，且其网关必须指向 Director 的内网地址（DIP）。

#### 优缺点
*   **优点**：
    *   **网络兼容性**：Real Server 可以使用私有 IP，无需公网 IP，节省成本。
    *   **系统兼容性**：支持任何操作系统，无需修改内核。
*   **缺点**：
    *   **性能瓶颈**：所有请求和响应都必须经过 Director，流量加倍。在高并发下，Director 的带宽和处理能力（特别是 NAT 规则查找和 conntrack 内存消耗）容易成为瓶颈，通常适用于 10-20 节点规模。
    *   **扩展性受限**：难以支持大规模集群。

## 常见考点
1.  **为什么 NAT 模式扩展性差？**（答：进出流量都经过 LB，LB 成为瓶颈，且 conntrack 表大小有限）
2.  **NAT 模式下 Real Server 的网关指向哪里？**（答：必须指向 Director 的 DIP）
3.  **NAT 模式支持端口转换吗？**（答：支持，因为修改了 IP 报文头部的目的地址和端口）
4.  **相较于 DR 模式，NAT 模式最大的劣势是什么？**（答：性能瓶颈，DR 模式响应报文直接返回给客户端，不经过 LB）

**实战案例**：
在早期企业内网环境中，曾使用单台 LVS NAT 作为数百台内部测试服务器的入口。后来由于某台业务 RS 上传大文件，导致回包流量占满 LB 网卡带宽，阻塞了同 LB 下其他业务的正常访问，后续通过引入 DR 模式分流解决。

**代码示例（开启内核转发与配置 IPVS）**：
```bash
# 1. 开启 Director 核心转发（关键）
echo "1" > /proc/sys/net/ipv4/ip_forward

# 2. 配置 IPVS 规则
# 清空规则
ipvsadm -C
# 添加 VIP 服务，使用轮询算法
ipvsadm -A -t 192.168.0.100:80 -s rr
# 添加后端 RS，-m 表示 NAT 模式
# 注意：这里端口映射可以将 VIP:80 映射到 RIP:8080
ipvsadm -a -t 192.168.0.100:80 -r 10.0.0.1:8080 -m -w 1
ipvsadm -a -t 192.168.0.100:80 -r 10.0.0.2:8080 -m -w 1

# 3. 保存规则
service ipvsadm save
```

# 常见的限流算法有哪些？

1. **固定窗口计数器**：
- **原理**：将时间划分为固定窗口（如1分钟），在窗口内累加请求数，超过阈值拒绝。
- **问题**：**临界突变**。窗口0:59来了100个请求，1:01来了100个请求，虽然各自窗口未超，但在1:00这1秒内突发了200个请求，压垮系统。

2. **滑动窗口计数器**：
- **原理**：将大窗口（如1分钟）划分为多个小格子（如6个10秒），窗口随着时间滑动，丢弃最老的格子，加入新格子。统计当前窗口内的总和。
- **优点**：比固定窗口平滑，解决了边界突发问题。
- **实现**：Sentinel底层即采用此机制。

3. **漏桶算法**：
- **原理**：请求像水滴流入桶（队列），桶底以固定速率漏水（处理请求）。桶满则拒绝。
- **特点**：**强制恒定速率**流出。能够平滑突发流量。
- **缺点**：无法应对突发流量，即使系统空闲也只能恒定处理。
- **应用**：Nginx的 `limit_req` 模块（漏桶变种）。

4. **令牌桶算法（推荐）**：
- **原理**：系统以恒定速率向桶中放入令牌。请求到来时必须从桶中获取令牌（取走），无令牌则拒绝。
- **特点**：**限制平均速率，但允许突发**。如果桶容量大，积累了令牌，瞬间可以处理大量请求。
- **参数**：桶容量、生成令牌速率。
- **应用**：Guava RateLimiter（平滑突发）、Redis+Lua（分布式限流）。

**算法对比图**：
```text
固定窗口: [||||||] [||||||]  <- 边界突刺
滑动窗口:   [|||] [|||] [|||]  <- 平滑

漏桶:   请求 ---> (桶) ---> 恒定流出
               ^
               | (满了溢出)

令牌桶: 恒定放入 --> (桶) <-- 请求拿令牌
                        ^
                        | (无令牌拒绝)
```

**实战案例**：
在秒杀场景中，Nginx层使用漏桶算法将流量整形成固定速率（如每秒1000个），但应用层使用令牌桶算法允许瞬间突发消耗令牌，以最大化利用空闲时的缓存资源。

**代码示例（Redis Lua 令牌桶）**：
```lua
-- redis_key, limit_rate, capacity
local key = KEYS[1]
local rate = tonumber(ARGV[1])
local capacity = tonumber(ARGV[2])
local now = tonumber(ARGV[3])
local requested = 1

local info = redis.call('hmget', key, 'tokens', 'last_refill_time')
local tokens = tonumber(info[1]) or capacity
local last_refill = tonumber(info[2]) or now

-- 补充令牌
local delta = math.max(0, now - last_refill)
tokens = math.min(capacity, tokens + delta * rate)

-- 获取令牌
if tokens >= requested then
    tokens = tokens - requested
    redis.call('hmset', key, 'tokens', tokens, 'last_refill_time', now)
    return 1
else
    return 0
end
```

**对比总结**：
- **漏桶**：保护下游系统，严格限制速率，流量整形。
- **令牌桶**：保护自身系统，允许一定程度的流量突发，灵活性高。

**实际应用**：
- Nginx网关层：漏桶（流量整形）。
- 应用内部：滑动窗口/令牌桶（Sentinel/Resilience4j）。
- 分布式全局限流：Redis + Lua（实现令牌桶或滑动窗口）。

## 常见考点
1. **令牌桶的实现细节**：Guava RateLimiter支持“平滑突发”和“预热”模式，预热模式下令牌生成速率会缓慢上升，防止系统刚启动被大流量打死。
2. **分布式限流**：Redis Lua脚本解决原子性问题，性能依赖Redis单线程模型。超大规模场景可考虑Nginx+Lua网关限流。
3. **限流降级熔断的区别**：
   - 限流：目的保护系统，拒绝超出的流量。
   - 降级：目的保核心，牺牲非核心功能（如关闭评论）。
   - 熔断：目的防止雪崩，检测到下游不可用时自动断开调用。

# 微服务中API管理的作用和方案是什么？

### Hystrix 断路器机制
断路器很好理解，当 Hystrix Command 请求后端服务失败数量超过一定比例（默认50%），断路器会切换到开路状态。这时所有请求会直接失败而不会发送到后端服务。断路器保持在开路状态一段时间后（默认5秒），自动切换到半开路状态。这时会判断下一次请求的返回情况，如果请求成功，断路器切回闭路状态，否则重新切换到开路状态。

Hystrix 的断路器就像我们家庭电路中的保险丝，一旦后端服务不可用，断路器会直接切断请求链，避免发送大量无效请求影响系统吞吐量，并且断路器有自我检测并恢复的能力。

**断路器状态流转细节**：
*   **Closed（关闭）**：正常状态，请求通过。Hystrix 维护一个滚动时间窗（默认10秒），统计失败率。当失败率超过阈值（如50%）且请求数达到设定值（如20个）时，触发跳闸。
*   **Open（打开）**：熔断状态，所有请求直接降级返回，不发起实际调用。经过 `Sleep Window`（休眠窗，默认5秒）后，进入半开状态。
*   **Half-Open（半开）**：试探状态，允许放行一个请求。如果成功，认为服务恢复，切换回 Closed，重置计数器；如果失败，认为服务未恢复，切回 Open，并重新计时。

```text
    ┌─────────────┐
    │   Closed    │ ◄─── 失败率低 / 服务恢复
    │ (请求正常通过) │
    └──────┬──────┘
           │ 失败率超阈值 (如 >50% in 10s)
           ▼
    ┌─────────────┐
    │    Open     │ ───┐
    │ (直接快速失败) │   │ 经过 Sleep Window (5s)
    └──────┬──────┘   │
           │         ▼
           │    ┌─────────────┐
           └───►│  Half-Open  │
                │ (放行1个试探)│
                └──────┬──────┘
                       │
          ┌────────────┴────────────┐
          ▼                         ▼
    试探成功                     试探失败
    ───────►                       ───────►
```

### API 管理
在微服务架构中，API 管理的主要作用是统一管理服务的接口文档、接口定义、测试以及生命周期。常见的方案是使用 **Swagger**（现称 OpenAPI）。它允许生成自动化的 API 文档，并提供界面进行在线测试，解决了微服务接口数量庞大导致的管理混乱问题。

**API 网关与管理层的区别**：
面试中常容易混淆 API 网关（如 Spring Cloud Gateway, Kong）和 API 管理。
*   **API 网关**：运行时的流量入口，负责路由、鉴权、限流、熔断（技术实现层）。
*   **API 管理**：侧重于设计态和文档态，包括接口定义（OpenAPI Spec）、版本控制、Mock 服务、开发者门户等（协作与规范层）。

**实战案例**：在某跨国支付系统中，服务接口升级未及时同步导致下游调用失败。通过引入 Apigee 作为 API 管理平台，强制要求所有服务变更必须通过 API Manager 发布，并利用其 Portal 功能自动通知下游开发者，将接口不兼容导致的故障率降低了 90%。

**代码示例**：
```yaml
# Swagger (OpenAPI) 定义示例
swagger: '2.0'
info:
  version: 1.0.0
  title: Payment Service
paths:
  /pay:
    post:
      summary: Create payment
      responses:
        200:
          description: OK
```

**对比表格**：

| 维度 | API 网关 (Gateway) | API 管理 (Management) |
| :--- | :--- | :--- |
| **核心职责** | 流量控制、路由转发、安全加固 | 文档管理、生命周期、开发者协作 |
| **部署形态** | 在线服务，位于流量入口 | 通常为管理后台或 SaaS 平台 |
| **典型技术** | Spring Cloud Gateway, Kong, APISIX | Swagger, Apigee, Postman |
| **主要受众** | 运维、架构师 | 开发者、产品经理、第三方合作伙伴 |

# seata

Seata 是一款开源的分布式事务解决方案，致力于在微服务架构下提供高性能和简单易用的分布式事务服务。

### 实战案例
在库存服务与订单服务的分布式事务中，利用 Seata AT 模式，开发人员只需在业务方法上添加 `@GlobalTransactional` 注解，Seata 会在底层自动解析 SQL 生成回滚镜像，无需像 TCC 那样手写复杂的三个接口，大大降低了新人的上手门槛。

### 代码示例（Seata AT 模式使用）
```java
@GlobalTransactional(name = "create-order", rollbackFor = Exception.class)
public void createOrder(OrderDTO orderDTO) {
    // 1. 扣减库存（Seata 自动记录前镜像与后镜像）
    stockService.deduct(orderDTO.getProductId(), orderDTO.getCount());
    // 2. 创建订单
    orderService.insert(orderDTO);
    // 3. 若发生异常，Seata TC 自动协调回滚上述操作
}
```

### 核心特性
Seata 为用户提供了 **AT、TCC、SAGA 和 XA** 四种事务模式，打造一站式的分布式解决方案。

### 背景与生态
- 在开源之前，Seata 对应的内部版本在阿里经济体内部扮演着分布式一致性中间件的角色，历经历年双11的考验。
- 其商业化产品 GTS（Global Transaction Service）已在阿里云、金融云进行售卖。
- Seata 在 GitHub 拥有超过 1 万 Star，社区活跃。

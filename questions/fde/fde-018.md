# 如何快速搭建一个让客户能用的 AI 应用前端界面

- **AI 应用前端快速搭建方案(按速度排序)**

1. **Gradio**(10分钟):内部 demo / 简单交互
2. **Streamlit**(30分钟):PoC / 内部工具
3. **Next.js + Vercel**(1-3天):正式产品
4. **Vue + FastAPI**(2-5天):企业级应用

- **Streamlit 快速模板**
- 文件上传:st.file_uploader
- 问答界面:st.text_input + st.button
- 流式输出:st.write_stream()
- 反馈按钮:st.columns 布局点赞/点踩

- **客户体验加分项**
1. 流式输出(打字机效果)-> SSE 推送
2. 引用来源展示(点击可跳转原文)
3. 历史记录侧边栏
4. 暗色/亮色切换
5. 移动端适配

- **前后端交互架构 (SSE 流式)**
```
┌──────────┐        ┌──────────┐          ┌──────────┐
│ 浏览器端 │        │ 后端 API │          │ LLM 服务 │
│ (EventSource)───>│ (FastAPI)│─────────>│ (OpenAI) │
└──────────┘        └────┬─────┘          └──────────┘
                        │ Stream Chunks
                        │ (yield "token")
                        <────────
```

- **## 常见考点**
1. **流式传输实现原理**：后端如何保持长连接？通常使用 SSE (Server-Sent Events) 或 WebSocket。对于 LLM 场景，SSE 单向流更简单且 HTTP 兼容性好。
2. **Markdown 渲染安全**：前端展示 LLM 返回的 Markdown 时如何防范 XSS 攻击？必须使用 DOMPurify 等库进行清洗，且不要直接渲染 HTML 标签。
3. **Next.js 服务端组件**：在 Next.js 中如何减少客户端 JS 包体积？尽量使用 Server Components (RSC) 处理数据获取，仅将交互部分用 Client Components。

- **实战案例**：在给金融客户做 Demo 时，Streamlit 的默认样式过于“极客风”被嫌弃。后来使用 `st.set_page_config(layout="wide")` 并结合自定义 CSS 隐藏了“Made with Streamlit”页脚，加上客户的 Logo，立刻提升了交付可信度。

- **关键代码 (Python - FastAPI SSE 流式响应)**
```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from openai import OpenAI

app = FastAPI()
client = OpenAI()

@app.get("/chat")
async def chat_endpoint(query: str):
    def generate():
        stream = client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": query}],
            stream=True
        )
        for chunk in stream:
            if chunk.choices[0].delta.content:
                # SSE 格式要求 data: 前缀
                yield f"data: {chunk.choices[0].delta.content}\n\n"

    return StreamingResponse(generate(), media_type="text/event-stream")
```

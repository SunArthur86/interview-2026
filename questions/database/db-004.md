# 什么是主从复制的同步过程？

### 题目 3：什么是主从复制的同步过程？

### Redis 主从复制

**概念**：
主从复制是指将一个 Redis 节点（主节点，Master）的数据复制到其他节点（从节点，Slave）。主节点负责写操作，从节点负责读操作（读写分离），主节点自动将写命令同步给从节点。支持一主多从和级联复制（Slave -> Slave）。

**同步过程（分为三个阶段）**：

1.  **建立连接（握手与身份验证）**
    从节点发送同步命令（旧版 `SYNC`，新版 `PSYNC` <runid, offset>）请求连接主节点。从节点保存主节点信息。

2.  **数据同步（全量/部分）**
    *   **全量同步**：若是初次连接或缓冲区溢出，主节点执行 `BGSAVE` 生成 RDB 文件，同时将后续写命令记录到复制缓冲区。RDB 生成后发送给从节点。
    *   **部分同步**：如果复制积压缓冲区中有从节点需要的 offset 数据，则只发送增量部分，避免全量 RDB 开销。

3.  **命令传播**
    从节点加载 RDB 并解析缓冲区命令，将状态更新至与主节点一致。之后进入长连接模式，主节点每执行一个写命令，就通过心跳包发送给从节点。

**复制流程图**
```
[Slave] --(PSYNC runid offset)--> [Master]
         |                            |
         |<--(RDB File + Offset)-------|  (BGSAVE + Buffer Write)
         |                            |
[Slave] Load RDB                     |
         |                            |
         |<--(Buffered Commands)-------|  (增量补齐)
         |                            |
         |<--(Propagate Write Cmd)-----|  (持续同步)
```

**实战案例**
在网络抖动导致主从断连超过 `repl-backlog-size` 配置的时间后，Redis 会触发全量同步。对于大数据量（如 50GB+）的实例，主节点 fork 生成 RDB 会阻塞主线程处理请求，导致业务请求超时，且全量传输会占满带宽。优化手段是调大 `repl-backlog-size` 并优化网络质量。

**代码示例 (Redis CLI 配置)**
```bash
# 在 redis.conf 中关键配置
# 主节点
replica-serve-stale-data yes  # 从断连时是否响应旧数据
repl-backlog-size 100mb       # 实战：适当调大积压缓冲区，避免网络抖动触发全量重连

# 从节点
replicaof 192.168.1.100 6379
masterauth "yourpassword"     # 密码认证
```

**主从架构对比**
| 特性 | 主从复制 | 哨兵 模式 | 集群 模式 |
| :--- | :--- | :--- | :--- |
| **故障转移** | 手动（需人工介入） | 自动（监控 + 选举） | 自动（分片 + 迁移） |
| **数据写入** | 仅 Master | 仅 Master (自动切换) | 多 Master (分片写入) |
| **扩容能力** | 读水平扩展容易 | 读水平扩展容易 | 读写水平扩展 |
| **复杂度** | 低 | 中 | 高 |

## 常见考点
1.  **PSYNC 命令的 runid 和 offset 有什么作用？
    `runid` 标识主节点实例的唯一性，`offset` 标识复制进度。通过这两个参数，主节点可以判断从节点是否是之前连接过的，以及复制积压缓冲区中是否还有从节点缺失的数据。
2.  **什么是复制积压缓冲区？
    主节点维护的一个固定大小环形队列（默认 1MB）。用于存储最近发送的写命令。如果从节点断线重连，且 offset 仍在缓冲区范围内，主节点可以直接发送增量数据，否则必须进行全量重同步。
3.  **主从复制存在哪些问题？
    *   **复制延迟**：异步复制导致从节点数据可能滞后。
    *   **主机宕机数据丢失**：如果主节点宕机，尚未同步给从节点的数据将丢失（取决于配置 min-slaves-to-write）。
    *   **不具备自动故障转移**：需要配合 Sentinel（哨兵）系统实现自动监控和故障转移。

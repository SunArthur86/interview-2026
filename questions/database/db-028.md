# 什么是用户账号管理？

### 用户账号存储

在 MySQL 中，所有用户的账号、密码及全局权限存储在系统数据库 `mysql` 的 `user` 表中。
可以通过以下语句查看用户列表：
```sql
USE mysql;
SELECT user, host, authentication_string FROM user;
```
注意：密码经过加密存储（MySQL 8.0 使用 `caching_sha2_password`）。

### 用户账号管理

**原则**：
根据最小权限原则，只授予用户所需的权限，避免误操作和安全风险。

**1. 新增用户**
虽然可以直接操作 `user` 表 `INSERT` 记录，但出于安全考虑，**强烈建议使用标准 SQL**。
语法：
- MySQL 5.7: `CREATE USER 'username'@'host' IDENTIFIED BY 'password';`
- MySQL 8.0: 支持更复杂的密码验证插件，如 `CREATE USER 'username'@'host' IDENTIFIED WITH mysql_native_password BY 'password';`

例如：创建用户 `zhangsan`，密码 `123456`：
`CREATE USER 'zhangsan'@'localhost' IDENTIFIED BY '123456';`

**2. 修改用户名**
语法：
`RENAME USER 'old_name' TO 'new_name';`
例如：
`RENAME USER 'zhangsan'@'localhost' TO 'lisi'@'localhost';`

**3. 删除用户**
语法：
`DROP USER 'username'@'host';`

**4. 资源限制**
创建用户时可限制其资源使用，防止单个用户耗尽服务器资源：
```sql
CREATE USER 'u1'@'%' IDENTIFIED BY 'pwd'
    WITH MAX_QUERIES_PER_HOUR 100
         MAX_UPDATES_PER_HOUR 50;
```

### 常见考点
1. **Host 匹配规则**：`%` 匹配任意主机，但在某些场景下不匹配 `localhost`（因为 localhost 可能通过 Unix Socket 连接），需显式授权。
2. **密码加密方式变更**：MySQL 8.0 默认使用 `caching_sha2_password`，旧版 Navicat 可能不支持连接，需调整插件或客户端。
3. **匿名用户风险**：安装 MySQL 后默认存在 `''@'localhost'` 匿名用户，若未清理可能导致同名用户（如 `root`）匹配失败，建议删除。

### 实战深化

**实战案例**：
在迁移业务到 MySQL 8.0 时，曾遇到 Spring Boot 应用连接报错 `caching_sha2_password` 插件不支持。解决方法是在 JDBC URL 中显式指定 `allowPublicKeyRetrieval=true` 并升级驱动，或者在服务器端将用户修改回 `mysql_native_password` 插件以兼容旧版中间件。

**代码示例**：
```sql
-- 兼容性修改：将用户的加密插件降级为旧版（应急方案）
ALTER USER 'app_user'@'%' IDENTIFIED WITH mysql_native_password BY 'new_password';
FLUSH PRIVILEGES;
```

**方案对比**：

| 特性 | CREATE USER 语句 | 直接 INSERT mysql.user |
| :--- | :--- | :--- |
| **安全性** | 高（自动处理密码哈希、刷新） | 低（需手动处理加密、权限） |
| **内存/缓存** | 自动更新权限缓存 | 需手动执行 FLUSH PRIVILEGES |
| **版本兼容性** | 适配不同版本语法 | 字段结构随版本变化易出错 |
| **维护性** | 易读、标准操作 | 难以审计、容易破坏数据一致性 |

## 记忆要点

- 存储与底层：账号密码权限存于 mysql.user 表，强烈建议用标准 SQL 操作而非直接改表
- Host 匹配机制：账号由 'username'@'host' 组成，% 代表任意 IP，但本地 Socket 需 localhost
- 版本兼容大坑：MySQL 8.0 默认 caching_sha2_password 加密，旧客户端常报错需降级改插件
- 安全与资源限制：遵循最小权限原则，可通过 WITH MAX_QUERIES_PER_HOUR 限制资源

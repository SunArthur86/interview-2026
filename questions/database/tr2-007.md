# ES (Elasticsearch) 的倒排索引中，FST (Finite State Transducer) 是如何将 Term Dictionary 也就是词典存储在内存中的？相比传统的 B+ 树或 Hash 结构，它有什么优势？

ES 中的 FST 是一种类似 Trie 树但更节省内存的前缀压缩结构。它将 Term Dictionary 中的词汇映射到 Postings List（倒排表）的磁盘地址。FST 的优势在于极高的压缩率和前缀共享能力：它不仅压缩了公共前缀，还压缩了后缀。相比于 B+ 树，FST 在查询时不仅减少磁盘 I/O（常驻内存），而且查找时间复杂度仅与词长相关（通常是 O(k)），通常比 B+ 树的 O(log N) 更快。相比于 Hash 结构，FST 支持前缀模糊查询和范围查询。通过 FST，ES 可以将庞大的词典加载进内存，快速定位到磁盘上的倒排表位置，这是 ES 实现秒级全文检索的关键技术。

## 记忆要点

- FST是类似Trie树的前缀压缩结构，不仅压缩前缀也复用后缀
- FST将Term Dictionary常驻内存，映射到磁盘倒排表地址
- 对比B+树：FST内存压缩率极高，且查找复杂度O(k)比O(logN)更快
- 对比Hash：Hash不支持前缀查询，而FST支持前缀和范围查询

## 结构化回答

**30 秒电梯演讲：** FST利用前缀后缀共享将海量词项压缩进内存，实现O(k)快速寻址。打个比方，就像给几亿个词条建地铁线路图，同前缀的词共用一段轨道，不仅省地儿，还能沿着轨道一站站（字符）快速找到终点。

**展开框架：**
1. **FST是类似Trie树的前缀压缩结构** — 不仅压缩前缀也复用后缀
2. **FST将Term Dictionary常驻内存** — 映射到磁盘倒排表地址
3. **对比B+树** — FST内存压缩率极高，且查找复杂度O(k)比O(logN)更快

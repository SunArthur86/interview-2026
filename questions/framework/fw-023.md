# MVC流程是什么？

### Spring MVC 执行流程

Spring MVC 的模型-视图-控制器（MVC）框架是围绕一个 `DispatcherServlet` 来设计的，这个 Servlet 会把请求分发给各个处理器，并支持可配置的处理器映射、视图渲染、本地化、时区与主题渲染等，甚至还能支持文件上传。

**详细流程如下：**

1.  **用户请求**：用户发送请求到服务器。
2.  **DispatcherServlet**：前端控制器接收请求，它是整个流程的入口。
3.  **HandlerMapping**：处理器映射器根据 URL 查找对应的 Handler（Controller）。
4.  **HandlerAdapter**：处理器适配器执行 Handler，并返回 `ModelAndView`。
5.  **ViewResolver**：视图解析器根据逻辑视图名解析成具体的 View 对象。
6.  **View Rendering**：视图渲染，将模型数据填充到视图中。
7.  **响应**：将结果返回给用户。

**实战案例**：
在实际项目中，常利用 `HandlerInterceptor` 的 `preHandle` 进行登录鉴权和权限校验。一个典型的坑是如果在拦截器中手动处理了响应（如 JSON 异常返回），必须返回 `false`，否则流程会继续执行 Controller，导致“重复响应”错误或不必要的数据库查询消耗。

**代码示例**：
```java
@Component
public class AuthInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        // 鉴权逻辑
        if (request.getHeader("token") == null) {
            response.setStatus(401);
            response.getWriter().write("Unauthorized");
            return false; // 关键：返回false中断流程
        }
        return true;
    }
}
```

**补充关键细节：**
*   **HandlerExecutionChain**：HandlerMapping 返回的不仅是一个 Handler（Controller 方法），而是一个执行链，包含了 Handler 本身和一系列拦截器。

## 记忆要点

- 核心组件：一Servlet(DispatcherServlet)、二器(HandlerMapping与HandlerAdapter)、一解析(ViewResolver)
- 核心流程：用户请求到达DispatcherServlet，查Mapping找Controller，用Adapter执行业务
- 执行结果：Controller返回ModelAndView，交由ViewResolver解析视图并渲染数据
- 拦截器考点：preHandle返回false则中断后续Controller执行
- 关键区分：HandlerMapping负责找Controller，HandlerAdapter负责执行Controller

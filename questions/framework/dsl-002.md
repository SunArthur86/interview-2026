# Spring

### Spring Framework 概述

**1. Spring 与 SpringMVC、Spring Boot 的区别**

*   **Spring**：是一个全面的 Java 企业级应用程序开发框架，提供了广泛的功能，包括依赖注入（IOC）、面向切面编程（AOP）、事务管理等。它是整个生态的基础。
*   **Spring MVC**：是 Spring 框架中的一个核心模块，主要用于构建 Web 应用。它基于 MVC 设计模式，将业务逻辑、数据、显示分离，专注于处理 Web 请求、管理用户会话和控制应用程序流程。
*   **Spring Boot**：基于 Spring 框架，旨在简化和加速 Spring 应用的初始搭建以及开发过程。它采用“约定大于配置”的理念，通过自动配置和起步依赖，消除了繁琐的 XML 配置，使开发者能专注于业务逻辑，适合快速构建微服务。

| 维度 | Spring Framework | Spring MVC | Spring Boot |
| :--- | :--- | :--- | :--- |
| **核心定位** | 基础设施与 IOC/AOP 容器 | Web 层 MVC 框架 (模块) | 快速开发整合框架 (约定大于配置) |
| **配置方式** | XML / Java Config (繁琐) | XML / Java Config | 自动配置 (零/极简 XML) |
| **启动方式** | 需构建 Web 容器 (如 Tomcat) | 依赖 Servlet 容器 | 内嵌容器，`main` 方法直接运行 |
| **核心价值** | 解耦、企业级服务支持 | 分层架构、请求处理 | 开发效率、运维便利、微服务基石 |
| **典型场景** | 构建复杂的企业级底座 | 开发单体 Web 应用接口 | 云原生应用、微服务、快速原型 |

**2. Spring 框架体系结构**

Spring 框架至今已集成了 20 多个模块，主要分为以下几个部分：

**2.1 核心容器**
这是其他模块建立的基础，由以下模块组成：
*   **spring-core**：提供了框架的基本组成部分，包括 IOC 和 DI 功能。
*   **spring-beans**：提供了 BeanFactory，是工厂模式的经典实现，Spring 将管理的对象称为 Bean。
*   **spring-context**：建立在 Core 和 Beans 之上，提供了一种类似于 JNDI 注册器的框架对象访问方式（ApplicationContext 接口是核心）。
*   **spring-context-support**：支持整合第三方库到 Spring 应用上下文，如缓存和任务调度。
*   **spring-expression**：提供了强大的表达式语言用于在运行时查询和操作对象图。

**2.2 AOP 和 Instrumentation**
*   **spring-aop**：提供了面向切面编程的实现，允许定义方法拦截器和切入点，将业务逻辑与系统服务解耦。
*   **spring-aspects**：提供了与 AspectJ 的集成。
*   **spring-instrument**：提供了类植入支持和类加载器的实现。

**2.3 消息**
*   **spring-messaging**：从 Spring 4.0 开始新增，提供了对消息传递体系结构和协议的支持。

**2.4 数据访问/集成**
由 JDBC、ORM、OXM、JMS 和事务模块组成：
*   **spring-jdbc**：提供了 JDBC 的抽象层，消除了繁琐的 JDBC 编码。
*   **spring-orm**：为 JPA、Hibernate 等 ORM 框架提供集成层。
*   **spring-oxm**：提供了一个支持对象/XML 映射的抽象层。
*   **spring-jms**：用于生产和消费消息。
*   **spring-tx**：支持编程式和声明式事务管理。

**2.5 Web 层**
*   **spring-web**：提供了基本的 Web 开发集成功能，如文件上传、IOC 容器初始化。
*   **spring-webmvc**：即 Spring MVC，包含用于 Web 应用程序的 Spring MVC 和 REST Web Services 实现。
*   **spring-websocket**：提供了 WebSocket 和 SockJS 的支持。

**2.6 测试**
*   **spring-test**：支持使用 JUnit 或 TestNG 对 Spring 组件进行单元测试和集成测试。

**实战案例**
在旧项目中维护复杂的 Spring XML 配置常导致“配置地狱”。例如，曾遇到一个项目因 XML 缺少 `default-autowire` 属性，导致新增的 Service 注入失败，排查数小时。切换到 Spring Boot 后，利用其自动配置机制，不仅解决了版本冲突（如 Jackson 版本不匹配），还将项目启动时间从 45s 降低至 12s。

**代码示例 (Java Config 替代 XML)**
```java
@Configuration
@ComponentScan("com.example.service")
public class AppConfig {
    // 定义 DataSource Bean，替代 XML <bean id="dataSource">
    @Bean
    public DataSource dataSource() {
        return new HikariDataSource(); // 实战常用连接池
    }
}
```

## 记忆要点

- 核心定位：Spring是基础底座（提供IOC/AOP）；MVC是Web层模块；Boot是极速整合框架（约定大于配置）
- 配置演进：因为Spring繁琐的XML常引发配置地狱，所以Boot通过自动配置和Starter依赖实现了开箱即用
- 生态关系：Spring MVC和Boot都不是独立框架，它们最终都是构建在Spring Framework核心容器之上的扩展

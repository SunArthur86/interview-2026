# Mybatis是如何将sql执行结果封装为目标对象并返回的?都有哪些映射形式?

### MyBatis 结果映射

**1. 结果封装机制**

MyBatis 在执行 SQL 并获得 ResultSet 后，会将结果集封装为目标对象。主要通过 `ResultSetHandler` 接口实现，包含以下步骤：
1.  **映射查找**：根据 `MappedStatement` 中定义的 `resultMap` 或 `resultType`。
2.  **数据转换**：遍历 ResultSet，通过 `TypeHandler` 将 JDBC 类型转换为 Java 类型。
3.  **对象创建与填充**：利用反射创建目标对象实例，并将转换后的值填充到对象的属性中。

**4. 执行流程架构图**：

         │
         ▼

└────────┬──────────────────────────┘

         │

**2. 映射形式**

主要有两种方式将 SQL 列名映射到 Java 对象属性名：

*   **方式一：使用 <resultMap> 标签（推荐）**
    *   **适用场景**：数据库列名与 Java 属性名不一致，或者包含复杂的映射（如关联对象、关联集合）。
    *   **原理**：在 XML 中显式定义列名与属性的对应关系。
    *   **关键元素**：
        *   `<id>`：标记主键列，提升性能，用于区分对象实例。
        *   `<result>`：标记普通列映射。
        *   `<association>`：处理“一对一”关联（嵌套 JavaBean）。
        *   `<collection>`：处理“一对多”关联（嵌套集合）。

*   **方式二：自动映射 (Auto-mapping)**
    *   **适用场景**：列名与属性名完全一致（或符合数据库命名转驼峰规则）。
    *   **原理**：MyBatis 自动将列名转换为属性名进行赋值。
    *   **配置**：`mapUnderscoreToCamelCase=true`（开启下划线转驼峰）。

#### 实战案例
在处理遗留系统时，数据库字段全是下划线命名（如 `user_name`），而 Java 实体是驼峰命名（`userName`）。若不开启 `mapUnderscoreToCamelCase`，所有字段都需在 `<resultMap>` 手动映射。开启后只需定义特殊字段的映射，极大减少了 XML 配置量。但要注意，若 SQL 中使用了别名（如 `select user_name as name`），自动映射会按 `name` 去匹配属性而非 `userName`，导致映射失败。

#### 关键代码示例
```xml
<!-- 复杂映射：一对一 + 一对多 -->
<resultMap id="OrderMap" type="Order">
    <id property="id" column="order_id"/>
    <result property="orderNo" column="order_no"/>
    <!-- 一对一：association -->
    <association property="user" javaType="User">
        <id property="id" column="user_id"/>
        <result property="username" column="username"/>
    </association>
    <!-- 一对多：collection -->
    <collection property="items" ofType="Item">
        <id property="id" column="item_id"/>
        <result property="productName" column="product_name"/>
    </collection>
</resultMap>
```

**## 常见考点**
1.  **N+1 问题**：在使用 `<collection>` 或 `<association>` 且未配置 `select` 属性进行延迟加载或联合查询（`join`）时，可能会触发 N+1 次查询问题。解决办法是使用 `fetchType="lazy"` 或在 SQL 中直接 `join`。
2.  **延迟加载**：如何配置开启延迟加载（`lazyLoadingEnabled=true`），以及其原理（使用动态代理代理对象，只有在使用属性时才触发查询）。
3.  **TypeHandler**：当数据库类型（如 JSON 字符串）与 Java 类型（如 Object/List）不匹配时，如何自定义 TypeHandler 进行转换。

## 记忆要点

- 核心处理组件：因为由ResultSetHandler负责，所以能遍历结果集利用反射创建对象并填充属性。
- 类型转换器：因为依赖TypeHandler，所以能实现JDBC类型与Java类型的精准转换。
- 映射两极：自动映射开启驼峰转换，而复杂对象关联需通过<resultMap>定义。

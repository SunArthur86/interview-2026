# OpenFeign的工作原理是什么？

【核心原理】
OpenFeign 是一个声明式的 Web Service 客户端，它使得编写 Web Service 客户端变得更加简单。使用时，只需创建一个接口并在上面添加注解（如 `@FeignClient`），Spring Cloud 便会为其创建动态代理实现。

【实战案例】
在一次跨部门接口联调中，发现 OpenFeign 默认使用 `HttpURLConnection`，没有连接池且不支持 Keep-Alive，导致高并发下建立 TCP 连接开销过大，延迟飙升。我们在 `pom.xml` 中引入 `feign-httpclient` (Apache HttpClient) 并配置连接池参数后，接口 QPS 提升了约 40%，P99 延迟降低了 60ms。

【关键代码片段 (Feign 拦截器实现自定义 Header)】
```java
// 配置 Feign 使用 HttpClient 连接池
@Bean
public Client feignClient() {
    return new ApacheHttpClient();
}

// 实现 RequestInterceptor 在调用前透传 Token (如认证信息)
@Component
public class AuthRequestInterceptor implements RequestInterceptor {
    @Override
    public void apply(RequestTemplate template) {
        ServletRequestAttributes attributes = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        if (attributes != null) {
            HttpServletRequest request = attributes.getRequest();
            String token = request.getHeader("Authorization");
            if (token != null) {
                template.header("Authorization", token);
            }
        }
    }
}
```

## 记忆要点

- 核心机制：OpenFeign本质是声明式HTTP客户端，通过动态代理将接口方法转为HTTP请求。
- 负载均衡：集成LoadBalancer，根据serviceId从注册中心获取实例并路由请求。
- 请求解析：MethodHandler解析注解生成Request模板，交由底层Client发送。
- 性能优化：默认无连接池(JDK原生)，生产环境强烈建议替换为HttpClient或OkHttp。

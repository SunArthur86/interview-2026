# 说说你知道的关于BeanFactory和FactoryBean的区别？

### BeanFactory vs FactoryBean

这两者是 Spring 中非常容易混淆的概念，虽然名字很像，但职责完全不同。

#### 1. BeanFactory
BeanFactory 是 Spring 容器的**顶层接口**，也就是 Spring 的 IoC 容器。它负责实例化、定位、配置应用程序中的对象及建立这些对象间的依赖关系。

*   **本质**：它是一个工厂，用来生产和管理 Bean 的实例。
*   **职责**：负责 Bean 的生命周期管理（实例化、属性填充、初始化、销毁）。
*   **常见实现**：`DefaultListableBeanFactory`，`XmlBeanFactory`（已废弃），`ApplicationContext` (它是 BeanFactory 的扩展子接口)。
*   **获取 Bean**：调用 `getBean(String beanName)` 方法。
*   **特点**：原始的 BeanFactory 采用**延迟加载**（Lazy-load）策略，只有在 getBean 时才实例化 Bean。而 ApplicationContext 通常在启动时预实例化单例 Bean。

#### 2. FactoryBean
FactoryBean 是一个**接口**，它也是一种 Bean，但它是一种**特殊的工厂 Bean**。当一个类实现了 FactoryBean 接口后，它不再是一个普通的业务对象，而是一个用于生产或修饰其他对象的工厂。

*   **本质**：它是一个 Bean，是 Spring 容器管理的一个实例，但它用来创建另一个对象。
*   **核心方法**：
    *   `T getObject()`: 返回由 FactoryBean 创建的对象实例（这才是最终给开发者用的对象）。
    *   `Class<?> getObjectType()`: 返回 getObject() 方法返回的对象的类型。
    *   `boolean isSingleton()`: 返回创建的对象是否是单例。
*   **使用场景**：当创建一个 Bean 的逻辑非常复杂（如需要复杂的初始化代码、依赖外部资源、或需要代理对象）时，可以将这些逻辑封装在 FactoryBean 中。例如：`SqlSessionFactoryBean` (MyBatis), `ProxyFactoryBean` (AOP)。
*   **获取方式**：
    *   `context.getBean("myFactoryBean")`：获取的是它**生产的对象**（即 getObject() 的结果）。
    *   `context.getBean("&myFactoryBean")`：在 beanName 前加 `&` 符号，获取的是 **FactoryBean 本身的实例**。

#### 3. 两者的区别总结
| 特性 | BeanFactory | FactoryBean |
| :--- | :--- | :--- |
| **是什么** | IoC 容器，Spring 的基础设施 | 一个 Bean，生产其他对象的工厂 |
| **谁管理它** | 它自己管理别人 | 被 BeanFactory 管理 |
| **接口定义** | `org.springframework.beans.factory.BeanFactory` | `org.springframework.beans.factory.FactoryBean` |
| **作用** | 负责 Bean 的生命周期、依赖注入 | 负责创建、初始化特定类型的复杂 Bean |
| **获取对象** | `getBean(name)` 得到目标 Bean | `getBean(name)` 得到它生产的产品，`getBean("&name")` 得到它自己 |

**#### 实战案例**
在集成第三方客户端（如 Aliyun OSS）时，客户端构建过程需要十几步参数设置。我们定义了一个 `OssClientFactoryBean` 实现 `FactoryBean`，将复杂的构建逻辑封装在 `getObject()` 中，业务代码只需注入 `OssClient` 即可，保持代码整洁。

**#### 代码示例**
```java
@Component("customDataSource")
public class CustomDataSourceFactoryBean implements FactoryBean<DataSource> {

    @Override
    public DataSource getObject() throws Exception {
        // 这里可以编写复杂的连接池创建逻辑
        return new HikariDataSource(); 
    }

    @Override
    public Class<?> getObjectType() {
        return DataSource.class;
    }
}
// 使用时：getBean("customDataSource") 得到的是 DataSource 实例，而不是 FactoryBean 实例
```

## 记忆要点

- 定位不同：BeanFactory是IoC顶层容器接口，而FactoryBean是创建复杂Bean的特殊工厂接口。
- 核心方法：FactoryBean靠getObject()生产目标对象，getObjectType()声明类型。
- 获取差异：getBean(name)获取FactoryBean生产的产品；而getBean(&name)加&获取工厂本身。
- 使用场景：集成第三方复杂客户端（如MyBatis的SqlSessionFactoryBean）时，封装复杂创建逻辑。

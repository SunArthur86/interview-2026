# Spring中用到了哪些设计模式是什么？

### Spring中用到了哪些设计模式是什么？

1.  **工厂设计模式**
    *   **使用场景**：BeanFactory 和 ApplicationContext。
    *   **原理细节**：Spring 容器通过工厂模式来创建和管理 Bean 对象。`BeanFactory` 是基础工厂，而 `ApplicationContext` 是其高级实现，增加了事件发布、国际化支持等功能。`FactoryBean` 也是一个特殊的工厂，用于创建复杂的 Bean。

2.  **代理设计模式**
    *   **使用场景**：AOP 的核心实现。
    *   **原理细节**：
        *   **JDK 动态代理**：基于接口，利用反射机制生成代理类。
        *   **CGLIB 动态代理**：基于继承，如果是类代理，Spring 默认使用 CGLIB（Spring Boot 2.x 后默认倾向 CGLIB），通过字节码技术生成子类。
    *   **流程图**：
        ```text
        ┌──────────┐       ┌──────────────┐       ┌──────────┐
        │ Client   │──────▶│ Proxy (AOP)  │──────▶│ Target   │
        │ (Caller) │       │ (增强逻辑)   │       │ (Source) │
        └──────────┘       └──────────────┘       └──────────┘
                                 │
                                 ▼
                         ┌──────────────┐
                         │ Advice/Aspect│
                         │ (横切逻辑)    │
                         └──────────────┘
        ```

3.  **单例设计模式**
    *   **使用场景**：Spring Bean 默认作用域。
    *   **原理细节**：Spring 通过 `SingletonBeanRegistry` 接口管理单例 Bean，使用 `ConcurrentHashMap` 缓存单例实例，保证线程安全。

4.  **模板方法模式**
    *   **使用场景**：JdbcTemplate、RestTemplate、RedisTemplate 等。
    *   **原理细节**：定义算法骨架（如获取连接、处理异常、释放资源），将具体业务逻辑（如 SQL 执行）延迟到子类或回调接口中实现。

5.  **包装器设计模式**
    *   **使用场景**：Spring 中 Bean 的多重装饰，或者动态数据源的切换。
    *   **原理细节**：例如 DataSource 包装，可以根据上下文动态切换实际的数据源实现。

6.  **观察者模式**
    *   **使用场景**：ApplicationListener 和 ApplicationEvent。
    *   **细节**：实现了 `ApplicationListener` 的 Bean 可以监听容器发布的 `ApplicationEvent` 事件，实现业务解耦。

7.  **策略模式**
    *   **使用场景**：Resource 的加载（ClassPathResource, FileSystemResource）。
    *   **细节**：根据不同的资源路径前缀，使用不同的资源加载策略。

**补充：Bean的作用域配置**
Spring Bean 默认是单例，在配置文件中可以指定 scope：
- `singleton`：单例模式，默认值，容器中只存在一个实例。
- `prototype`：多例模式，每次请求都创建一个新的实例。
- `request`：每次 HTTP 请求创建一个实例（Web 环境）。
- `session`：同一个 HTTP Session 共享一个实例（Web 环境）。

**常见考点**

1.  **Spring 中单例 Bean 是线程安全的吗？**
    *   答：Spring 中的单例 Bean 本身并不是线程安全的。如果 Bean 中有可变的成员变量，需要开发者自行处理线程安全（如使用 ThreadLocal 或 synchronized）。

**实战案例**：
在实现多租户系统时，常利用 **ThreadLocal** 配合 **AbstractRoutingDataSource**（策略模式变种）来动态切换数据源。每个请求线程在拦截器中设置租户ID，DataSource 根据此ID路由到对应的物理库，而业务代码无感知。

**代码示例（观察者模式实现业务解耦）**：
```java
// 1. 定义事件
public class OrderCreatedEvent extends ApplicationEvent {
    private String orderId;
    public OrderCreatedEvent(Object source, String orderId) {
        super(source);
        this.orderId = orderId;
    }
}

// 2. 监听器
@Component
public class NotificationListener implements ApplicationListener<OrderCreatedEvent> {
    @Override
    public void onApplicationEvent(OrderCreatedEvent event) {
        System.out.println("发送短信: 订单 " + event.getOrderId() + " 创建成功");
    }
}
```

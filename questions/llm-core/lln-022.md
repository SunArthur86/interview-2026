# 手撕在MHA中加入KV Cache

在MHA（Multi-Head Attention）推理阶段，预分配K、V缓存张量，推理时将当前步的K、V写入缓存，并从缓存中读取历史K、V进行注意力计算，避免重复计算历史 Token 的投影。

### 核心原理与代码实现

自回归推理生成 $t$ 时刻的 token 时，query 仅来自当前 token，但 key 和 value 需要包含 $0$ 到 $t-1$ 的所有历史信息。KV Cache 就是将之前计算好的 K 和 V 矩阵存起来，下次只需计算当前的 K、V 并拼接即可。

```python
import torch
import torch.nn as nn

class MultiHeadAttentionWithCache(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads
        
        self.wq = nn.Linear(d_model, d_model, bias=False)
        self.wk = nn.Linear(d_model, d_model, bias=False)
        self.wv = nn.Linear(d_model, d_model, bias=False)
        self.wo = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x, past_kv=None, use_cache=False):
        # x shape: [batch_size, seq_len, d_model] (seq_len=1 for inference)
        batch_size, seq_len, _ = x.shape

        # 1. Linear Projection
        Q = self.wq(x)  # [bs, 1, d_model]
        K = self.wk(x)  # [bs, 1, d_model]
        V = self.wv(x)  # [bs, 1, d_model]

        # 2. Reshape for Multi-Head
        Q = Q.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2) # [bs, n_heads, 1, d_k]
        K = K.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2) # [bs, n_heads, 1, d_k]
        V = V.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2) # [bs, n_heads, 1, d_k]

        # 3. Handle Past KV Cache
        if past_kv is not None:
            # past_kv: tuple (past_k, past_v)
            # shape: [bs, n_heads, past_seq_len, d_k]
            past_k, past_v = past_kv
            # 拼接历史的 K, V 和当前的 K, V
            K = torch.cat([past_k, K], dim=2) # [bs, n_heads, past_len+1, d_k]
            V = torch.cat([past_v, V], dim=2) # [bs, n_heads, past_len+1, d_k]
        
        # 4. Prepare return cache
        present_kv = (K, V) if use_cache else None

        # 5. Scaled Dot-Product Attention
        # Score: [bs, n_heads, 1, d_k] @ [bs, n_heads, d_k, past_len+1]
        scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(self.d_k, dtype=torch.float32))
        
        # 6. Causal Mask (推理时通常仅在输入padding时需要，自回归生成时右上角为-Inf)
        # 实际推理中为了加速，常使用FlashAttention，此处展示标准逻辑
        # attn_weights = torch.softmax(scores, dim=-1)
        # output = torch.matmul(attn_weights, V)
        
        # 简化返回，仅展示KV Cache核心逻辑
        # output = output.transpose(1, 2).contiguous().view(batch_size, seq_len, self.d_model)
        # return self.wo(output), present_kv
        return None, present_kv # 返回供下一轮使用的KV
```

### 实战案例
*   **PagedAttention**：在vLLM等高性能推理框架中，KV Cache的显存管理是个大问题。传统的KV Cache是连续分配的，容易导致显存碎片。vLLM引入了PagedAttention，将KV Cache分块存储，类似操作系统的虚拟内存，允许在不连续的显存块上存储KV序列，极大提高了Batch Size和显存利用率。

### 对比表格

| 维度 | 无 Cache (Standard) | 有 Cache (KV Cache) |
| :--- | :--- | :--- |
| **计算复杂度** | $O(T^2)$ (每步重算历史) | $O(T)$ (每步仅算当前) |
| **显存占用** | 仅需激活值 | 需额外存储 [Batch, Heads, Seq, Dim] |
| **首字延迟** | 较低 (无需预分配) | 较高 (需初始化Cache) |
| **后续延迟** | 线性增长 | 恒定 (理想情况下) |
| **应用场景** | 训练阶段 / 非自回归推理 | Transformer 自回归推理 |

## 延伸：手撕MHA+KV Cache

> 合并自 `lln-004`（相似度 76%）

- **手撕 MHA + KV Cache**:
  - **核心逻辑**:
    1. 初始化全零 `cache_k`, `cache_v` (最大长度)。
    2. Forward中计算当前步的 `xk`, `xv`。
    3. 更新缓存: `cache_k[:, start_pos:...] = xk`。
    4. 拼接历史Key/Value进行Attention计算。
  - *数据流图*:
    ```text
    输入 x_t (当前 token)
       │
       ├──────────────────┐
       ▼                  ▼
    [Wq, Wk, Wv] 投影   RoPE 位置编码
       │                  │
       ▼                  ▼
    q_t (当前)          k_t, v_t (当前)
       │                  │
       │                  ▼
       │           ┌───────────────┐
       │           │ KV Cache 更新 │
       │           │ [k_hist | k_t] │
       │           │ [v_hist | v_t] │
       │           └───────┬───────┘
       │                   │
       └───────┬───────────┘
               ▼
         Attention(Q, K_cache, V_cache)
               │
               ▼
           Output y_t
    ```
  - **实战案例**: 在PagedAttention实现中，KV Cache往往不连续，需通过`block_tables`索引物理显存块，否则无法处理变长序列导致显存浪费。
  - **代码**:
```python

        self.cache_v = torch.zeros((args.max_batch_size, args.max_seq_len, self.n_heads, self.head_dim)).cuda()

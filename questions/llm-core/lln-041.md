# 大模型出现幻觉的原因

大模型出现幻觉的原因是多方面的，核心在于自回归生成机制和数据缺陷。

**主要原因分析**：

1. **自回归生成的累积误差（核心机制原因）**：
   - 模型采用逐 Token 生成，一旦前序生成了错误的 Token，后续为了保持上下文连贯性，模型会基于错误的上下文继续编造，导致错误像滚雪球一样积累且无法回溯。

2. **数据源的知识缺陷（数据原因）**：
   - 训练数据中存在错误、偏差或过时的信息。模型“记住”了这些错误知识，并在推理时复现。
   - 对于训练数据中极少出现的“长尾知识”，模型并未真正学好，仅仅是模糊的记忆，导致生成时一本正经地胡说八道。

3. **概率生成的本质（模型原因）**：
   - LLM 本质上是概率分布模型 $P(w_t | w_{<t})$，它生成的是“最可能”的下一个词，而不是“事实正确”的词。当多个候选词概率相近时，可能选中逻辑通顺但事实错误的词。

4. **对齐税与 Sycophancy（阿谀奉承）**：
   - 经过 RLHF 等对齐训练后，模型倾向于生成让用户满意的回答，有时会顺从用户的错误引导（如用户故意问错误事实时，模型可能会附和），而非坚持事实真相。

**实战案例**：
在医疗问答系统中，模型常会编造不存在的药物名称。经排查发现，这是由于 RLHF 阶段偏好数据过于强调回答的“完整性”而非“准确性”，导致模型倾向于生成虚假细节来凑字数。引入基于知识图谱的约束后显著改善。

**代码示例 (检测与规避)**：
```python
# 使用 LogProb 检测低置信度生成（可能的幻觉）
outputs = model.generate(input_ids, max_new_tokens=100, output_scores=True, return_dict_in_generate=True)

transition_scores = model.compute_transition_scores(outputs.sequences, outputs.scores, normalize_logits=True)
avg_prob = torch.exp(transition_scores).mean()

if avg_prob < 0.3:
    print("Alert: High hallucination risk (Low confidence)")
```

**常见考点**
1. 如何缓解大模型幻觉？（提示工程增强、检索增强生成 RAG、知识图谱约束、思维链 CoT 提高推理准确性）
2. 什么是“知识边界”问题？（模型对未知问题的回答态度，是拒绝回答还是胡编乱造，这通常通过训练时的 refusal data 来调节）

## 技术原理

- **自回归的"雪崩效应"（核心机制）**：LLM 生成是 $P(w_t | w_{<t})$ 的链式分解——每个 token 的生成都以**前面所有 token 为条件**。一旦第 $k$ 个 token 错了，$w_{k+1}$ 基于错误的 $w_k$ 生成，$w_{k+2}$ 基于 $w_{k+1}w_k$ 生成……错误像滚雪球累积。模型没有"回头修改"的机制（生成不可逆），只能"硬着头皮编下去"以保持表面连贯。这是 Decoder-only 架构的固有局限。
- **概率分布 vs 事实正确**：LLM 学到的是"统计上最可能的下一个词"，不是"事实上正确的下一个词"。当训练数据中某个错误事实出现频率高（如维基百科的偏见条目），模型会概率化地复现这个错误。对于训练数据中**罕见的长尾知识**，模型只有模糊表示，会用"看起来合理"的词填充，产生"一本正经胡说八道"。
- **知识存储的"参数化记忆"局限**：知识被压缩在参数里（类似 lossy compression）。模型无法区分"我学过这个知识" vs "我在编"。它对所有输出都给出一个置信度，但这个置信度反映的是"概率高低"而非"事实确信"。当 prompt 涉及训练时见过的相似但不同的实体，模型容易混淆（如"林肯"vs"肯尼迪"的刺杀细节）。
- **Sycophancy（阿谀奉承）的来源**：RLHF 的偏好数据让模型学到"顺从用户 = 高分"。当用户在 prompt 中暗示某个错误观点（"听说地球是平的，对吧？"），模型会附和而非纠正。这是 reward 优化的副作用——纠正用户可能被判"不友好"。

## 代码示例

```python
import torch

# ============ 1. 检测幻觉：基于 logprob 的置信度 ============
def detect_hallucination(model, tokenizer, prompt, threshold=0.3):
    """低平均 logprob → 高幻觉风险"""
    inputs = tokenizer(prompt, return_tensors="pt")
    with torch.no_grad():
        outputs = model.generate(
            **inputs, max_new_tokens=100,
            output_scores=True, return_dict_in_generate=True,
            do_sample=False
        )
    # 每个 token 的生成概率
    transition_scores = model.compute_transition_scores(
        outputs.sequences, outputs.scores, normalize_logits=True
    )
    probs = torch.exp(transition_scores[0])   # (gen_len,)
    avg_prob = probs.mean().item()
    min_prob = probs.min().item()

    response = tokenizer.decode(outputs.sequences[0][inputs.input_ids.shape[1]:], skip_special_tokens=True)
    return {
        "response": response,
        "avg_prob": avg_prob,
        "min_prob": min_prob,
        "hallucination_risk": "HIGH" if avg_prob < threshold else "LOW"
    }

# ============ 2. 缓解：Self-Consistency（多次采样投票）============
def self_consistency(model, tokenizer, prompt, n=5):
    """采样 n 次取多数答案，分歧大说明幻觉风险高"""
    answers = []
    for _ in range(n):
        out = model.generate(**tokenizer(prompt, return_tensors="pt"),
                            do_sample=True, temperature=0.7, max_new_tokens=100)
        answers.append(tokenizer.decode(out[0], skip_special_tokens=True))
    # 若 n 个答案高度一致 → 可信；分歧大 → 幻觉风险
    return answers

# ============ 3. RAG：用外部知识约束生成 ============
def rag_generate(model, tokenizer, retriever, query, k=3):
    """检索真实文档拼入 prompt，约束模型基于事实回答"""
    docs = retriever.search(query, k=k)
    context = "\n".join([d.text for d in docs])
    prompt = f"""仅根据以下资料回答，若资料不足请说"我不知道"：
<资料>{context}</资料>
问题：{query}
回答："""
    return model.generate(tokenizer(prompt, return_tensors="pt").input_ids, max_new_tokens=200)
```

## 对比/选型

| 缓解方法 | 原理 | 效果 | 成本 |
|---------|------|------|------|
| **RAG** | 检索真实文档约束生成 | 强（事实性场景） | 中（需知识库） |
| **Self-Consistency** | 多次采样投票 | 中（数学/推理） | 高（N× 推理） |
| **Chain-of-Thought** | 引导分步推理 | 中（推理幻觉） | 低 |
| **Logprob 过滤** | 拒绝低置信输出 | 弱（概率≠事实） | 低 |
| **RLHF for Truthfulness** | 训练奖励真实性 | 强（需大量数据） | 极高 |
| **知识图谱约束** | 解码时约束实体 | 强（领域场景） | 高（构建 KG） |
| **Refusal 训练** | 训练模型说"不知道" | 中（减少编造） | 中 |

## 常见坑/注意事项

- **RAG 不能完全消除幻觉**：即使提供真实文档，模型仍可能忽略文档而编造（尤其是参数中存储了强先验）。需用强约束 prompt（"仅根据以下资料，资料不足就说不知道"）+ 后处理校验。
- **温度调高反而减少某些幻觉**：贪心解码（temperature=0）容易陷入"局部最优的错误路径"，适度采样（temperature=0.7）+ Self-Consistency 能跳出。但温度过高又增加随机幻觉。
- **"模型不知道自己不知道"**：LLM 对所有问题都给出一个答案，缺乏校准的"不确定性"。logprob 不等于事实置信度——高概率的回答也可能是幻觉（训练数据中的常见错误）。
- **长上下文中的"中间遗忘"**：长 prompt（>4K token）时，模型对中间位置的信息利用效率下降（Lost in the Middle），可能基于前文/后文编造中间细节。把关键信息放 prompt 开头或结尾。
- **Sycophancy 测试**：评测时故意在 prompt 中植入错误假设（"为什么太阳绕着地球转？"），看模型是否纠正。优秀的模型应拒绝错误前提而非顺从。

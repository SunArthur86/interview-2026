# 迁移学习在大模型中的运用

在大模型时代，迁移学习主要体现在 **“预训练 + 微调”** 的范式中。

**1. 核心思想**
- 先在通用海量数据（如全网文本）上进行预训练，让模型学习通用的语言知识和世界知识。
- 将学到的通用知识“迁移”到特定的下游任务（如医疗问诊、代码生成）。

**2. 具体运用方式**
- **全参数微调**：解冻所有参数，在任务数据上更新（成本高）。
- **参数高效微调（PEFT）**：如 LoRA、Prompt Tuning。冻结大模型主体，只训练极少量的额外参数或提示词，将任务适配给模型。

**3. 优势**
- **降低数据门槛**：下游任务往往不需要大量数据，利用迁移学习即可获得良好效果。
- **降低计算成本**：避免从零训练大模型，仅需微调即可部署。

**4. 关键细节与原理**
- ** catastrophic forgetting（灾难性遗忘）**：在微调过程中，模型可能会过度适应新数据而丢失预训练时学到的通用知识。通常通过正则化技术或回放部分预训练数据来缓解。
- **领域差异**：当下游任务领域与预训练数据分布差异较大时（如从通用文本迁移到医学影像），简单的微调可能效果不佳，此时可能需要中间任务的“渐进式微调”。
- **参数高效微调原理**：
    - 假设预训练权重为 $W_0$，微调时的更新量为 $\Delta W$。LoRA 假设 $\Delta W$ 是低秩的，即 $\Delta W = B \cdot A$，其中 $A \in \mathbb{R}^{r \times d}, B \in \mathbb{R}^{d \times r}$，且 $r \ll d$。
    - 训练时只更新 $A$ 和 $B$，推理时将 $B \cdot A$ 加回原权重 $W_0$。这样大幅减少了显存占用（无需存储优化器状态）并提升了训练速度。

**6. 实战案例：垂直领域适配的“性格”丢失**
在实际将通用 LLM 迁移至金融情感分析任务时，初期使用全参数微调导致模型丢失了原有“多轮对话”的能力，变成了只会打分的单模型。后改用 **LoRA** 并在训练数据中混入 10% 的通用对话数据，成功保留了通用能力的同时，将金融情感分类 F1 分数从 0.75 提升至 0.92。

**7. 代码示例：LoRA 微调配置 (Python/PEFT)**
```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

# 加载基座模型
model = AutoModelForCausalLM.from_pretrained("llm-base-model")

# 配置 LoRA：只更新注意力权重中的矩阵
cnf = LoraConfig(
    target_modules=["q_proj", "v_proj"], 
    r=16,              # 秩，控制参数量
    lora_alpha=32,     # 缩放因子
    lora_dropout=0.05
)

# 包装模型，冻结原参数，注入可训练的 A, B 矩阵
model = get_peft_model(model, cnf)
model.print_trainable_parameters() # 输出: trainable params: 0.5% || all params: 100%
```

**8. 微调方式对比选型**

| 维度 | 全参数微调 (Full Fine-tuning) | LoRA / Adapter | Prompt Tuning |
| :--- | :--- | :--- | :--- |
| **显存占用** | 极高 (需存所有梯度/优化器状态) | 低 (仅存少量侧边参数) | 极低 (仅存 Prompt Embedding) |
| **训练速度** | 慢 | 快 | 最快 |
| **存储开销** | 每个 Task 存一份完整权重 (几十GB) | 每个 Task 仅存几 MB 权重 | 极小 |
| **模型上限** | **最高** (适应能力最强) | 高 (逼近全参数) | 低 (受限于 Prompt 空间) |
| **灾难性遗忘** | 风险高 | 风险低 (主体被冻结) | 无风险 |
| **实战适用** | 基座模型迭代、领域差异极大时 | **最推荐：通用垂直任务适配** | 极低资源、简单任务探索 |

### 迁移学习及其在大模型中的应用

**1. 迁移学习的概念**
*   **定义**：迁移学习是指将一个领域（源领域）中学到的知识迁移到另一个领域（目标领域）中应用的技术。通过这种方式，模型在目标任务中可以利用先前训练的模型参数，从而减少对大规模数据的需求和训练时间。
*   **核心假设**：源领域和目标领域之间存在某种相关性或共享的特征分布。例如，在 ImageNet 上学习到的边缘和纹理特征可以迁移到医学图像识别中。
*   **应用场景**：迁移学习通常用于数据量不足或训练成本较高的任务中，通过使用已经在相关任务上训练过的模型进行微调或调整，使其能够适应新的任务。

**2. 迁移学习在大模型中的应用**
在大模型（例如 GPT、BERT）时代，迁移学习体现为 **预训练-微调** 范式。

*   **预训练**：
    *   在海量的通用数据（如互联网文本）上进行自监督学习（如语言建模）。
    *   目标是学习通用的语言表示、世界知识和推理能力。
    *   这一阶段成本极高，通常只需进行一次。

*   **微调**：
    *   将预训练好的模型作为初始点，在特定的下游任务（如情感分析、问答）的小规模数据集上进行有监督训练。
    *   全量微调会更新所有参数，而参数高效微调（PEFT，如 LoRA, Adapter）仅更新少量参数。

```text
大模型迁移学习范式流程图：

+-------------------+      +----------------------+      +-------------------+
|   大规模通用数据   | ---> |  预训练     | ---> |  预训练模型  |
| (Text/Web/Crawl)  |      | (Self-Supervised)    |      | (通用知识)         |
+-------------------+      +----------------------+      +-------------------+
                                                                |
                                                                v (迁移)
+-------------------+      +----------------------+      +-------------------+
|   特定任务数据     | ---> |  微调    | <--- |  下游任务模型 |
| (Labeled/Small)   |      | (Supervised/PEFT)    |      | (领域适配)         |
+-------------------+      +----------------------+      +-------------------+
```

**3. 迁移学习的优势**
*   **节省计算资源**：避免从零开始训练模型，节省大量计算资源和时间（预训练是“一次投入，无限复用”）。
*   **减少数据需求**：即使目标任务的数据量不足，预训练的知识仍然可以帮助模型在目标任务上取得良好的效果。
*   **适应新任务**：迁移学习允许大模型快速适应不同的任务和领域，具有很好的灵活性。如今，通过 **Instruction Tuning**（指令微调），迁移学习甚至能让模型通过自然语言指令直接适应新任务，无需特定数据集。

#### 实战案例：金融领域垂类适配
在对通用 LLaMA 模型进行金融研报摘要生成的微调时，直接微调导致模型会“忘记”通用知识（如基本的数学运算），引入 LoRA（低秩适应）并冻结主模型参数，不仅节省了 4 张 A100 显存，还在保留通用推理能力的同时，将领域指令遵循率提升了 30%。

#### 代码示例：LoRA 微调配置
```python
from peft import LoraConfig, get_peft_model

# 配置 LoRA 参数：只更新部分注意力权重
lora_config = LoraConfig(
    r=16,                    # LoRA 秩，控制可训练参数量
    lora_alpha=32,           # 缩放因子
    target_modules=["q_proj", "v_proj"], # 指定对 Q 和 V 矩阵进行低秩分解
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# 获取可训练的 PEFT 模型
model = get_peft_model(base_model, lora_config)
model.print_trainable_parameters() # 输出：trainable params: 0.1% || all params: 7B
```

## 常见考点
1.  **在大模型语境下，什么是“负迁移”？如何避免？**（Hint：当源领域和目标领域差异过大时，预训练知识可能不仅无益反而有害。避免方法包括领域自适应预训练或使用更强的参数高效微调方法。）
2.  **全量微调与参数高效微调（PEFT）的区别？**（Hint：全量微调整个模型参数，效果最好但成本高；PEFT 冻结主干，训练少量适配层，成本低且易于切换任务。）
3.  **为什么说预训练-微调范式改变了 AI 的开发模式？**（Hint：从“针对特定任务收集数据并训练模型”转变为“利用通用基础模型进行适配”，降低了 AI 落地的门槛。）

## 记忆要点

- 迁移学习：将源领域知识迁移到目标领域，减少数据需求。
- 大模型应用：预训练（通用知识）+ 微调（领域适配）范式。
- PEFT（如LoRA）是迁移学习的高效实现，仅更新少量参数。
- 优势：降低算力成本，快速适配新任务，避免灾难性遗忘。

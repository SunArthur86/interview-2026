# Llama的结构

LLaMA 模型的结构基于 Transformer 架构，并引入了多项改进以提升训练稳定性和性能。

### 1. RMS Pre-Norm (归一化)
- **RMSNorm**：LLaMA 使用 RMSNorm 替代传统的 Layer Norm。RMSNorm 移除了均值计算，仅基于均方根进行归一化，计算更简单高效，且效果相当。
- **Pre-Norm**：归一化操作（RMSNorm）被放置在子层（如注意力层或 FFN）的输入之前，而不是输出之后。
- **优势**：Pre-Norm 有助于梯度在深层网络中的稳定传播，缓解梯度消失问题，使得训练更深的模型变得更加容易和稳定。

### 2. SwiGLU 激活函数
- **结构**：在前馈神经网络（FFN）中，LLaMA 使用 SwiGLU 替代了传统的 ReLU 或 GeLU。SwiGLU 结合了 Swish 激活函数和门控线性单元（GLU）。
- **优势**：SwiGLU 具有平滑的梯度和门控机制，能够提升模型的表达能力和收敛速度，通常在大规模语言模型中表现更好。

### 3. 旋转位置嵌入 (RoPE)
- LLaMA 使用旋转位置嵌入来编码位置信息。RoPE 通过旋转矩阵将位置信息注入到 Query 和 Key 中，能够更好地捕捉相对位置关系，且具备较好的外推性。

### 4. 其他细节
- **无偏置**：去除了大多数线性层中的偏置项，简化计算。
- **GQA (新版)**：Llama 3 及后续版本引入了 GQA（分组查询注意力）以加速推理。

### 5. 实战案例与代码
**实战案例**：在微调 Llama-2 模型时，若简单移除 Pre-Norm 层或改用 Post-Norm 而不调整学习率，往往会导致训练初期 Loss 爆炸或梯度消失。此外，RoPE 的维度必须与 Head Dim 一致，否则在长文本场景下（如 32k context）如果不使用 NTK-aware scaling 等插值技巧，模型效果会急剧下降（PPL 飙升）。

**代码示例 (RMSNorm)**：
```python
class RMSNorm(torch.nn.Module):
    def __init__(self, dim: int, eps: float = 1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim)) # 缩放参数 gamma

    def _norm(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)

    def forward(self, x):
        output = self._norm(x.float()).type_as(x)
        return output * self.weight
```

| 组件 | 传统 Transformer | Llama 结构 | 优势/改进点 |
| :--- | :--- | :--- | :--- |
| **归一化** | Layer Norm (Post-Norm) | **RMSNorm (Pre-Norm)** | 计算更快 (去均值)；训练更稳 (Pre-Norm) |
| **激活函数** | ReLU / GeLU | **SwiGLU** | 平滑梯度；性能提升约 1-2% (参数量增加) |
| **位置编码** | Sinusoidal / Learnable | **RoPE** | 相对位置感知；更好的长文本外推性 |
| **线性层** | Bias $\neq$ 0 | **Bias = 0** | 减少显存占用；简化计算 (无偏置项) |
| **Attention** | MHA | **GQA (Llama 3)** | 极大降低推理时显存带宽压力 |

## 常见考点
1. **RoPE 的数学原理**：解释如何通过复数乘法实现位置嵌入的旋转，以及为什么它具有相对位置编码的性质。
2. **SwiGLU 参数量**：相比标准 FFN (4d -> d)，SwiGLU 引入了额外的门控参数，通常维度是 $(2/3)*4d$，计算具体的参数量变化。
3. **Pre-Norm vs Post-Norm**：在模型初始化缩放上的区别（Pre-Norm 通常需要更深的 warmup 或调整残差权重）。

## 延伸：介绍一下Llama的结构

> 合并自 `llm-038`（相似度 76%）

LLaMA 模型在标准 Transformer 架构的基础上进行了多项改进，以提升训练稳定性和性能。以下是主要结构特点：

### 1. RMS Pre-Norm (预归一化)
*   **RMSNorm**：LLaMA 使用 **RMSNorm** 替代传统的 Layer Norm。RMSNorm 去掉了均值中心化的减法操作，仅基于均方根进行缩放。公式为 $\bar{x}_i = \frac{x_i}{\sqrt{\frac{1}{n}\sum x_j^2}} \cdot \gamma_i$。计算更高效，且效果相当。
*   **Pre-Norm**：将归一化层放在注意力层和 FFN 层**之前**（即 $x' = x + Sublayer(LayerNorm(x))$）。这有助于稳定深层网络的梯度传播，缓解梯度消失或爆炸问题。

### 2. SwiGLU 激活函数
*   LLaMA 在前馈神经网络（FFN）中使用了 **SwiGLU** 替代传统的 ReLU。
*   SwiGLU 结合了 Swish 激活函数和门控线性单元（GLU），虽然计算量增加了约 1/3，但显著提升了模型的性能和收敛速度。

### 3. RoPE (Rotary Positional Embeddings)
*   使用 **旋转位置编码** 替代绝对位置编码。RoPE 通过旋转矩阵将位置信息注入到 Query 和 Key 的向量中，能够更好地捕捉相对位置关系，且具有外推性。

### 4. 实战案例与代码
**实战案例**：在使用 RoPE 进行长文本微调时，如果直接将训练长度从 2k 扩展到 16k 而不使用 NTK-Aware Scaling 等插值技巧，模型会遭遇“灾难性遗忘”，注意力分数崩塌；而在实现 SwiGLU 时，由于其包含三个线性层，显存占用比标准 FFN 高，在量化部署（如 INT4）时要注意权重的排布以避免计算效率低下。

**代码示例**：
```python
def rms_norm(x, eps=1e-6):
    # x: [batch, seq_len, hidden_size]
    variance = x.pow(2).mean(-1, keepdim=True)
    return x * torch.rsqrt(variance + eps)

def apply_rotary_pos_emb(q, k, cos, sin):
    # RoPE 应用逻辑: q, k: [batch, seq_len, heads, head_dim]
    # cos, sin: [seq_len, head_dim]
    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)
    return q_embed, k_embed
```

### 5. 架构对比 (改进 vs 原版 Transformer)

| 组件 | 原版 Transformer | LLaMA 1/2/3 | 改进优势 |
| :--- | :--- | :--- | :--- |
| **归一化** | Layer Norm | RMSNorm | 计算更简单 (减去均值)，数值稳定性相当 |
| **归一化位置** | Post-Norm | Pre-Norm | 梯度更稳定，支持更深网络，减少 Warmup 步数 |
| **激活函数** | ReLU / GeLU | SwiGLU | 平滑且非单调，提升模型效果，虽增加 FLOPs 但性价比高 |
| **位置编码** | Sinusoidal / Learned | RoPE | 相对位置编码，具备更好的外推性 (支持长文本) |
| **注意力机制** | MHA | MHA (v1) / GQA (v2/3) | GQA 大幅降低推理显存，提升吞吐量 |

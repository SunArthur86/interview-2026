# 从MHA到MQA到GQA再到MLA的演进过程是怎样的?各自解决了什么问题

这是大模型KV Cache优化的演进路线,核心目标:减少推理时的KV Cache显存占用,从而提高吞吐量。

### 演进路线图

```text
┌──────────┐         ┌──────────┐         ┌──────────┐         ┌──────────┐
│   MHA    │  --->   │   MQA    │  --->   │   GQA    │  --->   │   MLA    │
│(Original)│         │(Shared)  │         │(Grouped) │         │(Latent)  │
└──────────┘         └──────────┘         └──────────┘         └──────────┘
KV Size: Large        Small                Medium             Tiny (Compressed)
Quality: High         Low                  High               High
Speed:  Slow          Fast                 Fast               Very Fast
```

| 方法 | Key/Value共享方式 | KV Cache大小 | 推理加速 | 效果 | 代表模型 |
|------|-------------------|-------------|----------|------|----------|
| **MHA** | 每个head独立K/V | $N_{heads} \times d_{head}$ | 1x | 最佳 | BERT, LLaMA-1 |
| **MQA** | 所有head共享1组K/V | $1 \times d_{head}$ | High (内存带宽不再瓶颈) | 下降明显 (复用能力弱) | PaLM, Claude-v1 |
| **GQA** | 分组共享K/V | $N_{groups} \times d_{head}$ | High | 接近MHA | LLaMA-2/3, Mistral |
| **MLA** | 低秩压缩K/V | $d_c$ (压缩维度) | Very High | 几乎无损 | DeepSeek-V2 |

### 详解:

**1. MHA (Multi-Head Attention)**
- 标准多头，每个 Query Head 对应独立的 Key 和 Value Head。
- KV Cache 显存占用最大，限制了 Batch Size 和 Max Length。

**2. MQA (Multi-Query Attention)**
- **做法**：所有 Query Head 共享同一份 Key 和 Value。
- **收益**：极大减少显存，提高推理吞吐量（GPU 读写显存带宽是主要瓶颈，减少 KV 读写即提速）。
- **代价**：模型表达能力受损，因为不同头的注意力模式无法关注不同的 Key 细节。

**3. GQA (Grouped-Query Attention)**
- **做法**：折中方案。将 Query Heads 分组，每组内的 Heads 共享一组 KV。
- **参数**：通常 `kv_head_num` < `q_head_num`。例如 LLaMA-3 中 `q=32`, `kv=8`，即每 4 个 Query Head 共享一组 KV。
- **优势**：在保持接近 MHA 性能的同时，获得接近 MQA 的推理速度。

**4. MLA (Multi-head Latent Attention)**
- **背景**：DeepSeek-V2 为了支持 MoE 和极长上下文设计的极致压缩方案。
- **做法**：
  - **Up-projection**：将 Key/Value 压缩到一个极小的 latent vector $c_K, c_V$ (维度 $d_c$ 远小于 $d_{model}$)。
  - **Storage**：Cache 中只存压缩后的 $c_K, c_V$。
  - **Decoding**：读取时通过 RoPE 和上投影矩阵恢复 Key/Value 用于 Attention 计算。
- **收益**：KV Cache 减少至原来的 1/4~1/8，支持 128K 甚至更长的上下文，且显存占用极低。

- **实战案例**：在将 LLaMA-2 (MHA) 迁移至生产环境推理时，受限于显存，Batch Size 只能设为 4。通过权重重训改用 GQA 后，在同显存下 Batch Size 提升至 16，QPS 翻倍且困惑度 (PPL) 几乎无损失。

# PPL及其数学公式

### PPL (Perplexity) 及其数学公式

#### 1. 定义
**困惑度** 是衡量语言模型预测下一个 token 难易程度的经典指标。它反映了模型对测试数据的“不确定度”。PPL 值越低，表示模型对数据的预测越准确、越“不困惑”。

#### 2. 数学公式
给定一个序列 $X = (x_1, x_2, \dots, x_N)$，语言模型的目标是计算该序列的联合概率 $P(X)$。对于自回归模型，该概率分解为：

$$P(X) = P(x_1) \cdot P(x_2 | x_1) \cdot P(x_3 | x_1, x_2) \cdots P(x_N | x_1, \dots, x_{N-1})$$

通常使用**平均负对数似然** 来衡量模型的好坏：
$$NLL = -\frac{1}{N} \sum_{n=1}^N \log P(x_n | x_{<n})$$

**PPL 定义为 NLL 的指数**：
$$PPL(X) = \exp(NLL) = \exp\left(-\frac{1}{N} \sum_{n=1}^N \log P(x_n | x_{<n})\right)$$

#### 3. 直观理解
- **PPL = 1**：模型完全确定下一个词（概率为 1），没有困惑。
- **PPL = 10**：模型在预测下一个词时，犹豫程度相当于在 10 个等可能的选项中做选择。
- **PPL 越小越好**：意味着模型分配给真实文本的概率越高，模型对语言的建模能力越强。

**计算流程**：
```text
Input: "I love natural language"

Step 1: Predict probabilities for next token at each position
Start -> P(love | I)
I, love -> P(natural | I, love)
...

Step 2: Get probability of the ACTUAL next token (from ground truth)
p1 = P("love") = 0.8
p2 = P("natural") = 0.05
...

Step 3: Calculate Geometric Mean (inverse of exp(NLL))
PPL = (p1 * p2 * ... * pn) ^ (-1/n)
```

#### 4. 实战拓展

**实战案例**：
在业务上线 A/B 测试中，曾遇到过模型 A 在通用测试集 PPL 优于模型 B，但线上点击率（CTR）却更低。排查发现模型 A 为了降低 PPL，倾向于生成高概率的“废话”（如 "I think that..."），导致回答冗长乏味，这说明 PPL 并不总是与人类偏好强相关。

**代码示例 (Python)**：
```python
import torch
import torch.nn.functional as F

def compute_ppl(model, input_ids, attention_mask):
    outputs = model(input_ids, attention_mask=attention_mask, labels=input_ids)
    # outputs.loss 是 CrossEntropy (Mean NLL)
    # 记得处理 padding 的影响，通常 HF 模型会自动处理 -100
    avg_nll = outputs.loss
    ppl = torch.exp(avg_nll)
    return ppl.item()
```

#### ## 常见考点
1. **为什么 PPL 使用几何平均而不是算术平均？**
   - 答：因为概率 $P$ 是相乘关系（联合概率），取对数后求和再取指数（几何平均）能保证概率的数学一致性。且几何平均对长度的变化不那么敏感（虽然仍受长度影响，但比直接乘积稳定）。
2. **BPE (Byte Pair Encoding) 对 PPL 计算有什么影响？**
   - 答：词表大小和切分粒度会影响 PPL。切分越细（如字符级），单步预测容易，PPL 较低；切分越粗（如单词级），单步预测难，PPL 较高。因此不同词表模型间的 PPL 通常不可直接比较。
3. **训练 Loss 和 PPL 的关系？**
   - 答：如果 Loss 是 Cross Entropy（即平均负对数似然），则 $PPL = e^{Loss}$。它们是单调关系，直接一一对应。

# 大模型推理的输入和输出是什么?Temperature 参数有什么作用

- **推理** 是将输入文本送入模型,生成输出的过程.

- **输入**:一段文本序列
- **输出**:模型预测的下一个 Token(逐个生成)

- **Temperature 参数**
  - **Temperature = 0**:每次选概率最高的 token(确定性输出)
  - **Temperature = 1**:按原始概率分布采样(有创造性)
  - **Temperature > 1**:概率分布更平坦(更随机,可能不连贯)

- **实用建议**
  - 代码生成/事实问答:Temperature = 0~0.3
  - 创意写作/头脑风暴:Temperature = 0.7~1.0
  - 超过 1.2 通常不建议

- **一句话理解**:Temperature 低 = 严谨(选最优),Temperature 高 = 创意(加点随机性).

- **实战案例**：在构建 SQL 生成器时，设置 Temperature=0.7 导致生成的 SQL 语法正确但表名经常随机变换（如 users, user_list, app_users 混用），导致执行报错。将 Temperature 强制降为 0 后，模型对于同一问题的 SQL 输出变得完全确定，后端可以通过缓存结果大幅降低数据库压力。

- **解码策略参数对比表**：
| 参数 | 作用 | 推荐值 (严谨任务) | 推荐值 (创意任务) | 效果差异 |
| :--- | :--- | :--- | :--- | :--- |
| **Temperature** | 控制概率分布的平滑度 | 0.0 - 0.2 | 0.7 - 1.0 | 越高越随机，越低越确定 |
| **Top-p (Nucleus)** | 累积概率截断，保留质量高的词 | 0.9 - 0.95 | 0.9 - 1.0 | 防止采样到长尾低概率词 |
| **Top-k** | 仅从概率最高的 k 个词中选 | 40 - 50 | 40 - 100 | 限制候选词范围，提高稳定性 |
| **Frequency Penalty** | 惩罚重复出现的词 | 0.0 - 0.2 | 0.5 - 0.8 | 降低重复度，增加多样性 |

- **代码示例 (OpenAI API 调用)**：
```python
import openai

response = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "写一首关于春天的诗"}],
    temperature=0.8,  # 创意调高
    top_p=0.9,        # 常用 0.9
    max_tokens=200
)
```

## 技术原理

**值越低输出越确定，适合逻辑任务**
Temperature 越低，softmax 概率分布越尖锐（峰值被放大），模型倾向于选择概率最高的 token，输出更确定、更"保守"。Temperature=0 时退化为贪心解码（Greedy Decoding），每次都选最高概率词，输出完全确定。这种模式适合代码生成、事实问答、SQL 生成等需要"标准答案"的任务。

**值越高输出越随机，适合创意任务**
Temperature 越高，概率分布越平坦（被"加热"平滑），原本低概率的词也有机会被采中，输出更具创造性和多样性。但过高（如 >1.2）会导致概率分布过度平坦，输出变得不连贯、胡言乱语。创意写作、头脑风暴、对话生成通常用 0.7~1.0。

**本质是调整概率分布的平滑度**
数学上，Temperature 通过对 logits 除以 T 再做 softmax 实现：`p_i = softmax(logit_i / T)`。T<1 放大差异（尖锐），T>1 缩小差异（平坦），T=1 保持原始分布。它不改变候选词集合，只改变采样概率。

**通常设置为 0 到 1 之间**
实战经验值：代码/事实任务 0~0.3，对话/总结 0.3~0.7，创意写作 0.7~1.0。一般不建议超过 1.2。配合 Top-p（核采样）一起使用效果更好——Top-p 截断长尾低概率词，防止采样到完全不相关的词。

## 代码示例

```python
# OpenAI API：不同任务的参数配置
# 1. 代码生成/SQL：低温度追求确定性
response = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "把这句 SQL 优化一下: ..."}],
    temperature=0.0,     # 完全确定，可缓存结果
    top_p=1.0,
)

# 2. 创意写作：高温度追求多样性
response = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "写一首关于春天的诗"}],
    temperature=0.9,     # 增加随机性
    top_p=0.9,           # 截断长尾，保证连贯
    frequency_penalty=0.5,  # 降低重复
    max_tokens=200,
)
```

```python
# 底层原理：Temperature 对 softmax 的作用
import torch
import torch.nn.functional as F

def temperature_softmax(logits, T):
    return F.softmax(logits / T, dim=-1)

# logits = [2.0, 1.0, 0.5]  三个候选词
# T=0.1 -> [0.999, 0.001, 0.000]  几乎必选第一个（确定）
# T=1.0 -> [0.506, 0.186, 0.113]  原始分布
# T=2.0 -> [0.336, 0.245, 0.210]  分布平坦（随机）
```

## 注意事项

- 输入输出：输入文本序列，输出逐个生成的预测 Token。
- Temperature=0：确定性输出，选概率最高词，适合代码/问答。
- Temperature>0：增加随机性，值越高越有创意，适合写作。
- Top-p：核采样，从累积概率达 p 的最小词集中采样。
- 低 Temperature 任务可缓存结果降本；高 Temperature 任务需配合 repetition_penalty 防止循环。

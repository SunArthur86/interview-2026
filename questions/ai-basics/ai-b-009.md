# BERT、GPT、T5 三种预训练范式的本质区别是什么?各自适合什么任务

- **三种预训练范式对比**

| 特性 | BERT (Encoder-only) | GPT (Decoder-only) | T5 (Encoder-Decoder) |
| :--- | :--- | :--- | :--- |
| **注意力机制** | 双向 | 单向 | Encoder双向+Decoder单向 |
| **训练目标** | MLM (完形填空) | CLM (预测下一个词) | Span Corruption (去噪重构) |
| **典型任务** | NLU (分类、实体识别) | NLG (对话、写作) | Seq2Seq (翻译、摘要) |
| **生成能力** | 弟 | 极强 | 强 |
| **理解能力** | 极强 | 强 | 强 |
| **代表模型** | RoBERTa, DeBERTa | GPT-4, Llama | T5, BART, FLAN-T5 |

- **1. BERT (Encoder-only)**
- **结构**：Transformer Encoder，堆叠多个自注意力层。
- **训练目标**：**Masked Language Model (MLM)**，随机遮盖句子中的词，让模型根据上下文预测。
- **注意力机制**：双向注意力，预测当前词时能看到“过去”和“未来”的所有词。
- **擅长**：自然语言理解（NLU）任务，如文本分类、命名实体识别（NER）、阅读理解（QA）。
- **缺陷**：无法直接用于生成，因为训练时的 Mask 在推理时不存在。

- **2. GPT (Decoder-only)**
- **结构**：Transformer Decoder，带有因果掩码。
- **训练目标**：**Causal Language Model (CLM)**，根据上文预测下一个 Token。
- **注意力机制**：单向（因果）注意力，当前词只能看左边的词，不能看右边。
- **擅长**：自然语言生成（NLG）任务，如对话、创意写作、代码生成。
- **缺陷**：处理需要双向上下文的任务（如情感分析、指代消解）时，直觉上不如 Encoder 强力。

- **3. T5 (Encoder-Decoder)**
- **结构**：Transformer Encoder + Decoder。
- **训练目标**：**Span Corruption**，类似 BERT 的 Mask，但 Decoder 需要生成完整的被遮盖片段。
- **注意力机制**：Encoder 双向理解输入，Decoder 单向生成输出。
- **擅长**：序列到序列任务，如机器翻译、文本摘要、复杂 QA。
- **缺陷**：参数利用率不如 Decoder-only，推理速度通常较慢（需要两个部分串行计算）。

- **架构注意力图示**
```
BERT (双向):     GPT (单向):       T5 (混合):
[ A  B  C  D ]    [ A  B  C  D ]    Enc: [A  B  C  D ]
  ↑  ↑  ↑  ↑      ↑              (双向)  ↓
  └──┴──┴──┘      │             Dec: [  x  ] (单向)
 (看全部)      [看左边]         (生成对应部分)
```

- **实战案例**：在开发“合同条款提取”功能时，最初尝试用 BERT 做序列标注，但遇到跨长距离依赖时效果一般。后来改用 Llama 3 (GPT架构) 配合 Prompt：“请提取合同中的金额条款...”，利用 LLM 的上下文理解和指令跟随能力，在几乎零样本的情况下达到了 95% 的准确率，且泛化性远强于微调后的 BERT。

- **代码示例**：
```python
import torch
from transformers import AutoTokenizer, AutoModelForCausalLM

# 加载 GPT (Decoder-only) 模型
tokenizer = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

input_text = "The future of AI is"
inputs = tokenizer(input_text, return_tensors="pt")

# 生成时，Decoder 利用 KV Cache 进行单向自回归生成
outputs = model.generate(**inputs, max_length=20, pad_token_id=tokenizer.eos_token_id)
print(tokenizer.decode(outputs[0]))
```

- **为什么 Decoder-only (GPT) 成为大模型主流?**
1. **目标简单统一**：只需要预测下一个 token，数据获取容易（无需构造成对的输入输出，任何文本均可）。
2. **Scaling Law 最清晰**：随着参数量增加，Decoder-only 模型的性能提升最稳定、可预测。
3. **工程优势**：KV Cache、并行推理优化、vLLM 等推理框架均针对 Causal LM 优化。
4. **涌现能力**：在大规模参数下，GPT 展现出了逻辑推理、思维链等能力，证明了scale的有效性。

- **一句话总结**：BERT 善“懂”，GPT 善“编”，T5 善“译”。但在 LLM 时代，GPT 因其极致的扩展性胜出。

- ## 常见考点
1. **BERT 的训练模式缺陷**：BERT 预训练时的 `[MASK]` 标记在微调时不存在，存在 Pretrain-Finetune gap；解决方案如 XLNet 的排列语言模型。
2. **KV Cache 的作用**：解释在 Decoder-only 生成过程中，缓存 Key 和 Value 如何减少重复计算，加速推理。
3. **T5 的文本到文本**：T5 将所有任务都统一转化为 text-to-text 格式（如翻译：“translate English to German: ...”），这种统一性带来的优势。

## 记忆要点

- BERT：Encoder-only，双向注意力+MLM，擅长NLU（理解），生成能力弱。
- GPT：Decoder-only，单向注意力+CLM，擅长NLG（生成），扩展性最强。
- T5：Encoder-Decoder，双向理解+单向生成，擅长Seq2Seq（翻译/摘要）。
- 一句话总结：BERT善懂，GPT善编，T5善译；LLM时代GPT因Scaling Law胜出。

## 结构化回答

**30 秒电梯演讲：** 三种范式一句话分清：BERT 善"懂"（双向注意力+MLM 擅长理解），GPT 善"编"（单向自回归+CLM 擅长生成），T5 善"译"（Encoder-Decoder 擅长 Seq2Seq）。但 LLM 时代 GPT 的 Decoder-only 架构因为 Scaling Law 扩展性最强，最终胜出。

**展开框架：**
1. **BERT（Encoder-only）** — 双向注意力 + MLM 完形填空，擅长 NLU（分类、实体识别），缺陷是生成弱、训练推理有 gap。
2. **GPT（Decoder-only）** — 单向因果注意力 + CLM 预测下一词，擅长 NLG（对话、写作），扩展性最强，KV Cache 加速推理。
3. **T5（Encoder-Decoder）** — Encoder 双向理解 + Decoder 单向生成，Span Corruption 去噪重构，擅长翻译摘要，但参数利用率不如 Decoder-only。

# 线程池原理是什么？

### 线程池原理

#### 核心作用
线程池主要用于控制运行线程的数量，处理过程中将任务放入队列，在线程创建后启动这些任务。如果线程数量超过最大数量，超出数量的线程排队等候，等其它线程执行完毕，再从队列中取出任务来执行。主要特点包括：线程复用、控制最大并发数、管理线程。

#### 线程复用原理
Thread 类的 `start` 方法会调用 `run` 方法。线程池的实现原理是：工作线程的 `run` 方法中不断循环从队列中获取 `Runnable` 任务并执行。在获取下一个任务之前，线程可以是阻塞的，从而避免线程频繁创建销毁的开销。

#### 核心组成
1. **线程池管理器**：用于创建并管理线程池。
2. **工作线程**：线程池中执行任务的线程。
3. **任务接口**：每个任务必须实现的接口（如 Runnable），用于工作线程调度运行。
4. **任务队列**：用于存放待处理的任务，提供缓冲机制。

#### 构造参数
`ThreadPoolExecutor` 构造方法核心参数：

1. **corePoolSize**：核心线程数，即使空闲也不会被回收（除非设置 `allowCoreThreadTimeOut`）。
2. **maximumPoolSize**：线程池中允许的最大线程数量。
3. **keepAliveTime**：当线程数大于 corePoolSize 时，多余的空闲线程存活的时间。
4. **unit**：keepAliveTime 的时间单位。
5. **workQueue**：任务队列，被提交但尚未被执行的任务（如 LinkedBlockingQueue）。
6. **threadFactory**：线程工厂，用于创建线程。
7. **handler**：拒绝策略，当任务太多来不及处理时的处理方式。

#### 工作流程
1. 如果当前运行的线程数 < `corePoolSize`，创建新线程运行任务。
2. 如果当前运行的线程数 >= `corePoolSize`，将任务放入队列。
3. 如果队列已满且当前线程数 < `maximumPoolSize`，创建非核心线程运行任务。
4. 如果队列已满且当前线程数 >= `maximumPoolSize`，执行拒绝策略（默认抛出异常）。

```text
       提交任务
          │
          ▼
  ┌───────────────────┐
  │ 线程数 < coreSize? │──Yes──▶ 创建核心线程执行
  └───────────────────┘
          │ No
          ▼
  ┌───────────────────┐
  │   任务入队成功？   │──Yes──▶ 等待线程调度
  └───────────────────┘
          │ No (队列满)
          ▼
  ┌─────────────────────┐
  │ 线程数 < maximumSize?│──Yes──▶ 创建非核心线程执行
  └─────────────────────┘
          │ No
          ▼
  ┌───────────────────┐
  │   执行拒绝策略     │
  └───────────────────┘
```

#### 拒绝策略
1. **AbortPolicy**（默认）：直接抛出 `RejectedExecutionException`。
2. **CallerRunsPolicy**：由调用者线程执行该任务（“背压”机制，降低提交速度）。
3. **DiscardOldestPolicy**：丢弃队列里最老的任务，并尝试重新提交当前任务。
4. **DiscardPolicy**：直接丢弃任务，不予处理。

#### 状态转换
线程池内部使用 `ctl` 变量（AtomicInteger）的高3位存储运行状态，低29位存储工作线程数。5种状态如下：
1. **RUNNING**：接受新任务，处理队列任务。
2. **SHUTDOWN**：不接受新任务，但处理队列任务（调用 `shutdown()`）。
3. **STOP**：不接受新任务，不处理队列任务，中断正在执行的任务（调用 `shutdownNow()`）。
4. **TIDYING**：所有任务已终止，工作线程数为0，过渡到此状态后会执行 `terminated()` 钩子方法。
5. **TERMINATED**：`terminated()` 方法执行完成。

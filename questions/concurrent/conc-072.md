# 什么是Java 线程安全的实现？

### Java 锁的升级

JDK 1.6 对 `synchronized` 进行了优化，锁会根据竞争情况逐渐升级，只能升级不能降级：

1. **偏向锁**：
   - 假设锁主要由同一个线程多次获得。
   - 线程访问同步块时，会在对象头中记录线程 ID。以后该线程再进入时无需加锁，降低代价。

2. **轻量级锁**：
   - 当有第二个线程尝试获取偏向锁时，偏向锁升级为轻量级锁。
   - 线程通过 **自旋**（循环尝试）来获取锁，不会阻塞，提高性能。

3. **重量级锁**：
   - 当自旋超过一定次数仍未获取到锁，锁膨胀为重量级锁。
   - 线程会被阻塞（挂起），涉及用户态与内核态切换，性能降低。

### Volatile 关键字

`volatile` 是 Java 虚拟机提供的轻量级同步机制。

**特性**：
1. **保证可见性**：
   - 当一个线程修改了 `volatile` 变量，新值会立即同步到主内存，其他线程读取时会从主内存重新读取，确保看到最新值。
2. **禁止指令重排序**：
   - 通过插入内存屏障，阻止特定类型的编译器重排序和处理器重排序，保证有序性（如单例模式的双重检查锁）。
3. **不保证原子性**：
   - 例如 `i++` 操作即使加了 `volatile` 也不是线程安全的，需要配合 `synchronized` 或原子类。

### Java 线程安全的实现策略

1. **互斥同步（阻塞同步）**：
   - 悲观锁策略，假设会发生冲突。
   - 实现：`synchronized` 关键字、`ReentrantLock` 等。

2. **非阻塞同步**：
   - 乐观锁策略，假设没有冲突，先操作如有冲突再重试。
   - 核心是 **CAS (Compare And Swap)**，利用硬件指令保证原子性。
   - 问题：ABA 问题（通过版本号解决）、CPU 开销大、只能保证一个共享变量的原子性。

3. **无同步方案**：
   - 可重入代码：不依赖共享资源。
   - 线程本地存储：`ThreadLocal`，为每个线程提供独立的变量副本。

        ▼                             │           │
   ┌─────────┐ (CAS替换Mark Word到栈帧)│           │
   │轻量级锁 │                         │           │
   └────┬────┘<────────(自旋成功)──────┘           │

        ▼                                          │

   │重量级锁 │─────────────────────────────────────┘

### ## 常见考点
1. **Synchronized 与 ReentrantLock 的区别**：实现层面（JVM vs API）、锁释放（自动 vs 手动）、公平性（非公平 vs 可选公平）、条件变量等。
2. **CAS 的 ABA 问题及解决方案**：`AtomicStampedReference` 通过版本号解决。此外，如果在极端高并发下 CAS 自旋失败过多，有什么优化手段？

### ## 面试追问
1. **偏向锁的撤销与重偏向**：偏向锁在什么情况下会撤销？如果一个对象经历过偏向锁撤销，达到一定阈值后，JVM 会如何处理（重偏向 vs 批量撤销）？
2. **锁粗化与锁消除**：JIT 编译器除了锁升级，还有哪些优化手段？请举例说明锁粗化和锁消除的场景。
3. **Synchronized 在 JDK 1.6 之后的自适应自旋**：自旋锁的自旋次数是固定的吗？自适应自旋是如何根据上次获取锁的成功率动态调整自旋时间的？

### ## 易错点
1. **锁降级**：虽然 JVM 支持锁升级，但在标准实现中，**锁是不支持降级**的。一旦锁膨胀为重量级锁，它将保持重量级状态，直到被释放。虽然 HotSpot 有支持锁降级的研发尝试，但在一般面试和通用认知中，应坚持“不能降级”的结论。
2. **Volatile 的可见性范围**：`volatile` 只能保证对单个变量读/写的可见性，但不能保证复合操作（如 `i++`）的原子性，也不能保证多个 `volatile` 变量之间的原子性（例如修改 `flag` 和 `count`，其他线程可能看到中间状态）。

- 三分法：互斥同步（悲观锁）、非阻塞同步（乐观锁/CAS）、无同步方案（ThreadLocal）。
- synchronized锁只能升级不能降级：无锁 -> 偏向锁 -> 轻量级锁(自旋) -> 重量级锁。
- volatile保证可见性和有序性（禁止重排序），但绝不保证复合操作的原子性。
- CAS核心缺陷：ABA问题、高竞争下自旋消耗大、仅能保证单变量原子操作。

# 什么是操作系统的特征？

操作系统的四大特征：

1.  **并发**：宏观上多个程序同时运行（通过时间片轮转），微观上交替执行。并发是并行的前提。
2.  **共享**：系统资源可被多个并发进程共同使用。分为互斥共享（打印机，需加锁）和同时访问（磁盘，可并发读）。
3.  **虚拟**：通过某种技术把物理实体变为逻辑上的多个。如虚拟内存、虚拟 CPU、虚拟终端。
4.  **异步**：进程以不可预知的速度推进，走走停停。

**最基本特征：并发和共享（二者互为存在条件）。**

**实战案例**：在秒杀系统中，千万级流量并发访问数据库（共享），虽然数据库连接池只有几百个，但通过虚拟技术和异步 I/O（如 Node.js 或 Netty），系统能在物理资源有限的情况下处理大量请求，体现了操作系统的并发与异步特征。

**对比表格**：

| 特征 | 核心描述 | 典型技术/实例 | 关联点 |
| :--- | :--- | :--- | :--- |
| **并发** | 宏观同时，微观交替 | 时间片轮转, 多线程 | 共享的前提 |
| **共享** | 资源被多进程复用 | 互斥锁 (打印机), 共享内存 (磁盘) | 并发的条件 |
| **虚拟** | 物理实体 -> 逻辑多个 | 虚拟内存 (空分复用), 分时系统 (时分复用) | 无资源变有资源 |
| **异步** | 执行速度不可预知 | 进程阻塞/唤醒, 系统调用 | 程序执行的非线性 |

注意：操作系统的并发≠并行。并发是「同时处理」（单核也可并发），并行是「同时执行」（必须多核）。

### 常见考点
1.  **并发与并行的区别**：强调单核也能并发（交替执行），但并行必须多核（物理上同时）。
2.  **虚拟技术的具体实现**：能举例说明时分复用（虚拟处理器）和空分复用（虚拟内存）。
3.  **互斥共享与同时共享的实例**：互斥共享（如打印机、临界区代码） vs 同时共享（如只读文件、可重入代码）。

## 记忆要点

- 四大特征：并发、共享、虚拟、异步。并发和共享是最基本特征，互为存在条件。
- 因为单核通过时间片轮转实现宏观同时微观交替，所以并发是并行前提（单核并发，多核并行）。
- 共享分两类：互斥共享需加锁（如打印机），同时共享可并发（如磁盘读）。
- 虚拟靠复用技术：空分复用造虚拟内存，时分复用造虚拟处理器。

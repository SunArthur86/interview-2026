# 什么是操作系统的特征？

操作系统的四大特征：

1.  **并发**：宏观上多个程序同时运行（通过时间片轮转），微观上交替执行。并发是并行的前提。
2.  **共享**：系统资源可被多个并发进程共同使用。分为互斥共享（打印机，需加锁）和同时访问（磁盘，可并发读）。
3.  **虚拟**：通过某种技术把物理实体变为逻辑上的多个。如虚拟内存、虚拟 CPU、虚拟终端。
4.  **异步**：进程以不可预知的速度推进，走走停停。

**最基本特征：并发和共享（二者互为存在条件）。**

**实战案例**：在秒杀系统中，千万级流量并发访问数据库（共享），虽然数据库连接池只有几百个，但通过虚拟技术和异步 I/O（如 Node.js 或 Netty），系统能在物理资源有限的情况下处理大量请求，体现了操作系统的并发与异步特征。

**对比表格**：

| 特征 | 核心描述 | 典型技术/实例 | 关联点 |
| :--- | :--- | :--- | :--- |
| **并发** | 宏观同时，微观交替 | 时间片轮转, 多线程 | 共享的前提 |
| **共享** | 资源被多进程复用 | 互斥锁 (打印机), 共享内存 (磁盘) | 并发的条件 |
| **虚拟** | 物理实体 -> 逻辑多个 | 虚拟内存 (空分复用), 分时系统 (时分复用) | 无资源变有资源 |
| **异步** | 执行速度不可预知 | 进程阻塞/唤醒, 系统调用 | 程序执行的非线性 |

注意：操作系统的并发≠并行。并发是「同时处理」（单核也可并发），并行是「同时执行」（必须多核）。

### 常见考点
1.  **并发与并行的区别**：强调单核也能并发（交替执行），但并行必须多核（物理上同时）。
2.  **虚拟技术的具体实现**：能举例说明时分复用（虚拟处理器）和空分复用（虚拟内存）。
3.  **互斥共享与同时共享的实例**：互斥共享（如打印机、临界区代码） vs 同时共享（如只读文件、可重入代码）。

## 技术原理

操作系统的四大特征并非独立，而是由"多道程序设计"这一基础技术派生出的统一体系：

- **并发的实现原理（时分复用）**：单核 CPU 通过**时间片轮转**实现并发——每个进程分配一个时间片（如 10ms），时间片用完触发时钟中断，操作系统切换到下一个进程。宏观上多个进程"同时"运行，微观上 CPU 在快速交替执行。多核 CPU 则能真正并行（多个核同时执行不同进程）。并发的核心开销是**上下文切换**——保存当前进程的寄存器/PC/内存映射，加载下一个进程的状态，约耗时 1~10μs。
- **共享的实现原理（资源互斥 vs 同时访问）**：
  - **互斥共享**：打印机、临界区代码这类资源，任意时刻只能被一个进程使用，靠锁（互斥量、信号量）协调。
  - **同时访问**：磁盘、只读文件这类资源，多个进程可同时读不冲突。技术上靠缓存一致性协议和只读不变性保证安全。
- **虚拟的实现原理（空分/时分复用）**：
  - **虚拟内存（空分复用）**：把磁盘的 swap 区和物理内存合并，每个进程看到的都是连续的虚拟地址空间（如 4GB），实际物理内存可能只有 16GB 被多进程分时共享。靠 MMU + 页表把虚拟地址映射到物理地址。
  - **虚拟处理器（时分复用）**：一个物理 CPU 通过时间片轮转"变成"多个逻辑 CPU，每个进程以为自己独占 CPU。
- **异步的来源**：进程的执行速度受 CPU 调度、I/O 阻塞、中断等影响不可预知。进程会"走走停停"——运行到 I/O 操作时阻塞让出 CPU，I/O 完成后被唤醒重新调度。这种非线性的推进顺序就是异步。

## 代码示例

用代码体现并发与共享的协作（多线程共享资源的互斥访问）：

```java
public class ConcurrencyDemo {
    // 共享资源：模拟打印机这类互斥资源
    private static int sharedCounter = 0;
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        // 并发：多线程时分复用 CPU
        Runnable task = () -> {
            for (int i = 0; i < 10000; i++) {
                // 互斥共享：synchronized 保证同一时刻只有一个线程修改
                synchronized (lock) {
                    sharedCounter++;   // 非原子操作，不加锁会丢失更新
                }
            }
        };

        Thread t1 = new Thread(task, "worker-1");
        Thread t2 = new Thread(task, "worker-2");
        long start = System.nanoTime();
        t1.start(); t2.start();
        t1.join(); t2.join();   // 异步：主线程等待子线程完成
        long elapsed = System.nanoTime() - start;

        System.out.println("结果: " + sharedCounter);        // 正确: 20000
        System.out.printf("耗时: %.2f ms（含上下文切换开销）%n", elapsed / 1e6);
    }
}

// 虚拟内存的体现：JVM 堆是虚拟地址空间，物理内存可能不足触发 GC 或 swap
// 可用 jcmd <pid> VM.flags 查看 -Xmx（最大堆 = 虚拟内存上限）
```

```
# 观察并发与上下文切换（Linux）
$ pidstat -w -p <pid> 1    # 每秒报告自愿/非自愿上下文切换次数
$ vmstat 1                 # 观察 cs（context switch）列
```

## 注意事项

- **并发 ≠ 并行**：并发是"同时处理"（单核也可，靠时间片轮转），并行是"同时执行"（必须多核）。面试常考这个区别，单核 CPU 也能并发但不能并行。
- **并发和共享互为存在条件**：没有并发（单任务）就不需要共享管理；没有共享（资源无限）并发就无意义。两者是操作系统最基本的特征，虚拟和异步是它们的衍生。
- **上下文切换有成本**：线程/进程过多时，CPU 大量时间耗在切换而非计算，反而降低吞吐。这是线程池要限制最大线程数的根本原因。
- **异步带来确定性难题**：进程执行速度不可预知，导致竞态条件、死锁等问题。必须用锁、原子操作、内存屏障等同步原语保证正确性。


## 记忆要点

- 四大特征：并发、共享、虚拟、异步。并发和共享是最基本特征，互为存在条件。
- 因为单核通过时间片轮转实现宏观同时微观交替，所以并发是并行前提（单核并发，多核并行）。
- 共享分两类：互斥共享需加锁（如打印机），同时共享可并发（如磁盘读）。
- 虚拟靠复用技术：空分复用造虚拟内存，时分复用造虚拟处理器。

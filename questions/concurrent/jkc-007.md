# JAVA多线程并发

Java 中实现多线程主要有以下几种方式：

**1. 继承 Thread 类**
Thread 类本质上是实现了 Runnable 接口的一个实例。启动线程的唯一方法是通过 Thread 类的 `start()` 实例方法。`start()` 是一个 native 方法，它会启动一个新线程并执行 `run()` 方法。
```java
public class MyThread extends Thread {
    public void run() {
        System.out.println("MyThread.run()");
    }
}
MyThread myThread = new MyThread();
myThread.start();
```

**2. 实现 Runnable 接口**
如果自己的类已经 extends 另一个类，就无法直接 extends Thread，此时可以实现 Runnable 接口。
```java
public class MyThread extends OtherClass implements Runnable {
    public void run() {
        System.out.println("MyThread.run()");
    }
}
// 使用方式：new Thread(new MyThread()).start();
```

**3. 实现 Callable 接口（补充）**
Callable 接口与 Runnable 类似，但支持返回执行结果并能抛出异常，通常配合 FutureTask 使用。
```java
FutureTask<Integer> future = new FutureTask<>(new MyCallable());
new Thread(future).start();
Integer result = future.get(); // 阻塞获取结果
```

**4. 使用线程池（推荐）**
在实际生产环境中，为了避免频繁创建和销毁线程带来的性能开销，通常使用线程池（如 `ExecutorService`）。

```text
Java 线程启动流程
┌─────────────┐
│   Main 线程  │
└──────┬──────┘
       │ new Thread(Runnable) / Callable
       ▼
┌─────────────┐     start()    ┌──────────────────┐
│  Thread 对象 │ ──────────────▶ │  NEW (新建状态)   │
└─────────────┘                 └────────┬─────────┘
                                        │ JVM 调度
                                        ▼
                               ┌──────────────────┐
                               │ RUNNABLE (运行中) │ ◀── CPU 时间片
                               └────────┬─────────┘
                                        │ wait()/sleep()
                               ┌────────▼─────────┐
                               │ BLOCKED/WAITING  │ (阻塞/等待)
                               └──────────────────┘
```

### 实战案例
在生产环境中**严禁**直接使用 `new Thread()`，因为这会导致无限制创建线程，耗尽系统资源引发 OOM。应始终使用线程池（如 `ThreadPoolExecutor`）来复用线程，并通过 `Future` 拿取任务执行结果。

### 代码示例 (Java)
```java
// 推荐方式：使用线程池提交 Callable 任务
ExecutorService executor = Executors.newFixedThreadPool(10);
Future<String> future = executor.submit(() -> {
    // 模拟耗时计算
    Thread.sleep(1000); 
    return "Task Result";
});

try {
    // get() 会阻塞直到任务完成，也可使用 get(timeout, unit) 避免永久等待
    System.out.println(future.get()); 
} catch (Exception e) {
    e.printStackTrace();
}
```

### 常见考点
1. **run() 和 start() 的区别**：`run()` 是普通方法调用，在当前线程执行；`start()` 会启动新线程，由 JVM 调用 `run()`。直接调用 `run()` 不会开启多线程。
2. **线程的生命周期状态**：Java 线程有哪些状态（NEW, RUNNABLE, BLOCKED, WAITING, TIMED_WAITING, TERMINATED）？什么情况下会发生状态转换？
3. **Callable 与 Runnable 的区别**：`Callable` 有返回值、能抛出受检异常，而 `Runnable` 不能。


## 记忆要点

- 创建四方式：继承 Thread、实现 Runnable/Callable、推荐用线程池。
- 启动看 start()，因为它由 JVM 调度新线程；直接调 run() 只是普通方法。
- Callable 有返回且抛异常，而 Runnable 无返回不抛受检异常。
- 生产严禁 new Thread，因为无限制创建会耗尽资源导致 OOM。

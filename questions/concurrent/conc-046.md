# Java中有哪些锁？它们的分类和特点是什么？

### Java 锁分类与特点

#### 1. 乐观锁 vs 悲观锁
*   **乐观锁**：认为读多写少，并发冲突概率低。操作时不加锁，更新时判断数据是否被修改（通常通过版本号或 CAS）。Java 中主要通过 **CAS**（Compare And Swap）实现。
*   **悲观锁**：认为写多，并发冲突概率高。操作时直接加锁，确保数据独占。Java 中的 `synchronized` 和 `ReentrantLock`（AQS 框架）均属此类。

**实战案例**：在电商扣减库存场景中，若并发量不大，使用数据库 `version` 字段实现乐观锁（CAS 思想）比直接排他锁性能更好，避免大量线程阻塞；但在秒杀高并发写场景下，冲突频繁，乐观锁会导致大量重试，此时应降级为悲观锁（如 Redis `setnx` 或数据库行锁）。

#### 2. 自旋锁
*   **原理**：线程获取锁失败时，不进行阻塞（不放弃 CPU），而是执行一个空循环（自旋）等待锁释放。
*   **目的**：减少线程上下文切换（用户态与内核态切换）的开销。
*   **缺点**：如果锁被占用时间长，自旋会浪费 CPU 资源。
*   **优化**：JDK 1.6 引入了 **适应性自旋**，自旋次数不再固定，而是由前一次在同一个锁上的自旋时间及锁拥有者状态决定。

**代码示例**：
```java
// 简易自旋锁实现 (CAS)
AtomicReference<Thread> lock = new AtomicReference<>();
public void lock() {
    Thread current = Thread.currentThread();
    // 循环尝试获取锁
    while (!lock.compareAndSet(null, current)) {
        // TODO: 可在此增加自旋次数限制或让出CPU
    }
}
```

#### 3. Synchronized 同步锁
*   **性质**：属于悲观锁、独占锁、可重入锁、非公平锁。
*   **作用范围**：
    1.  修饰实例方法：锁住当前对象实例 (`this`)。
    2.  修饰静态方法：锁住类的 Class 对象。
    3.  修饰代码块：锁住括号内指定的对象。
*   **Monitor 机制**：基于对象头的 Mark Word 实现。包含 Wait Set（等待队列）、Entry List（竞争队列）、Owner（持有锁的线程）等组件。
*   **锁升级**：JDK 1.6 后进行了优化，锁状态会从 **无锁 -> 偏向锁 -> 轻量级锁 -> 重量级锁** 升级，以提高性能。

**实战案例**：在某个高吞吐量日志打印组件中，最初使用 `synchronized` 修饰代码块。升级到 JDK 1.6 后，由于单线程顺序写入居多，偏向锁自动生效，吞吐量提升了约 20%。但引入多线程异步刷盘机制后，锁竞争加剧，升级为重量级锁导致 CPU 飙升，后改为 `ReentrantReadWriteLock` 解决。

#### 4. 锁升级流程图
Synchronized 的性能优化核心在于锁升级，不同竞争程度采用不同锁策略。

```text
    线程 A 访问同步块
           │
           ▼
    ┌─────────────┐
    │   无锁状态   │ (Mark Word 存储对象 HashCode)
    └──────┬──────┘
           │ 检查 Mark Word
           ▼
    ┌─────────────┐
    │   偏向锁     │ (Mark Word 存储线程 A ID，无 CAS 开销)
    └──────┬──────┘
           │ 线程 B 尝试获取（撤销偏向）
           ▼
    ┌─────────────┐
    │  轻量级锁    │ (Mark Word 指向栈中 Lock Record，自旋 CAS)
    └──────┬──────┘
           │ 自旋超过阈值或竞争激烈
           ▼
    ┌─────────────┐
    │  重量级锁    │ (依赖 OS Mutex，线程阻塞挂起)
    └─────────────┘
```

#### 5. 锁机制对比表

| 特性 | Synchronized | ReentrantLock |
| :--- | :--- | :--- |
| **实现层面** | JVM 关键字，C++ 实现 | JDK 类，基于 AQS (Java 实现) |
| **释放锁** | 自动释放 (代码块执行完或异常) | 手动释放 (必须在 finally 中 unlock) |
| **公平性** | 非公平 | 可选 (构造函数传 true/false) |
| **中断机制** | 不可中断 (必须等待锁释放) | 支持中断 (lockInterruptibly) |
| **条件队列** | 单个 WaitSet (wait/notify) | 多个 Condition (支持精细唤醒) |
| **锁类型** | 悲观锁、独占、可重入 | 悲观锁、独占、可重入 |

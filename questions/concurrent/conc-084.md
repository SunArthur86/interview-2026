# 什么是内存中对象？

### 对象在内存中的布局

在 HotSpot 虚拟机中，对象在内存中存储的布局可以分为三块区域：**对象头**、**实例数据** 和 **对齐填充**。

#### 1. 对象头
对象头包含两部分信息：

1.  **运行时元数据**：
    -   **哈希码**：对象的哈希值（未调用 hashCode 时默认为 0，计算后存入）。
    -   **GC 分代年龄**：记录对象在 Survivor 区经过的 GC 次数，达到阈值晋升老年代（默认 15 岁）。
    -   **锁状态标志**：标识当前锁的状态（无锁、偏向锁、轻量级锁、重量级锁）。
    -   **偏向线程 ID**、**偏向时间戳**：用于实现偏向锁。
    -   线程持有的锁。
2.  **类型指针**：
    -   指向对象的类元数据（Method Area），虚拟机通过这个指针确定该对象是哪个类的实例。
    -   如果是数组对象，还需要存储数组长度（4 字节）。

#### 2. 实例数据
实例数据是对象真正存储的有效信息，包括程序代码中定义的各种类型的字段（包括从父类继承的）。

**存储规则**：
-   **相同宽度的字段分配在一起**：如 long/double 放在一起，int 放在一起。
-   **父类变量在子类之前**：即使父类变量定义在子类之后。
-   **CompactFields**：如果开启，子类窄变量可能插入到父类变量的空隙中，节省空间。

#### 3. 对齐填充
-   HotSpot 要求对象的起始地址必须是 8 字节的整数倍（为了提高内存访问效率，64位 CPU 一次读取 64 位）。
-   对象头已经被设计为 8 字节的倍数，如果实例数据部分大小不是 8 字节的倍数，就需要通过对齐填充来补全。

### 对象的访问定位

Java 栈中的引用指向堆中的对象，通过引用访问对象的方式主要有两种：

1.  **句柄访问**：
    -   堆中划分一块区域作为句柄池，句柄中存储了对象实例数据和类型数据的指针。
    -   栈中的引用指向句柄。
    -   **优点**：对象移动（GC 整理）时只需修改句柄中的指针，引用本身不变。

2.  **直接指针访问**（HotSpot 采用）：
    -   栈中的引用直接存储堆中对象的地址。
    -   对象中包含指向类型数据的指针。
    -   **优点**：速度更快，节省了一次指针定位的开销。

```text
Object Memory Layout (HotSpot 64-bit)

+------------------------+  <- Object Start Address
|      Mark Word         |  (8 bytes, Lock info, GC age, Hashcode)
+------------------------+
|    Class Pointer       |  (8 bytes, pointer to Class metadata)
+------------------------+
|   Array Length (Opt)   |  (4 bytes, only if array object)
+------------------------+
|                        |
|   Instance Data        |  (Fields, aligned by width)
|   (Parent fields)      |
|   (Child fields)       |
|                        |
+------------------------+
|   Padding              |  (0~7 bytes, make total size multiple of 8)
+------------------------+  <- Object End Address
```

## 常见考点
1. **对象头的大小**：在 64 位 JVM 中，开启指针压缩（-XX:+UseCompressedOops，默认开启）时，Mark Word 8 字节，Class Pointer 4 字节，共 12 字节；不开启压缩时，共 16 字节。
2. **为什么需要内存对齐**：CPU 访问内存时不是按字节逐个读取，而是按块（如 Cache Line, 通常 64 字节）读取。不对齐可能导致跨 Cache Line 读取，增加 CPU 消耗；同时也为了原子性保证（如 64 位 long 读写）。
3. **引用类型存储**：栈帧中的 LocalVariableTable 存储的是引用的句柄或直接指针，引用本身指向堆内存。

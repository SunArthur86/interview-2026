# 什么是PCB-“切换桢”？

**PCB（Process Control Block，进程控制块）**

是操作系统内核为每个进程维护的数据结构，记录了进程的所有管理信息（如进程标识符、处理器状态、进程调度信息等），是进程存在的唯一标志，也是系统进行进程调度的依据。

**切换帧（Context/Context Switch Frame）**

“切换帧”通常指上下文切换时需要保存和恢复的执行现场。当发生线程或进程切换时，操作系统需要将当前的执行状态（即“上下文”）保存到 PCB（进程）或 TCB（线程）中，主要内容包括：

1.  **程序计数器（PC）**：CPU 下一条要执行的指令地址。
2.  **通用寄存器**：当前计算产生的中间结果数据。
3.  **栈指针（SP）**：指向当前线程栈顶的位置。
4.  **程序状态字（PSW）**：包含 CPU 状态标志（如进位、溢出、中断屏蔽位等）。

**线程与进程切换的区别**

- **线程切换**：由于同一进程内的线程共享虚拟地址空间，切换时不需要切换页表（地址映射），开销相对较小。
- **进程切换**：涉及虚拟地址空间的切换（TLB 刷新），开销远大于线程切换。

**实战案例**：
在压测中发现吞吐量上不去，通过 `vmstat` 发现 `cs`（上下文切换次数）高达数万。经排查是代码中存在 `synchronized` 锁竞争激烈且 CAS 自旋失败，导致大量线程频繁在用户态与内核态之间切换。

**代码示例（Linux 命令监控）**：
```bash
# 1秒输出一次，观察 cs (context switch) 列
vmstat 1
# 或使用 pidstat 监控指定 Java 进程的线程切换
pidstat -w -p <pid> 1
```

**对比表格（切换开销对比）**：

| 切换类型 | 寄存器保存 | TLB 刷新 | 缓存（L1/L2）影响 | 典型开销 |
| :--- | :--- | :--- | :--- | :--- |
| **进程切换** | 需要 | 需要（CR3寄存器改变） | 失效（地址空间变了） | 高（微秒级） |
| **线程切换** | 需要 | 不需要（同进程） | 部分失效 | 低（亚微秒级） |
| **协程切换** | 仅需保存部分寄存器 | 不需要 | 几乎不影响 | 极低（纳秒级） |

**补充细节：**
- **上下文切换的开销**：上下文切换不仅仅是保存寄存器，还包括 CPU 缓存的失效（特别是 L1/L2 缓存），TLB（转换后备缓冲器）的刷新，以及调度器本身的逻辑执行。高频的上下文切换会显著降低系统吞吐量。
- **Linux 中的实现**：在 Linux 内核中，并没有严格独立的 TCB，线程被视为一种“共享进程资源的轻量级进程”，线程的上下文信息也保存在 `task_struct`（即 Linux 的 PCB）中。
- **用户态与内核态切换**：Java 中的 `synchronized` 锁膨胀到重量级锁时，会涉及用户态到内核态的切换，这本身也是上下文切换的一部分开销。

**上下文切换示意图：**

```text
时间轴 ──────────────────────────────────────────────────────────────>

   任务 A 运行                任务 B 运行                任务 A 运行
      │                          │                          │
      ▼                          ▼                          ▼
┌──────────┐               ┌──────────┐               ┌──────────┐
│ 用户态执行│               │ 用户态执行│               │ 用户态执行│
└─────┬────┘               └─────┬────┘               └─────┬────┘
      │ 时片中断/系统调用         │ 时片中断/系统调用         │
      ▼                          ▼                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                    内核态调度器                          │
│                                                                 │
│   1. 保存 A 的上下文           2. 恢复 B 的上下文        ... │
│   (寄存器 -> PCB/栈)           (PCB/栈 -> 寄存器)            │
│                                                                 │
│   (若进程切换：切换 CR3 寄存器/页表，刷新 TLB)                   │
└─────────────────────────────────────────────────────────────────┘
```

## 常见考点
1. **如何减少上下文切换？**：使用无锁编程（CAS）、协程、减少线程数量（利用线程池）、让同一线程处理亲和性任务。
2. **Java 中如何监控上下文切换次数？**：在 Linux 下可以使用 `vmstat 1`

## 记忆要点

- 因为线程共享地址空间，所以切换时无需切换页表，开销相对较小

# 什么是线程池？

### 什么是线程池

**1. 基本概念**
线程池是一种基于池化思想管理线程的工具。它维护着多个线程，等待监督管理者分配可并发执行的任务。这避免了在处理短时间任务时创建与销毁线程的额外开销。

**2. 为什么需要线程池 (解决的问题)**
- **资源消耗**：频繁创建和销毁线程会消耗大量 CPU（上下文切换）和内存资源。
- **响应速度**：任务到达时，线程池中的空闲线程可直接执行，无需等待线程创建的延迟。
- **可管理性**：线程是稀缺资源，如果无限制创建，不仅消耗系统资源，还会降低系统稳定性。线程池可以统一分配、调优和监控。

**3. 线程池的优点**
1. **降低资源消耗**：重用已创建的线程，减少对象创建、销毁造成的垃圾回收（GC）压力和内存抖动。
2. **提高响应速度**：任务到达时，若有空闲线程则立即执行，无需排队等待创建。
3. **提高线程的可管理性**：
   - 控制最大并发数，防止资源耗尽导致系统崩溃。
   - 提供定时、定频、单线程串行等高级功能。

**4. Java 提供的 4 种典型线程池 (通过 Executors 工具类创建)**
虽然生产环境建议直接使用 `ThreadPoolExecutor` 构造，但理解这些典型配置有助于面试：

1. **FixedThreadPool (固定大小线程池)**
   - **特征**：核心线程数 = 最大线程数，队列使用 `LinkedBlockingQueue` (无界)。
   - **适用**：负载较重的服务器，为了限制资源使用。
   - **风险**：队列无界可能导致 OOM。

2. **SingleThreadExecutor (单线程线程池)**
   - **特征**：核心线程数 = 最大线程数 = 1，队列使用 `LinkedBlockingQueue`。
   - **适用**：需要保证任务顺序执行（FIFO）的场景。
   - **风险**：队列无界可能导致 OOM。

3. **CachedThreadPool (缓存线程池)**
   - **特征**：核心线程数 = 0，最大线程数 = `Integer.MAX_VALUE`，队列使用 `SynchronousQueue`。线程空闲 60 秒回收。
   - **适用**：大量短生命周期、异步的小任务。
   - **风险**：最大线程数无限，高并发下可能创建大量线程导致 OOM 或 CPU 飙升。

4. **ScheduledThreadPoolExecutor (定时任务线程池)**
   - **特征**：支持定时及周期性任务执行。核心线程数固定，最大线程数 `Integer.MAX_VALUE`，队列使用 `DelayedWorkQueue`。
   - **适用**：需要延时执行或周期执行的任务。
   - **变体**：`SingleThreadScheduledExecutor` 是它的特例（核心线程数为1）。

**线程池处理任务流程图：**
```
       +------------+
       | 新任务到达 |
       +-----+------+
             |
             v
+---------------------+
| 核心线程池是否已满? | ---- No ---> [执行任务] (Core Thread)
+----------+----------+
           | Yes
           v
+---------------------+
|    工作队列是否已满? | ---- No ---> [入队列] (Queue)
+----------+----------+
           | Yes
           v
+-----------------------+
| 线程池是否达到最大值? | ---- No ---> [创建非核心线程] (Max Thread)
+----------+----------+
           | Yes
           v
+-----------------------+
|      执行拒绝策略      | (Reject Policy)
+-----------------------+
```

## 常见考点
1. **线程池中 submit() 和 execute() 方法的区别？**
   - `execute()`：提交不需要返回值的任务，无法捕获异常。
   - `submit()`：提交需要返回值的任务，返回 `Future` 对象。通过 `Future.get()` 可以获取返回值，且 `get()` 会阻塞直到任务完成；同时 `Future.get()` 可以捕获执行过程中的异常。
2. **如何关闭线程池？**
   - `shutdown()`：不再接受新任务，等待已提交任务（包括队列中的）执行完。
   - `shutdownNow()`：不再接受新任务，尝试停止正在执行的任务（通过 `Thread.interrupt()`），并返回队列中未执行的任务列表。

## 记忆要点

- 核心作用：降低资源消耗、提高响应速度、提高线程可管理性
- 任务流转：核心线程满 -> 队列满 -> 最大线程满 -> 执行拒绝策略
- 四大池对比：Fixed定长且无界队列，Cached无限线程且同步队列，Single单线程串行
- 方法对比：execute()无返回值且无法捕获异常，submit()返回Future且可获取执行异常
- 避坑指南：禁止用Executors创建，因为默认无界队列或无限线程极易导致OOM

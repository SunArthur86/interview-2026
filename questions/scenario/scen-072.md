# 如何设计一个网盘系统？类似百度网盘，支持大文件上传下载。

【场景分析】
网盘系统需求：多端文件同步、大文件上传、断点续传、秒传、文件分享、权限管理。

【核心功能】
1. 文件上传：分片上传 + 断点续传 + 秒传
2. 文件下载：分片下载 + 断点续传
3. 文件管理：增删改查 + 目录树 + 回收站
4. 文件分享：链接分享 + 密码保护 + 有效期
5. 文件同步：多设备文件同步

【秒传实现】
1. 上传前计算文件MD5
2. 查询服务端是否已存在相同MD5
3. 如存在 → 直接建立引用关系（不实际上传）
4. 如不存在 → 正常上传

【分片上传 + 断点续传】
```
1. 客户端：
   - 大文件分片（每片5MB）
   - 计算每片MD5
   - 上传前询问服务端已上传哪些分片
   - 只上传缺失分片
2. 服务端：
   - 记录分片上传状态
   - 所有分片上传完成后合并
   - 合并后校验整体MD5
```

【实战案例】
在处理10GB大文件上传时，如果客户端将所有分片MD5一次性上传到服务端校验，会导致URL过长或Header溢出。实际开发中应改为“预上传”接口：客户端先上传整体文件Hash，服务端返回缺失的分片序号列表（如 `[1, 3, 5]`），客户端再按需上传。

【代码示例】
```java
// 服务端：合并分片校验 (Java)
public boolean mergeFile(String fileId, String fileHash, int totalChunks) {
    List<File> chunks = getUploadedChunks(fileId);
    if (chunks.size() != totalChunks) return false; // 分片缺失
    
    File mergedFile = mergeChunks(chunks); // 顺序合并
    String actualHash = DigestUtils.md5Hex(new FileInputStream(mergedFile));
    
    if (!actualHash.equals(fileHash)) {
        mergedFile.delete();
        return false; // 文件损坏，校验失败
    }
    saveToOSS(mergedFile); // 上传对象存储
    return true;
}
```

【存储架构】
1. 元数据存储：
   - MySQL：文件信息（名称/大小/路径/权限）
   - Redis：目录树缓存
2. 内容存储：
   - 对象存储（OSS/S3）：文件实际内容
   - 小文件合并：多个小文件合并存储
3. CDN加速：
   - 下载走CDN
   - 热点文件预缓存

【存储方案对比】

| 方案 | 适用场景 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| **自建NAS/文件服务器** | 私有云、内网 | 完全自主可控，带宽成本低 | 扩容难，需维护硬盘故障，性能瓶颈明显 |
| **对象存储 (OSS/S3)** | 公有网盘 | 无限扩容，高可靠，自带CDN集成 | 存储成本随量线性增加，流量费较高 |
| **纠删码存储** | 极致成本控制 | 存储利用率高，容错性好 | 读写性能损耗大，恢复数据计算耗时 |

【文件分享】
- 生成分享链接 + 提取码
- 有效期控制（1天/7天/永久）
- 访问次数限制
- 权限控制（查看/下载/编辑）

【存储优化】
- 去重：相同MD5的文件只存一份
- 压缩：自动压缩文本文件
- 冷热分离：30天未访问的文件迁移到低价存储
- 配额管理：用户存储空间限制

【多端同步】
- 增量同步：只同步变化的文件
- 冲突解决：基于版本号 + 时间戳
- 同步队列：变更事件入MQ，异步同步

【安全设计】
- 传输加密：HTTPS
- 存储加密：服务端加密（SSE）
- 访问控制：ACL + Token
- 防盗链：Referer检查 + 签名URL

## 记忆要点

- 秒传原理：上传前先算文件MD5比对服务端，存在则直接加引用关系，零流量传输
- 断点续传：大文件切片(如5MB)，服务端通过预上传接口返回缺失分片号，实现按需续传
- 最终校验：所有分片合并后，必须重新计算整体MD5与客户端上报值比对，防文件损坏
- 存储架构：MySQL存元数据+目录树，大文件实体压入对象存储(OSS/S3)
- 成本优化：相同MD5物理去重只存一份，冷热分离将长期未访问文件迁移至低价存储

## 结构化回答

**30 秒电梯演讲：** 通过哈希去重实现秒传，分片传输支持断点续传。打比方——像搬家，大件拆开搬(分片)，搬一半停了下次接着搬(断点)，别人有一样的直接借(秒传)。落到工程上，文件指纹(MD5/SHA1)去重，只存一份。

**展开框架：**
1. **秒传** — 文件指纹(MD5/SHA1)去重，只存一份
2. **分片上传** — 大文件切片，支持断点续传
3. **元数据与存储分离** — 元数据与存储分离，冷热数据分层

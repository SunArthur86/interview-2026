# 如何设计一个实时风控系统？毫秒级识别欺诈交易。

【场景分析】
实时风控目标：在交易发生的毫秒内判断风险，拦截欺诈/盗刷/洗钱。

**【规则引擎与模型对比】**

| 维度 | 规则引擎 | ML模型 |
| :--- | :--- | :--- |
| **处理速度** | 极快 (微秒级) | 快 (毫秒级，依赖特征提取) |
| **可解释性** | 强 (明确触发了哪条规则) | 弱 (黑盒评分) |
| **维护成本** | 高 (需人工配置/调优) | 中 (依赖数据训练/迭代) |
| **对抗能力**| 弱 (黑产容易绕过固定规则) | 强 (能识别潜在异常模式) |

【风控流程】
```
交易请求 → 规则引擎 → ML模型 → 决策引擎 → 放行/拦截/验证
    ↓           ↓          ↓
  基础检查    规则匹配   模型评分
```

【规则引擎（实时）】
- 速度规则：同账户1分钟内交易>5次 → 拦截
- 位置规则：异地交易 → 验证
- 金额规则：单笔>5万 → 人工审核
- 设备规则：新设备交易 → 验证
- 黑名单：命中黑名单 → 直接拦截
- 规则配置可视化（DSL）

【ML模型（毫秒级）】
1. 特征工程（实时计算）：
   - 用户画像：历史交易频率/金额/品类
   - 设备指纹：设备ID/IP/位置
   - 行为特征：操作速度/滑动轨迹
   - 关联特征：同设备关联的其他账号
2. 模型选择：
   - XGBoost/LightGBM：表格数据效果好
   - DNN：复杂非线性特征
   - GNN：关联网络（团伙检测）
3. 模型Serving：
   - TF Serving / ONNX Runtime
   - 推理延迟<50ms
   - 特征从Redis实时读取

**【实战代码示例】**
```java
// 简易规则匹配引擎
public RiskResult checkRule(TransactionEvent event) {
    // 1. 黑名单检查 (O(1) 查找)
    if (blacklistService.isBlackUser(event.getUserId())) {
        return RiskResult.block("Hit Blacklist");
    }
    // 2. 频率检查 (Redis HyperLogLog 或 计数器)
    long count = redisTemplate.opsForValue().increment("freq:" + event.getUserId());
    if (count == 1) redisTemplate.expire("freq:" + event.getUserId(), 60, TimeUnit.SECONDS);
    if (count > 5) {
        return RiskResult.verify("High Frequency");
    }
    return RiskResult.pass();
}
```

【实时特征计算（Flink）】
```java
// 实时计算用户近1小时交易频率
transactions
    .keyBy(TxEvent::getUserId)
    .window(SlidingEventTimeWindows.of(Time.hours(1), Time.minutes(5)))
    .aggregate(new CountAggregator())
    .addSink(new RedisSink());  // 写入Redis供风控查询
```

【决策引擎】
```
综合风险分 = 规则分 × W1 + 模型分 × W2

if (风险分 > 0.9) → 拦截
else if (风险分 > 0.6) → 二次验证（短信/人脸）
else → 放行
```

**【实战案例】**
某支付平台曾遇到"团伙欺诈"：数千个账号在短时间内从同一IP段发起小额盗刷。由于单账号频率未触发阈值，规则引擎漏过。后引入**图计算**，通过"设备-IP-账号"的关联度分析，识别出该IP段关联了异常密集的账户群，成功拦截黑产团伙，挽回损失近千万。

【图谱风控（关联分析）】
- 构建用户-设备-IP-手机号-银行卡 关系图
- 图数据库（Neo4j/JanusGraph）
- 检测：同一设备关联N个账号 → 团伙
- 检测：资金环形流转 → 洗钱

【灰度与反馈】
- 新规则灰度上线（只告警不拦截）
- 用户申诉 → 反馈标注 → 模型迭代
- 误拦截率监控

【系统要求】
- 风控决策延迟 < 100ms
- 规则更新实时生效
- 99.99%可用性（风控挂了交易也要能放行）

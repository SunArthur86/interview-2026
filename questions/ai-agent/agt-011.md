# Reflexion 框架

### 1. 概念解释
Reflexion 框架的核心在于“反思”。它不只是尝试，而是在每次失败后生成语言化的反思文本，作为记忆存储，指导下一次尝试。这是一个典型的**试错-反思-改进**的强化循环。

### 2. 核心机制
- **Actor (智能体)**：执行任务，生成输出或动作。
- **Evaluator (评估器)**：评估结果（如单元测试结果、人类打分、Self-Check）。
- **Reflector (反思器)**：根据评估反馈生成具体的、语言化的反思文本。

### 3. 工作流与数据流

│                       Task Input                         │
└─────────────────────┬────────────────────────────────────┘
                      ▼
         ┌────────────────────────────┐
         │         Actor (LLM)        │
         │   (Prompt + History)       │
         └─────────────┬──────────────┘
                       ▼
               ┌───────────────┐
               │    Output    │
               └───────┬───────┘
                       ▼

         │      Evaluator             │
         │  (Unit Test / Heuristic)   │
         └─────────────┬──────────────┘
                       │
            ┌──────────┴──────────┐
            ▼                     ▼
      (Success)              (Failure)
            │                     │
            ▼                     ▼
      ┌─────────┐      ┌──────────────────────┐

### 4. 实战深化
#### 4.1 实战案例
在实际的 Python 数据分析任务中，Agent 可能会因为 Pandas 的 `SettingWithCopyWarning` 而生成逻辑正确但运行报错的代码。Reflexion 的 Evaluator 捕获堆栈信息后，Reflector 不会只报错，而是会生成具体的反思建议：“建议在修改切片数据前显式使用 `.copy()`”，从而在第二轮直接修正，比简单的重试效率高很多。

#### 4.2 关键代码示例

```python
# Python 伪代码：Reflector 生成反思并更新上下文
if not evaluator.check_result(actor_output):
    error_trace = get_error_trace(actor_output)
    # Reflector 是一个独立的 LLM 调用，专注于分析错误
    reflection_text = reflector_llm.invoke(
        f"Task failed. Error: {error_trace}. "
        f"What should be done differently next time? Be specific."
    )
    # 将反思文本作为记忆存入 Actor 的 Prompt 历史中
    actor_memory.append(f"[PREVIOUS REFLECTION]: {reflection_text}")
    # Actor 重试
    new_output = actor.invoke(task_input, memory=actor_memory)
```

## 记忆要点

- 核心机制：试错-反思-改进，失败后生成语言化反思文本指导重试。
- 三组件：Actor执行任务，Evaluator评估结果，Reflector生成反思。
- 流程：Actor输出 -> 评估失败 -> Reflector生成反思 -> 追加到上下文重试。
- 优势：比简单重试效率高，能针对性修正具体错误（如代码Bug）。
- 关键点：反思文本必须具体且语言化，作为记忆存储供下次调用。

## 结构化回答

**30 秒电梯演讲：** Reflexion 就是给 Agent 配一本"错题本"——每次失败后不是傻重试，而是让 Reflector 生成一段具体的反思文本塞进上下文，指导下一次怎么避坑。它和简单重试的区别是：重试是"再来一遍"，Reflexion 是"想清楚错在哪再来一遍"。

**展开框架：**
1. **三组件分工** — Actor 执行任务，Evaluator 用单测或规则判断成败，Reflector 拿到错误堆栈生成语言化反思。
2. **反思即记忆** — 反思文本会被 append 到 Actor 的 Prompt 历史里，作为跨轮复用的经验，而不是一次性丢掉。
3. **关键约束** — 反思必须具体（比如"忘记用 .copy()"而不是"代码有错"），否则模型无法据此修正。
4. **适用边界** — 适合有明确评估指标的任务（代码题、数学题）；没有客观对错的任务反思价值有限。

**收尾：** 我在 Pandas 数据分析任务里用过，Reflector 捕获 SettingWithCopyWarning 后给出"修改切片前先 .copy()"的建议，第二轮就过了。您想深入聊 Evaluator 怎么设计、反思格式还是和 ReAct 的结合？

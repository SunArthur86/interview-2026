# Function Calling 基础

### Function Calling 基础

**定义**：大模型不直接输出答案，而是输出结构化的“函数调用指令”（函数名及参数），由宿主程序执行函数后将结果返回给模型，形成闭环。

**工作原理**：
1. **定义**：客户端将工具元数据（名称、描述、参数 JSON Schema）发送给模型。
2. **决策**：模型根据意图输出 `tool_calls`（包含 ID、函数名、参数）。
3. **执行**：宿主程序解析参数，调用本地函数。
4. **回传**：将执行结果以 `role=tool` 消息写回上下文，再次请求模型生成最终答案。

**核心要点**：
*   **Tool Choice**：`auto`（模型自选）、`none`（禁止）、`required`（强制调用）、`{type: function, function: {name: xxx}}`（强制特定函数）。
*   **JSON Schema**：用于参数校验及给模型提供结构化提示，减少参数胡编。
*   **并行调用**：模型可一次返回多个 `tool_calls`，支持并行执行（需注意线程安全）。

**实战案例**：
在开发订票 Agent 时，模型常将 "明天" 错误解析为具体日期字符串而非相对时间对象。通过在 JSON Schema 的 `description` 字段中添加严格示例（如 `"Description: ISO 8601 format, e.g., 2023-10-01"`）并配合 Pydantic 验证器，将参数格式错误率从 15% 降至 0.5%。

**代码示例（Pydantic 强校验）**：
```python
from pydantic import BaseModel, Field

# 定义参数模型，自动生成 JSON Schema 并校验
class WeatherParams(BaseModel):
    city: str = Field(..., description="城市名称，拼音或中文")
    unit: str = Field(default="celsius", enum=["celsius", "fahrenheit"])

def call_weather_function(args_json: str):
    try:
        params = WeatherParams.model_validate_json(args_json)
        # 安全调用，此时 params.unit 必定在枚举中
        return get_real_weather(params.city, params.unit)
    except ValidationError as e:
        return f"参数错误: {e}"
```

## 常见考点
1. **如果模型生成的 JSON 格式错误怎么办？**
   使用 Output Fixing Parser 或尝试再次 Prompt 模型修复；在代码层做 Try-Catch 和默认值处理。
2. **Function Calling 能否实现多轮对话？**
   可以，需要维护完整的 Messages History（包含 User, Assistant, Tool 消息），随每轮请求发送给模型以保持上下文。
3. **Tool Choice 参数的适用场景？**
   `required` 用于必须调用的场景（如数据录入）；`auto` 最通用，让模型判断是否需要工具；`none` 用于纯对话场景。

## 结构化回答

**30 秒电梯演讲：** Function Calling 就是让模型"下单"不"做菜"——模型只输出结构化的函数调用指令（函数名+参数 JSON），由宿主程序执行后把结果写回，模型再总结。核心是"模型决策，应用执行"，模型绝不直接跑代码。

**展开框架：**
1. **五步闭环** — 发工具定义 → 模型返回 tool_calls → 本地解析执行 → 结果以 role=tool 写回 → 模型生成最终答案。
2. **Tool Choice 控策略** — auto（模型自选）、none（禁用）、required（强制调用）、指定函数名（强制特定工具），按场景选。
3. **JSON Schema 是命门** — 跨语言标准、运行时自动校验、给模型明确结构化提示，减少参数胡编。
4. **实战技巧** — Schema 描述加 Few-shot 示例（如 ISO 8601 格式）、Pydantic 强校验、并行调用注意线程安全。

**收尾：** 我做过订票 Agent，"明天"老被解析错，Schema 里加了 ISO 8601 示例 + Pydantic 校验后参数错误率从 15% 降到 0.5%。您想深入聊 Tool Choice、Schema 设计还是并行调用？

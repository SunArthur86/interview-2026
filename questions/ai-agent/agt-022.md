# 检索策略

### 检索策略

**6.1-6.3 检索模式**
- **向量检索**：擅长语义理解，弱于专有名词匹配（如型号、ID）。
- **关键词检索 (BM25)**：擅长精确匹配（关键词、ID），基于词频统计。
- **混合检索**：融合两者分数，兼顾语义与字面。

**实战案例**：在客服机器人中，用户报错“错误代码 0x800405”，纯向量检索容易返回语义相近的“网络错误”文档，导致无效排查；引入 BM25 混合检索后，精准匹配到代码对应的KB文档，解决率提升 30%。

**6.4 多路检索与融合 (RRF)**
RRF (Reciprocal Rank Fusion) 不依赖分数尺度，仅根据排名融合，公式：$1/(k+rank)$。有效解决向量分数与 BM25 分数量纲不一致的问题。

**检索流程图**：
```text
   Query
     │
     ├──────────────┐
     ▼              ▼
 [向量库]        [全文库]
 ANN Search      BM25 Search
     │              │
     ▼              ▼
  List A         List B
     │              │
     └──────┬───────┘
            ▼
      [ 融合算法 ]
    (RRF / 加权求和)
            │
            ▼
      Final Ranked List
```

**面试 Q11：为什么混合检索比单路向量更有效？**
A：向量捕获语义，但可能漏掉专有名词；BM25 补精确匹配。二者融合提高鲁棒性，尤其在技术文档与电商场景。

**对比表格**：
| 维度 | RRF (倒数排名融合) | 加权 Score 融合 |
| :--- | :--- | :--- |
| **依赖性** | 仅依赖排名，不依赖分数绝对值 | 依赖分数归一化 (如 Min-Max) |
| **鲁棒性** | 强，避免单一高分主导 | 弱，需精细调参权重 |
| **实现难度** | 低，无需处理分数量纲 | 中，需处理不同分布的分数 |
| **适用场景** | 跨模态/多路召回通用 | 单模态内分数分布一致时 |

**6.5 查询改写**
将口语/模糊问题转化为利于检索的形式，或生成多路查询。
- **Query Expansion**: 添加同义词或扩展词。
- **Rewrite**: 使用 LLM 将 Query 改写为更匹配文档风格的形式。

**6.6 HyDE**
**H**ypothetical **D**ocument **E**mbeddings。先生成假设性答案，再用假设答案去检索，拉近 Query 与 Doc 的向量距离。

**面试 Q12：HyDE 适合什么场景？**
A：适合表述差距大、短查询难对齐长文档；不适合强事实约束且易胡编的领域。

**代码示例：RRF**
```python
def rrf_fuse(ranked_lists, k=60):
    scores = {}
    for ranks in ranked_lists:
        for rank, doc_id in enumerate(ranks, start=1):
            scores[doc_id] = scores.get(doc_id, 0) + 1.0 / (k + rank)
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

## 常见考点
1. **RRF 中的参数 k 一般取多少？**
   通常取 60，是一个经验值，用于平滑排名的影响。
2. **混合检索中 Reciprocal Rank Fusion 和 加权Score融合的区别？**
   RRF 不需要对分数归一化，实现简单鲁棒；加权融合需要对分数做 Min-Max 归一化或 Sigmoid 映射，否则某个强分值会主导结果。
3. **HyDE 失效的场景？**
   需要高精度的数值回答或极度依赖精确关键词匹配时，生成的假设答案可能引入噪声。

## 结构化回答

**30 秒电梯演讲：** 混合检索就是"语义+关键词"双管齐下——向量检索懂意思但漏专有名词，BM25 抠字眼能匹配 ID 和型号，两路召回再用 RRF 融合排名，鲁棒性立刻上一个台阶。用户问"错误代码 0x800405"，纯向量会返回不相干的网络错误文档，加了 BM25 才精准命中。

**展开框架：**
1. **双路互补** — 向量捕获语义适合模糊提问，BM25 补精确匹配适合 ID/型号/代码，融合后两者都不漏。
2. **RRF 是融合首选** — 倒数排名融合公式 1/(k+rank)，只看排名不看分数，天然解决向量分数和 BM25 分数量纲不一致的问题。
3. **Query 改写弥合差异** — 把口语转成文档风格、扩展同义词、或生成多路查询提升召回。
4. **HyDE 桥梁策略** — 先让 LLM 生成假设答案，再用假设答案去检索，适合短查询对齐长文档的场景。

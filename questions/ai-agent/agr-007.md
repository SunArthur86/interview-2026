# RRF(Reciprocal Rank Fusion)为什么比加权分数融合更鲁棒

RRF（Reciprocal Rank Fusion，倒数排名融合）用 `score(d) = Σ 1/(k + rank_i(d))` 聚合多路检索结果，只看每路检索器中文档的**排名**而完全忽略原始分数。相比加权分数融合，RRF 天然规避了不同检索器分数量纲不一致的问题（BM25 的 0-100 vs 向量的 0-1），对异常分数鲁棒，参数 k 通常取 60 通用性强无需微调，是多路异构检索（BM25+向量）融合的主流方案。

## 技术原理

- **只看排名不看分数**：RRF 用 `score(d) = Σ 1/(k + rank_i(d))` 聚合多个检索器的结果。它把每个检索器对文档的排名 `rank` 作为唯一输入，完全忽略检索器返回的原始分（如 BM25 的 15.3 或向量的 0.87），因此天然规避了跨检索器的分数量纲不一致问题。
- **天然归一化，鲁棒性强**：由于 `1/(k+rank)` 的取值范围固定在 `(0, 1/k]`（k=60 时上界约 0.0167），任何单一检索器的极端高分或离群分数最多只能通过它在该检索器中的排名贡献有限权重，不会"绑架"最终排序，从而对异常分数和分布偏移都很鲁棒。
- **避免单一系统偏差**：不同检索器（BM25/TF-IDF/向量/学习排序）各有偏好，加权融合需要为每个检索器手动调权重，且权重对数据分布敏感。RRF 让每个检索器只通过"排名投票"贡献，天然实现多源信息的平等融合，降低了对单一系统系统性偏差的依赖。

## 公式与对比

RRF 核心公式（k 通常取 60）：

```
score(d) = Σ_{i=1}^{N}  1 / (k + rank_i(d))
```

| 维度 | 加权分数融合 | RRF（倒数排名融合） |
|------|--------------|----------------------|
| 输入信号 | 原始分数（量纲不一） | 排名（无量纲） |
| 归一化 | 需要 Min-Max/Z-score，易失真 | 天然归一化 |
| 权重调优 | 需为每个检索器调权重 | 仅一个超参 k |
| 对异常分数 | 敏感，易被极值主导 | 鲁棒，极值影响有限 |
| 适用场景 | 同质检索器、分数已校准 | 异构检索器（BM25+向量） |

## 代码示例

Elasticsearch 内置 RRF，可直接组合 BM25 与 kNN 向量检索：

```json
GET /products/_search
{
  "size": 10,
  "retriever": {
    "rrf": {
      "rank_window_size": 50,
      "rank_constant": 60,
      "retrievers": [
        { "standard": { "query": { "match": { "title": "无线耳机" } } } },
        { "knn": { "field": "embedding", "query_vector": [0.1, 0.2, "..."], "num_candidates": 100 } }
      ]
    }
  }
}
```

Python 手写 RRF 融合两路结果：

```python
def rrf_fuse(rankings, k=60):
    scores = {}
    for ranking in rankings:                 # rankings: list of list[doc_id]
        for rank, doc in enumerate(ranking, start=1):
            scores[doc] = scores.get(doc, 0) + 1.0 / (k + rank)
    return [doc for doc, _ in sorted(scores.items(), key=lambda x: -x[1])]

bm25_hits = [101, 203, 305, 407]              # BM25 排名
vec_hits  = [305, 101, 888, 407]              # 向量检索排名
print(rrf_fuse([bm25_hits, vec_hits]))        # 305, 101 在两路都靠前 → 综合靠前
```

## 常见坑/注意事项

- **k 不是越小越好**：k 太小（如 k=1）会让头部排名（rank=1）权重过大，退化成"谁第一谁说了算"；k 太大（如 k=1000）会抹平排名差异。k=60 是论文经验值，适用大多数场景，无需针对数据集微调。
- **仅靠排名会丢失分数信息**：如果某个检索器的分数本身包含重要语义（如"非常相关"vs"勉强相关"），RRF 会丢失这种程度差异，极端情况下可能不如校准良好的加权融合。
- **候选集对齐问题**：若两路检索器的候选文档 ID 空间不一致（如一路是文档、一路是段落），需先做 ID 归一化或映射，否则融合无效。

## 记忆要点

- RRF使用倒数排名融合(1/(k+rank))，不依赖分数绝对值，只看相对排序。
- 加权融合受不同检索器分数量纲影响(如0-1 vs 0-100)，需复杂归一化。
- RRF对异常分数鲁棒，且参数k(通常60)通用性强，无需针对数据集微调。
- 结论：RRF消除了分数校准的难题，在多路异构检索器融合中更稳定。
